## 使用指南

### 何时创建调研报告？

- ✅ Phase 0 识别出知识缺口，需要系统性调研
- ✅ 遇到复杂的技术决策，需要充分的证据支撑
- ✅ 对某个领域缺乏了解，影响任务进行
- ✅ 需要验证某个假设或最佳实践

### 调研报告的价值

1. **避免重复踩坑**：下次遇到类似问题，直接参考报告
2. **支撑决策**：ADR 可以引用调研报告作为证据
3. **知识传承**：新的 agent 或开发者可以快速学习
4. **持续改进**：发现问题和解决方法的完整记录

### 调研完成后

- [ ] 将关键结论提取到相关知识资产（ADR、Patterns、Lessons）
- [ ] 更新 `knowledge-gaps.md`，标记缺口已填补
- [ ] 更新 `improvement-backlog.md`，标记调研任务完成
- [ ] 在调研报告索引中添加条目

---

**记住**：好的调研报告不仅解答当前问题，更为未来的决策提供坚实基础。
