### DS-001: 采用"感受驱动设计"作为核心方法论

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary & Guardian

#### 上下文

在成立工作室时，需要确定核心的设计方法论。市场上主流的方法包括：
- 功能驱动设计（Feature-Driven Design）
- 数据驱动设计（Data-Driven Design）
- 用户体验驱动设计（UX-Driven Design）
- 感受驱动设计（Feeling-Driven Design）

#### 问题陈述

如何确保工作室的设计作品具有独特的品质和深度，而不是成为功能堆砌或数据优化的产物？

#### 选项分析

##### 选项 A: 功能驱动设计

- 描述: 以功能列表为中心，满足用户需求
- 优点: 清晰、可量化、易于管理
- 缺点: 容易导致功能堆砌，缺乏灵魂
- 技术风险: 低
- 实现成本: 低

##### 选项 B: 数据驱动设计

- 描述: 以数据指标为导向，优化用户行为
- 优点: 可衡量、科学、可持续优化
- 缺点: 容易陷入局部优化，忽视整体体验
- 技术风险: 中（需要数据基础设施）
- 实现成本: 中

##### 选项 C: 感受驱动设计

- 描述: 以玩家目标感受为起点，设计所有细节
- 优点: 作品有灵魂，体验深刻，容易被记住
- 缺点: 主观、难以量化、依赖设计师能力
- 技术风险: 低
- 实现成本: 高（需要更多时间打磨）

#### 决策

选择 **选项 C: 感受驱动设计**

#### 理由

1. **差异化竞争**: 市场上功能驱动和数据驱动的产品已经很多，感受驱动设计可以让我们脱颖而出
2. **深度体验**: 我们追求的是让玩家记住的作品，而非快速消费的产品
3. **符合定位**: "奢侈品式设计"的核心就是对细节和感受的极致追求
4. **长期价值**: 功能会过时，但感受和体验会被记住
5. **创始人优势**: Visionary 擅长感受设计，Guardian 擅长把控质量，形成互补

#### 后果

**正面**:
- 作品具有独特性和辨识度
- 建立工作室的品牌认知（"有灵魂的游戏设计"）
- 吸引重视品质的玩家群体
- 设计过程更有意义和成就感

**负面**:
- 开发周期可能更长
- 评价标准更主观，需要建立内部共识
- 对设计师能力要求更高
- 难以用数据说服外部利益相关者

**需要注意**:
- 必须建立明确的感受评审标准，避免过度主观
- 需要定期回顾和验证感受目标是否达成
- 要平衡理想和可行性，避免无限延期

#### 实施要点

- [x] 建立"目标感受"作为设计文档的必需章节
- [ ] 制定感受评审的流程和标准
- [ ] 积累感受设计的案例库和模式库

### DS-002: 建立双创始人对话制决策机制

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary & Guardian

#### 上下文

在采用感受驱动设计后，需要确定如何做出设计决策。单一决策者容易陷入盲区，完全民主化决策则效率低下。

#### 问题陈述

如何建立既能保证决策质量，又能保持效率的决策机制？

#### 选项分析

##### 选项 A: 单一创始人决策制

- 描述: 一位创始人负责所有决策
- 优点: 效率高、方向统一
- 缺点: 容易陷入个人盲区，缺乏制衡
- 实现成本: 低

##### 选项 B: 完全民主投票制

- 描述: 团队投票决定所有事项
- 优点: 民主、多样化视角
- 缺点: 效率低、容易折中妥协
- 实现成本: 高

##### 选项 C: 双创始人对话制

- 描述: 两位创始人通过对话达成共识
- 优点: 平衡理想与现实、互补盲区、效率适中
- 缺点: 需要两人能够有效沟通和妥协
- 实现成本: 中

#### 决策

选择 **选项 C: 双创始人对话制**

#### 理由

1. **互补性**: Visionary 关注感受和愿景，Guardian 关注逻辑和瑕疵，两者互补
2. **制衡**: 避免单一视角的盲区和极端化
3. **效率**: 比民主投票更高效，比独裁更深思熟虑
4. **质量**: 通过对话达到更深的理解和更好的方案
5. **文化**: 体现工作室"对话式决策"的文化

#### 后果

**正面**:
- 决策质量更高（两个视角的碰撞）
- 避免盲区和极端
- 建立良好的讨论和思考文化
- 决策有充分的理由支撑

**负面**:
- 需要更多时间达成共识
- 两位创始人需要保持良好的沟通
- 可能出现僵持不下的情况

**需要注意**:
- 建立对话的基本规则和流程
- 明确哪些事项需要双方共同决策
- 建立僵持时的打破机制

#### 实施要点

- [x] 在宣言中明确双创始人对话制
- [ ] 制定对话决策的流程文档
- [ ] 建立决策记录的模板和规范
- [ ] 定期回顾决策机制的有效性

### DS-003: 定位为"奢侈品式游戏设计工作室"

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary & Guardian

#### 上下文

在确定工作室方法论后，需要明确工作室的市场定位和品牌形象。游戏行业有多种定位方式：
- 量产型工作室（快速迭代，多产品线）
- 技术驱动型工作室（技术创新）
- 内容驱动型工作室（故事、IP）
- 品质驱动型工作室（精品、细节）

#### 问题陈述

如何让工作室在市场中有清晰的辨识度和定位？

#### 选项分析

##### 选项 A: 快速迭代型工作室

- 描述: 快速推出多款产品，测试市场
- 优点: 降低单一产品失败风险，快速学习
- 缺点: 容易陷入平庸，缺乏品牌积累
- 适合度: 不适合（与感受驱动设计理念冲突）

##### 选项 B: 技术创新型工作室

- 描述: 以技术创新为卖点
- 优点: 差异化明显，技术壁垒
- 缺点: 技术不一定带来好体验，投入大
- 适合度: 一般（不是核心优势）

##### 选项 C: 奢侈品式精品工作室

- 描述: 追求极致品质和细节，慢工出细活
- 优点: 品牌溢价，忠诚用户，长期价值
- 缺点: 周期长，风险高，对能力要求高
- 适合度: 高（与感受驱动设计完美契合）

#### 决策

选择 **选项 C: 奢侈品式精品工作室**

#### 理由

1. **一致性**: 与"感受驱动设计"理念完美契合
2. **差异化**: 游戏行业鲜有明确以"奢侈品"自居的工作室
3. **价值主张**: 清晰传达"我们追求品质而非数量"
4. **用户定位**: 吸引重视品质、愿意等待的玩家
5. **长期价值**: 建立品牌认知和溢价能力
6. **文化认同**: 符合创始人的价值观和追求

#### 后果

**正面**:
- 品牌定位清晰，容易传播
- 吸引志同道合的玩家和合作者
- 为高品质正名，获得更多耐心
- 建立长期品牌价值

**负面**:
- 需要持续证明品质
- 承受更高的期待和审视
- 容易被模仿定位但难以模仿实质
- 需要更长的产品周期

**需要注意**:
- "奢侈品"不是借口，必须真正做到极致品质
- 需要在每个作品中体现"奢侈品级"的细节
- 避免自大和脱离实际
- 持续学习奢侈品行业的品质管理方法

#### 实施要点

- [x] 在宣言中明确"奢侈品式设计"理念
- [ ] 研究奢侈品行业的品质标准和方法
- [ ] 在所有对外沟通中强化这一定位
- [ ] 建立与定位相符的品质评审标准
- [ ] 积累能体现"奢侈品级"细节的案例

### FD-001: Fishing 项目 - 采用状态机设计钓鱼流程

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

钓鱼系统是 Fishing 项目的核心输入端，需要设计一个短周期、高频次的采集玩法。需要确定如何组织钓鱼流程的逻辑结构。

#### 问题陈述

如何设计钓鱼流程，才能既简单易懂，又有足够的反馈和趣味性？

#### 选项分析

##### 选项 A: 简单点击获取

- 描述: 点击钓点 → 立即获得鱼
- 优点: 简单快速
- 缺点: 无期待感、无惊喜、太机械
- 感受评估: ❌ 无法传递"期待 → 惊喜"

##### 选项 B: 简化状态机（投竿 → 等待 → 成功）

- 描述: 无咬钩触发，等待后自动成功
- 优点: 流程简单
- 缺点: 缺少玩家参与感和技巧要素
- 感受评估: ⚠️ 期待感有，但缺少惊喜触发时刻

##### 选项 C: 完整状态机（投竿 → 等待 → 咬钩 → 收杆 → 结果）

- 描述: 有明确的咬钩触发和收杆反应窗口
- 优点: 节奏清晰，有参与感，有技巧要素
- 缺点: 实现复杂度稍高
- 感受评估: ✅ 完整传递"期待 → 惊喜 → 满足"

#### 决策

选择 **选项 C: 完整状态机**

**状态设计**:
```
IDLE → CASTING → WAITING → HOOKED → SUCCESS/FAILED
```

#### 理由

1. **感受完整性**: 每个状态对应明确的感受（期待/紧张/兴奋/满足）
2. **参与感**: 咬钩时刻需要玩家反应，增加参与感
3. **节奏感**: 投竿-等待-触发-结算，有张有弛
4. **反馈丰富**: 每个状态转换都可以有视觉/听觉反馈
5. **可扩展性**: 状态机结构清晰，便于后续扩展（如 QTE、多阶段等）

#### 后果

**正面**:
- 玩家体验完整的情感流程
- 钓鱼不是单调的点击，有节奏和惊喜
- 技术实现清晰（状态机是经典模式）
- 易于调优（每个状态的时间和反馈可独立调整）

**负面**:
- 实现复杂度高于简单点击
- 需要平衡反应窗口的难度

**关键参数**:
- 投竿时间: 0.5s
- 等待时间: 3-15s（随机）
- 咬钩窗口: 2s
- 结算时间: 2s

### FD-002: Fishing 项目 - 产消比设定为 3:1 以上

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

放置游戏需要平衡资源产出和消耗的关系。产出太少玩家卡关，产出太多升级无意义。

#### 问题陈述

如何确定合理的产消比，让玩家感到富足但仍有升级动力？

#### 选项分析

##### 选项 A: 产消比 1:1（紧平衡）

- 描述: 产出刚好够升级
- 优点: 资源紧张，每次选择都有重量
- 缺点: 玩家感到紧缺、压力大、无优化空间
- 适合: 策略游戏、生存游戏

##### 选项 B: 产消比 2:1（温和富余）

- 描述: 产出是消耗的 2 倍
- 优点: 有一定余量
- 缺点: 富余不够明显，优化空间有限
- 适合: 平衡型游戏

##### 选项 C: 产消比 3-5:1（明显富余）

- 描述: 产出是消耗的 3-5 倍
- 优点: 富足感强、有大量优化空间、容错率高
- 缺点: 可能导致后期资源过剩
- 适合: 放置游戏、养成游戏

#### 决策

选择 **选项 C: 产消比 3-5:1**

**实际数据**:
- 30 分钟：产出 27,135 / 消耗 1,318 = **20.6:1**
- 2 小时：产出 289,000 / 消耗 7,486 = **38.6:1**

#### 理由

1. **放置游戏特性**: 放置游戏强调"富足感"而非"紧缺感"
2. **容错率**: 玩家可以尝试不同的展示配置，无需担心资源浪费
3. **正反馈**: 持续的金币增长给予正向反馈
4. **目标感**: 虽然钱多，但下一个升级目标仍然需要一定时间积累
5. **扩展空间**: MVP 允许金币富余，为后续内容（展示位扩展、装饰等）预留消耗点

#### 后果

**正面**:
- 玩家感到"有钱花"的满足感
- 可以自由尝试不同的展示配置
- 避免卡关导致的挫败感
- 升级间隔仍然合理（30-120 秒）

**负面**:
- 后期金币过多，需要补充消耗点
- 资源管理的策略深度降低

**缓解措施**:
- 后续版本加入更多消耗点（展示位扩展、装饰、新鱼竿等）
- 在 UI 中明确标注"后续版本将加入更多内容"

#### 相关

- 相关决策: 无
- 相关模式: [经济平衡设计](skills/designPatterns/economy-balance-pattern.md)
- 来源文档: `projects/fishing/design/economy.md`

### FD-003: Fishing 项目 - 咬钩窗口设为 2 秒

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

在状态机设计中，咬钩阶段需要玩家在限定时间内按收杆键。窗口时间直接影响难度和挫败感。

#### 问题陈述

咬钩窗口应该设置为多长时间，才能既有参与感，又不会太难导致挫败？

#### 选项分析

##### 选项 A: 窗口 1 秒

- 描述: 咬钩后必须在 1 秒内收杆
- 优点: 有挑战性，考验反应速度
- 缺点: 对反应慢的玩家不友好，挫败感强
- 适合: 动作游戏、硬核游戏

##### 选项 B: 窗口 2 秒

- 描述: 咬钩后有 2 秒的反应时间
- 优点: 宽松但需要注意力，适合大部分玩家
- 缺点: 对硬核玩家可能太简单
- 适合: 休闲游戏、放置游戏

##### 选项 C: 窗口 3 秒以上

- 描述: 咬钩后有 3+ 秒的反应时间
- 优点: 极度宽松，几乎不会失败
- 缺点: 失去参与感和技巧性
- 适合: 完全放置类游戏

#### 决策

选择 **选项 B: 窗口 2 秒**

#### 理由

1. **目标受众**: Fishing 是休闲放置游戏，面向 12-25 岁Fortnite 玩家
2. **参与感**: 2 秒需要注意力，但不会太紧张
3. **失败率**: 预期失败率 15%（成功率 85%），属于合理范围
4. **感受平衡**: 既有"及时收杆"的成就感，又不会频繁失败导致挫败
5. **国际参考**: 主流钓鱼游戏的反应窗口通常在 1.5-3 秒

#### 后果

**正面**:
- 大部分玩家能成功收杆（85%）
- 少量失败提供轻微挑战感
- 不会因为难度劝退玩家
- 老手玩家也能偶尔走神失败（避免太无聊）

**负面**:
- 硬核玩家可能觉得太简单

**调优空间**:
- 如果测试发现太难：延长至 2.5s
- 如果测试发现太简单：缩短至 1.5s

### FD-004: Fishing 项目 - 使用指数递增的稀有度收益

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

展示系统中，不同稀有度的鱼会产生不同的收益。需要确定收益倍率的递增方式。

#### 问题陈述

如何设计稀有度收益倍率，才能让稀有鱼有明显价值，同时创造展示位优化的动力？

#### 选项分析

##### 选项 A: 线性递增（1x, 2x, 3x, 4x, 5x）

- 描述: 每个稀有度增加 1 倍
- 优点: 简单直观
- 缺点: 稀有鱼价值感不足，优化动力弱
- 数据: Epic(5x) 仅比 Common(1x) 高 5 倍

##### 选项 B: 平方递增（1x, 4x, 9x, 16x, 25x）

- 描述: 稀有度等级的平方
- 优点: 差距明显
- 缺点: 后期差距过大，可能导致只用最稀有的
- 数据: Epic(25x) 比 Common(1x) 高 25 倍

##### 选项 C: 指数递增（1x, 2x, 5x, 15x）

- 描述: 每个等级 2-3 倍递增
- 优点: 平衡，稀有有价值但不至于过度
- 缺点: 需要精细调优
- 数据: Epic(15x) 比 Common(1x) 高 15 倍

#### 决策

选择 **选项 C: 指数递增（1x, 2x, 5x, 15x）**

**具体倍率**:
- Common: 1x (基础)
- Uncommon: 2x (2 倍)
- Rare: 5x (2.5 倍递增)
- Epic: 15x (3 倍递增)

#### 理由

1. **明显价值**: Rare 比 Common 高 5 倍，Epic 比 Common 高 15 倍，价值感明显
2. **优化动力**: 替换一条 Common 为 Epic，收益从 1/秒 → 15/秒，巨大提升
3. **策略深度**: 3 个展示位的配置有多种组合，收益差异明显
4. **成长感**: 随着钓到更稀有的鱼，收益显著增长
5. **平衡性**: 不至于让低稀有完全无用（早期仍需使用）

#### 后果

**正面**:
- 钓到稀有鱼时的惊喜感更强（收益提升巨大）
- 展示位优化有明显效果
- 收集完整性有强烈动力
- 成长曲线平滑且有满足感

**负面**:
- 后期低稀有鱼基本无用（但符合游戏设计）

**配置示例**:
```
3x Common:    3 金币/秒
1x Rare + 2x Common: 7 金币/秒 (+133%)
3x Rare:      15 金币/秒 (+400%)
3x Epic:      45 金币/秒 (+1400%)
```

- 相关决策: 无

### FD-005: Fishing 项目 - 环境联动采用响应式设计

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

在设计环境联动系统（天气、海面、镜头等）时，需要确定环境系统与钓鱼状态机的关系：是独立运行还是响应式？

#### 问题陈述

环境系统应该如何与钓鱼玩法交互？

#### 选项分析

##### 选项 A: 独立运行

- 描述: 环境系统独立运行，有自己的时间轴
- 优点: 技术实现简单，环境更"真实"（如天气自然变化）
- 缺点: 玩家行为与环境脱节，缺乏因果关系

##### 选项 B: 响应式设计

- 描述: 环境系统监听钓鱼状态变化，响应式执行
- 优点: 玩家能感受到"我的行为影响了世界"，因果关系清晰
- 缺点: 技术实现稍复杂（需要事件系统）

#### 决策

选择 **选项 B: 响应式设计**

**架构**:
```
钓鱼状态机 (核心)
    │
    ├─ 通知 → 天气管理器 → 响应
    ├─ 通知 → 海面管理器 → 响应
    ├─ 通知 → 镜头管理器 → 响应
    ├─ 通知 → 特效管理器 → 响应
    └─ 通知 → 音效管理器 → 响应
```

#### 理由

1. **因果关系**: 玩家能清晰感受到"我投竿 → 海鸥惊飞"
2. **沉浸感**: 环境响应玩家行为，世界"活"了
3. **可控性**: 技术实现清晰，易于调试
4. **扩展性**: 未来添加新环境系统只需监听事件

#### 后果

**正面**:
- 玩家行为与环境同步，沉浸感强
- 技术架构清晰（事件驱动）
- 易于扩展新环境系统

**负面**:
- 需要实现事件系统（额外工作量）
- 环境变化可能显得"过于响应"（需要调优）

### FD-006: Fishing 项目 - 设计 3 种天气类型

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

需要确定设计多少种天气类型来创造差异化的钓鱼体验。

#### 问题陈述

应该设计多少种天气，才能在深度和广度之间达到最佳平衡？

#### 选项分析

##### 选项 A: 2 种（晴天 + 暴风雨）

- 优点: 对比强烈，易于实现
- 缺点: 变化太少，缺少中间状态的情感层次

##### 选项 B: 3 种（晴天 + 阴天 + 暴风雨）

- 优点: 覆盖完整的情感光谱（愉悦/沉思/刺激），差异化明显
- 缺点: 需要为每种天气设计完整的感受体验

##### 选项 C: 5 种（晴天 + 阴天 + 雨天 + 暴风雨 + 雾天）

- 优点: 变化丰富
- 缺点: 感受差异模糊（雨天 vs 暴风雨？），难以做到极致差异化

#### 决策

选择 **选项 B: 3 种天气**

**天气定义**:
1. **晴天** (60%)：温柔的海洋，愉悦放松
2. **阴天** (30%)：沉思的海洋，专注平静
3. **暴风雨** (10%)：愤怒的海洋，刺激挑战

#### 理由

1. **情感完整性**: 三种天气覆盖主要情感区间（愉悦-沉思-刺激）
2. **极致差异化**: 每种天气都能做到极致，感受完全不同
3. **叙事深度**: 每种天气有足够空间发展独特的叙事和氛围
4. **未来扩展**: 为后续天气类型（黄昏、夜晚、雾天）留下空间

#### 后果

**正面**:
- 玩家能清晰区分三种天气的独特感受
- 10% 暴风雨成为"特殊体验"，值得期待
- 每种天气都有深度，而非浅尝辄止

**负面**:
- 可能有玩家希望更多变化（可后续扩展）

#### 未来扩展方向

不是"现在不做"，而是"先做好这三种"：
- 🌅 黄昏钓鱼（温暖宁静）
- 🌙 夜间钓鱼（神秘浪漫）
- 🌫️ 雾天钓鱼（朦胧诗意）

### FD-007: Fishing 项目 - 暴风雨中加入慢动作效果

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

在设计暴风雨天气的 HOOKED 状态时，考虑是否加入慢动作效果以强化"史诗时刻"。

#### 问题陈述

暴风雨中的咬钩时刻是否应该加入慢动作效果？

#### 选项分析

##### 选项 A: 无慢动作

- 优点: 节奏统一，不打断流程
- 缺点: 暴风雨缺少"特殊感"

##### 选项 B: 加入慢动作（0.3s）

- 描述: HOOKED 触发时，世界慢动作 0.3 秒
- 优点: 强化"在混乱中抓住瞬间"的史诗感
- 缺点: 可能有人觉得打断流程

##### 选项 C: 加入慢动作 + 定格（0.2s 定格 + 0.3s 慢动作）

- 优点: 效果更戏剧化
- 缺点: 时间过长（0.5s），风险高

#### 决策

选择 **选项 B: 加入慢动作 0.3s**

**触发条件**:
- 仅在暴风雨天气
- 仅在 HOOKED 状态
- 可在设置中关闭

**效果细节**:
- 时间缩放: 50%（0.5x 速度）
- 持续时间: 0.3 秒
- 音效处理: 降低音高（Pitch -50%）
- 视觉强化: 雨点暂停或变慢

#### 理由

1. **心理真实性**: 极端环境下，关键时刻时间感知确实变慢
2. **史诗感**: 强化"在混乱中抓住瞬间"的感觉
3. **差异化**: 暴风雨成为"特殊体验"而非"困难版晴天"
4. **成功案例**: 类似《荒野之息》盾反、《战神》处决

#### 后果

**正面**:
- 暴风雨钓鱼成为"特殊体验"
- 强化玩家的成就感
- 创造记忆点

**负面**:
- 需要测试验证玩家反馈
- 可能不是所有人喜欢（需提供关闭选项）

**风险缓解**:
- 仅在暴风雨（10% 概率）
- 持续时间很短（0.3s）
- 可在设置中关闭
- 如果测试反馈差，可移除

### FD-008: Fishing 项目 - 环境不影响掉落率

**日期**: 2026-01-13  
**状态**: 已决定  
**决策者**: Visionary  
**项目**: Fishing

#### 上下文

在设计环境联动时，需要确定环境变化是否影响掉落率（如暴风雨掉落稀有鱼概率更高）。

#### 问题陈述

不同天气下的钓鱼掉落率应该相同还是不同？

#### 选项分析

##### 选项 A: 环境影响掉落率

- 描述: 暴风雨掉落稀有鱼概率 +50%
- 优点: 增加策略深度，暴风雨有"实际价值"
- 缺点: 玩家为了效率只在暴风雨钓鱼，破坏差异化体验

##### 选项 B: 环境不影响掉落率

- 描述: 所有天气掉落率完全相同
- 优点: 纯氛围差异，玩家选择天气基于感受偏好
- 缺点: 暴风雨没有"实际奖励"

#### 决策

选择 **选项 B: 环境不影响掉落率**

**核心规则**:
- 所有天气的钓鱼效率完全相同
- 所有天气的稀有鱼掉落率完全相同
- 环境仅影响氛围，不影响数值

#### 理由

1. **核心原则**: 环境是"氛围层"而非"玩法层"
2. **体验差异化**: 玩家选择天气基于"想要什么感受"而非"哪个更高效"
3. **避免优化陷阱**: 不会导致玩家只在某种天气钓鱼
4. **叙事一致性**: 海洋的礼物不因情绪变化而变得吝啬

#### 后果

**正面**:
- 玩家会尝试所有天气（体验差异化）
- 环境选择基于感受偏好而非数值优化
- 设计清晰（环境 = 氛围，不 = 玩法）

**负面**:
- 暴风雨缺少"实际奖励"（但有"特殊体验"）
- 一些玩家可能希望暴风雨有额外收益

**UI 明确告知**:
> "天气仅影响钓鱼氛围，不影响掉落率。选择你喜欢的感受吧！"

**未来扩展可能**:
- 暴风雨中出现"特殊鱼种"（不是更高概率，而是额外奖励）
- 例如："雷电鱼"仅在暴风雨出现（但不影响基础掉落）
