# 放置游戏核心循环模式 (Idle Game Core Loop Pattern)

> **来源**: Fishing 项目 - 核心循环设计  
> **适用场景**: 放置/养成类游戏的核心玩法循环  
> **目标感受**: 进步感 → 满足感 → 期待感（持续循环）

---

## 适用场景

当你需要设计一个：
- 主打"离线收益"和"自动产出"的游戏
- 玩家每次游玩 5-30 分钟，每天多次回归
- 强调收集、展示、数值成长的体验
- 适合碎片化时间游玩

**典型应用**：
- 放置 RPG（挂机打怪）
- 模拟经营（餐厅、农场）
- 收集养成（宠物、卡牌）
- 基地建造（展示馆、博物馆）

---

## 目标感受

### 核心感受循环

```
主动采集 (输入)
  ↓
获得战利品 (产出)
  ↓
放置展示 (投资)
  ↓
被动收益 (回报)
  ↓
升级强化 (进步)
  ↓
(返回主动采集，但能力更强)
```

**每个环节的感受**：

| 环节 | 核心感受 | 持续时间 | 情感强度 |
|------|----------|----------|----------|
| **主动采集** | 期待、参与感 | 10-30s/次 | 中 |
| **获得战利品** | 惊喜、成就感 | 2-3s | 高（尤其稀有时）|
| **放置展示** | 自豪、炫耀 | 5-10s | 中高 |
| **被动收益** | 满足、被欢迎 | 持续 | 低但持续 |
| **升级强化** | 进步、期待 | 30-120s/次 | 高 |

## 实现方式

### 1. 循环结构设计

```
┌─────────────────────────────────────────┐
│                                         │
▼                                         │
┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
│  钓鱼  │───▶│ 战利品 │───▶│  展示  │───▶│  收益  │
└────────┘    └────────┘    └────────┘    └────────┘
    ▲                                         │
    │         ┌────────┐                      │
    └─────────│  升级  │◀─────────────────────┘
              └────────┘
```

**设计要点**：

1. **输入端（主动采集）**
   - 玩家主动触发
   - 短周期（10-30s）
   - 有即时反馈
   - 产出随机性

2. **处理层（战利品/展示）**
   - 收集品有稀有度
   - 可视化展示
   - 空间有限（策略性）
   - 替换机制

3. **输出端（被动收益）**
   - 持续产出
   - 离线也累积
   - 与展示配置相关
   - 有上限（促进回归）

4. **反馈回路（升级）**
   - 消耗收益资源
   - 提升输入端能力
   - 加速整个循环
   - 有明确成长感

### 2. 时间节奏设计

**单次游玩周期**（15-30 分钟）：

```
0-5min: 快速循环建立（5-10 次采集）
  ↓
5-15min: 优化配置（调整展示位）
  ↓
15-30min: 完成升级，看到成长
  ↓
离线: 收益累积，等待回归
```

**每日多次回归**：
- 第一次（早上）：领取离线收益，快速几次采集
- 第二次（中午）：短暂游玩，完成升级
- 第三次（晚上）：专注游玩 30+ 分钟

**长期进度**（1-3 周）：
- Week 1: 核心循环熟悉，初期快速成长
- Week 2: 收集完成度 80%+，追求稀有
- Week 3: 满级，内容完成，等待更新

### 3. 数值平衡设计

**产出 vs 消耗平衡**：

```
产出速率 > 消耗速率 × 3
```

**设计理由**：
- 玩家需要感受到"有余"而非"紧缺"
- 富余资源促进尝试和优化
- 避免"刷钱"的负面循环

**成长曲线设计**：

```
收益/秒
  │
150├                                    ● 理想配置
  │                               ●
100├                          ●
  │                     ●
 50├               ●
  │          ●
 10├     ●
  │  ●
  0├─────┬─────┬─────┬─────┬─────▶ 游戏时间
      10min 30min  1h    2h
```

**曲线特征**：
- 早期陡峭（快速正反馈）
- 中期平稳（稳定成长）
- 后期放缓（延长内容寿命）

**升级成本递增**：

```
cost_n = base_cost × (multiplier ^ (n-1))

示例：
  base_cost = 100
  multiplier = 1.5
  
  Lv.2: 100
  Lv.3: 150
  Lv.4: 225
  Lv.5: 337
  ...
```

**关键指标**：
- 每次升级间隔: 30-120 秒
- 满级总时间: 1-3 小时
- 离线收益上限: 8-12 小时的主动收益

## 注意事项

### ⚠️ 必须避免的设计陷阱

1. **产出不足导致卡关**
   - ❌ 升级费用过高，玩家长时间无进展
   - ✅ 产出 > 消耗 × 3，保证持续进步感

2. **被动收益过高**
   - ❌ 离线收益 > 主动收益，玩家不再主动游玩
   - ✅ 被动收益仅补充，主动游玩更高效

3. **无策略深度**
   - ❌ 所有选择都一样，无优化空间
   - ✅ 展示位有限，需要策略配置

4. **稀有度无意义**
   - ❌ 稀有物品只是"数字大"
   - ✅ 稀有物品有独特视觉、音效、收益

5. **进度终点过早**
   - ❌ 1 小时就满级，无后续目标
   - ✅ MVP 控制在 2-3 小时，预告后续内容

### ✅ 必须做到的设计要点

1. **清晰的"下一步目标"**
   - 玩家永远知道"我接下来要做什么"
   - 进度 UI 明确显示下一个里程碑

2. **离线收益设计**
   - 离线也有收益（欢迎回归）
   - 但有上限（避免无限挂机）
   - 领取时有仪式感（弹窗、音效）

3. **展示的意义**
   - 展示不只是"放进仓库"
   - 有视觉变化（3D 模型、光效）
   - 其他玩家可见（社交价值）

4. **升级的反馈**
   - 升级后立即感受到能力提升
   - 视觉变化（鱼竿外观、特效）
   - 下一次采集就能验证效果

5. **节奏的"呼吸感"**
   - 快速循环（采集）+ 慢节奏（优化、升级）
   - 紧张（采集）+ 放松（收益累积）
   - 主动（游玩）+ 被动（离线）

## 变体与扩展

### 变体 1: 多资源型

**场景**: 有多种资源和多条成长线

```
采集 A → 资源 X → 展示 A → 收益 X → 升级 A
采集 B → 资源 Y → 展示 B → 收益 Y → 升级 B
```

**感受变化**: 增加深度，但需注意复杂度

### 变体 2: 社交型

**场景**: 加入访问和社交元素

```
核心循环 + 访问他人基地 → 点赞/留言 → 社交奖励
```

**感受变化**: 炫耀感、社交满足感

### 变体 3: 任务引导型

**场景**: 通过任务引导循环

```
接取任务 → 完成循环 → 领取奖励 → 解锁新任务
```

**感受变化**: 目标更明确，但可能限制自由

## 案例

### 案例 1: Fishing 项目

**循环结构**:
```
钓鱼 → 获得鱼 → 展示鱼 → 产出金币 → 升级鱼竿 → (更好的钓鱼)
```

**数值配置**:
- 单次钓鱼: 10 秒
- 展示位: 3 个
- 升级层级: Lv.1-10
- 满级时间: 2 小时

**效果评估**:
- 每分钟约 5 次钓鱼（输入端高频）
- 展示位有限（需策略配置）
- 产出 > 消耗 × 38（富余充足）
- 升级间隔 30-120 秒（节奏良好）

**成功指标**:
- 次日留存率 > 30%
- 单次游玩时长 > 10 分钟
- 循环完成率 > 80%

### 案例 2: 餐厅经营（概念扩展）

**循环结构**:
```
接待顾客 → 获得收益 → 升级菜品 → 吸引更多顾客 → (更高收益)
```

**循环特点**:
- 输入端: 自动（顾客自动上门）
- 展示层: 餐厅布局、菜单配置
- 被动收益: 持续营业
- 升级: 菜品、装修、服务员

## 设计检查清单

在使用本模式设计时，必须检查：

- [ ] 核心循环是否完整（输入→产出→展示→收益→升级→返回）？
- [ ] 单次循环时间是否在 10-30 秒？
- [ ] 产出是否 > 消耗 × 3？
- [ ] 是否有离线收益机制？
- [ ] 离线收益是否有上限？
- [ ] 展示是否有策略性（空间有限）？
- [ ] 升级是否有即时反馈？
- [ ] 稀有度是否有差异化体验？
- [ ] 是否有明确的"下一步目标"提示？
- [ ] 满级时间是否在 1-3 小时？
- [ ] 是否有后续内容预告？

## 与其他模式的关系

**组合使用**:
- **钓鱼循环模式**: 作为输入端的具体实现
- **收集系统模式**: 战利品管理
- **仪式设计模式**: 升级、开箱等关键时刻

**对比模式**:
- **策略游戏循环**: 更强调决策深度
- **动作游戏循环**: 更强调操作技巧
- **叙事游戏循环**: 更强调剧情推进

## 参数调优指南

| 参数 | 推荐值 | 调整范围 | 调整目的 |
|------|--------|----------|----------|
| 产消比 | 3:1 | 2:1 - 5:1 | 控制富余感 |
| 单次循环时间 | 10-30s | 5-60s | 控制节奏 |
| 升级间隔 | 30-120s | 20-300s | 控制成长速度 |
| 满级时间 | 2h | 1-5h | 控制内容寿命 |
| 离线收益上限 | 8h | 4-24h | 控制回归频率 |
| 展示位数量 | 3-5 | 2-10 | 控制策略深度 |

**调优场景**:

- **玩家反馈「升级太慢」**: 降低升级费用，或增加产出
- **玩家反馈「无事可做」**: 增加展示位，或增加鱼种
- **玩家反馈「太简单」**: 减少产出，或增加消耗
- **玩家反馈「离线收益太少」**: 提高离线收益倍率，或上限时间
