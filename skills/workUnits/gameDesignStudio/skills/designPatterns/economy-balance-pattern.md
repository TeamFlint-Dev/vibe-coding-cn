# 经济平衡设计模式 (Economy Balance Pattern)

> **来源**: Fishing 项目 - 经济设计  
> **适用场景**: 需要设计产出-消耗-成长的经济系统  
> **目标感受**: 富足感 → 进步感 → 期待感

---

## 适用场景

当你需要设计一个：
- 有资源产出和消耗的系统
- 需要平衡玩家成长速度的游戏
- 需要避免"通胀"或"缺钱"的经济系统
- 要让玩家感到"有进展"但不"太容易"

**典型应用**：
- 放置游戏的金币系统
- RPG 的经验/金币系统
- 经营游戏的收支平衡
- 抽卡游戏的货币系统

---

## 目标感受

### 核心感受目标

**富足感**（而非紧缺感）：
- 玩家应该感到"钱够用"
- 但不应该感到"钱太多没用"
- 余额应该持续增长（正反馈）

**进步感**（而非停滞感）：
- 每次升级后能立即感受到效果
- 升级间隔合理（不过快也不过慢）
- 成长曲线平滑（无断崖）

**期待感**（而非无聊感）：
- 始终有下一个目标
- 目标可达成但需努力
- 完成后有更远的目标

## 实现方式

### 1. 产消比设计

**核心公式**：

```
产出速率 ≥ 消耗速率 × 富足系数

推荐富足系数：3-5
```

**设计理由**：
- 系数 < 2：玩家感到"缺钱"，负面体验
- 系数 2-3：刚够用，无优化空间
- 系数 3-5：富足但不浪费，最佳体验
- 系数 > 5：钱太多，升级失去意义

**案例数据**（Fishing 项目）：

```
30 分钟游玩：
  产出：约 27,135 金币
  消耗：1,318 金币（升级到 Lv.6）
  富足系数：20.6（前期）

2 小时游玩：
  产出：约 289,000 金币
  消耗：7,486 金币（满级）
  富足系数：38.6（全流程）
```

### 2. 升级成本递增公式

**推荐公式**：

```
cost(n) = base_cost × (multiplier ^ (n - 1))

常用参数：
  multiplier = 1.3 - 2.0
  
示例（multiplier = 1.5）：
  Lv.2: 100
  Lv.3: 150
  Lv.4: 225
  Lv.5: 337
  Lv.6: 506
  ...
```

**multiplier 选择指南**：

| Multiplier | 成长速度 | 适用场景 |
|------------|----------|----------|
| 1.3 | 非常缓和 | 长线游戏，上百个等级 |
| 1.5 | 平衡 | 10-20 个等级，MVP 推荐 |
| 1.8 | 较快 | 5-10 个等级，短周期游戏 |
| 2.0 | 极快 | 3-5 个等级，超短流程 |

**cumulative cost 计算**：

```
total_cost(n) = Σ cost(i) for i = 2 to n

几何级数求和：
  total_cost(n) = base_cost × (multiplier^(n-1) - 1) / (multiplier - 1)
```

### 3. 收益成长曲线

**三段式曲线设计**：

```
收益/秒
  │
150├                                    ●───── 后期（缓和）
  │                               ●
100├                          ●
  │                     ●
 50├               ●
  │          ●
 10├     ●──────────────────────────── 早期（陡峭）
  │  ●
  0├─────┬─────┬─────┬─────┬─────▶ 游戏时间
      10min 30min  1h    2h
      
  早期（0-15min）: 快速成长（鼓励继续）
  中期（15-60min）: 平稳成长（稳定体验）
  后期（60min+）: 缓慢成长（延长内容）
```

**曲线参数**：

| 阶段 | 时间 | 收益增长率 | 设计目的 |
|------|------|-----------|----------|
| 早期 | 0-15min | +200%/5min | 快速正反馈 |
| 中期 | 15-60min | +100%/15min | 稳定进步 |
| 后期 | 60min+ | +50%/30min | 延长寿命 |

### 4. 升级节奏控制

**关键指标**：

```
升级间隔 = 升级费用 / 收益速率

目标范围：30 - 120 秒
```

**节奏分析**（Fishing 案例）：

| 升级 | 费用 | 收益速率 | 预计时间 | 评估 |
|------|------|----------|----------|------|
| Lv.1→2 | 100 | 3/秒 | 33s | ✅ 快速正反馈 |
| Lv.2→3 | 150 | 5/秒 | 30s | ✅ 持续流畅 |
| Lv.3→4 | 225 | 7/秒 | 32s | ✅ 节奏稳定 |
| Lv.5→6 | 506 | 10/秒 | 51s | ✅ 适度延长 |
| Lv.9→10 | 2,562 | 40/秒 | 64s | ✅ 终局挑战 |

**节奏调优规则**：

- 间隔 < 20s：过快，失去期待感
- 间隔 20-60s：流畅，推荐早中期
- 间隔 60-120s：有张力，推荐后期
- 间隔 > 120s：过慢，可能流失玩家

### 5. 稀有度经济影响

**稀有度收益倍率设计**：

```
收益倍率应呈指数增长

推荐倍率：
  Common:    1x   (基础)
  Uncommon:  2x   (2倍)
  Rare:      5x   (2.5倍递增)
  Epic:      15x  (3倍递增)
  Legendary: 50x+ (3倍以上递增)
```

**设计理由**：

- **线性递增**（1x, 2x, 3x, 4x）：
  - ❌ 稀有鱼价值感不足
  - ❌ 替换动力不足

- **指数递增**（1x, 2x, 5x, 15x）：
  - ✅ 稀有鱼有明显价值
  - ✅ 强烈的替换动力
  - ✅ 收集完整性奖励

**展示位优化动力**：

```
场景：3 个展示位

配置 A: 3x Common = 3 金币/秒
配置 B: 1x Rare + 2x Common = 7 金币/秒 (+133%)
配置 C: 3x Rare = 15 金币/秒 (+400%)
配置 D: 3x Epic = 45 金币/秒 (+1400%)

结论：优化展示位有巨大收益提升
```

## 注意事项

### ⚠️ 必须避免的设计陷阱

1. **通货膨胀失控**
   - ❌ 后期金币数百万，但没有消耗点
   - ✅ 预留后续消耗点（展示位扩展、装饰等）

2. **早期缺钱卡关**
   - ❌ 首次升级需要 1 小时积累
   - ✅ 早期升级快速（3-5 分钟内完成首次）

3. **升级成本爆炸**
   - ❌ Multiplier 过高（如 2.5），后期指数爆炸
   - ✅ 控制在 1.3-2.0 区间

4. **稀有度收益差距过小**
   - ❌ Common 1, Uncommon 1.2, Rare 1.5（无替换动力）
   - ✅ 指数递增，让稀有物有明显价值

5. **无消耗循环**
   - ❌ 只有产出无消耗，金币无意义
   - ✅ 确保有持续的消耗点

### ✅ 必须做到的设计要点

1. **产出 > 消耗**
   - 玩家应该感到富足而非紧缺
   - 富足系数 3-5 为最佳

2. **升级可感知**
   - 升级后立即能感受到能力提升
   - 下一次循环就能验证效果

3. **成长曲线平滑**
   - 避免断崖式的费用跳跃
   - 每个等级的间隔时间相近

4. **稀有度有意义**
   - 稀有物的收益显著高于普通物
   - 创造替换动力

5. **预留扩展空间**
   - MVP 允许金币富余
   - 为后续内容预留消耗点

## 变体与扩展

### 变体 1: 双货币系统

**场景**: 有"软货币"和"硬货币"

```
软货币（金币）：
  - 产出快
  - 用于常规升级
  - 可离线获得

硬货币（钻石）：
  - 产出慢
  - 用于特殊内容
  - 或付费购买
```

### 变体 2: 多阶段升级

**场景**: 升级有多个子项

```
鱼竿升级：
  ├─ 竿身强度 (Lv.1-10)
  ├─ 鱼线韧性 (Lv.1-10)
  └─ 鱼钩锋利 (Lv.1-10)

每个子项独立升级，共同影响能力
```

### 变体 3: 声望系统

**场景**: 用"软货币"换取"声望"

```
金币 → 捐献 → 声望 → 解锁特权

特权示例：
  - 更多展示位
  - 独特外观
  - 社交功能
```

## 案例

### 案例 1: Fishing 项目

**资源设计**:
- 主货币：金币（产出 + 消耗）
- 收集品：鱼（放置产出金币）

**产消比**:
- 30 分钟：27,135 / 1,318 = 20.6
- 2 小时：289,000 / 7,486 = 38.6

**升级设计**:
- 基础费用：100
- Multiplier：1.5
- 等级数：10
- 满级费用：7,486

**效果评估**:
- ✅ 玩家始终富足（产出 >> 消耗）
- ✅ 升级节奏良好（30-120s 间隔）
- ✅ 稀有度有价值（1x → 15x）
- ⚠️ 后期金币过多（需扩展消耗点）

### 案例 2: RPG 升级系统（概念扩展）

**资源设计**:
- 经验值（产出：打怪）
- 金币（产出：打怪，消耗：装备）

**升级曲线**:
```
exp_required(n) = 100 × n^2

Lv.1→2: 100 exp
Lv.2→3: 400 exp
Lv.3→4: 900 exp
...
```

**平衡点**:
- 平均打怪 exp：10/只
- 升级间隔：10-90 只怪
- 与关卡推进速度匹配

## 设计检查清单

在使用本模式设计时，必须检查：

- [ ] 产出是否 > 消耗 × 3？
- [ ] 升级费用 Multiplier 是否在 1.3-2.0 范围？
- [ ] 升级间隔是否在 30-120 秒？
- [ ] 稀有度收益是否呈指数增长？
- [ ] 早期升级是否足够快（5 分钟内首次升级）？
- [ ] 后期是否有金币消耗点？
- [ ] 成长曲线是否平滑无断崖？
- [ ] 满级时间是否在预期范围（1-3 小时）？
- [ ] 是否预留了后续扩展空间？
- [ ] 所有数值是否可配置（支持热更新）？

## 与其他模式的关系

**组合使用**:
- **放置循环模式**: 经济系统是循环的核心资源
- **进度系统模式**: 经济平衡影响进度节奏
- **收集系统模式**: 稀有度影响经济价值

**对比模式**:
- **固定价格系统**: 所有物品固定价格（无成长）
- **物物交换系统**: 无货币，直接交换

## 参数调优指南

| 参数 | 推荐值 | 调整范围 | 调整目的 |
|------|--------|----------|----------|
| 富足系数 | 3-5 | 2-10 | 控制富足感 |
| Multiplier | 1.5 | 1.3-2.0 | 控制成长速度 |
| 基础费用 | 100 | 50-500 | 控制早期节奏 |
| 升级间隔 | 30-120s | 20-300s | 控制体验节奏 |
| 稀有倍率 | 1/2/5/15 | 按需调整 | 控制替换动力 |

**调优场景**:

- **玩家反馈「钱不够」**: 增加产出，或降低费用
- **玩家反馈「升级太慢」**: 降低 Multiplier，或增加产出
- **玩家反馈「钱太多没用」**: 增加消耗点，或降低产出
- **玩家反馈「稀有鱼没价值」**: 增加稀有倍率差距
