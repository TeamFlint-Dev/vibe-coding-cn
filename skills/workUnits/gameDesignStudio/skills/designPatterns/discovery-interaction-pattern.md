# 发现式交互设计模式 (Discovery-Based Interaction Pattern)

> **来源**: Fishing 项目 - IDLE-F-01 "发现钓点"设计  
> **适用场景**: 需要玩家主动发现并触发的交互点设计  
> **目标感受**: 好奇 → 探索 → 发现 → 满足

---

## 适用场景

当你需要设计一个让玩家"主动发现"而非"被动提示"的交互系统时，使用此模式。

**典型应用**:
- 发现隐藏的采集点/宝箱/NPC
- 探索开放世界中的兴趣点
- 解谜游戏中的线索发现
- RPG 中的秘密区域发现

**不适用**:
- 主线剧情的关键触发点（需要明确引导）
- 新手教学阶段（玩家还不熟悉探索）
- 时间敏感的交互（如战斗中的快速反应）

---

## 目标感受

### 核心感受链

```
好奇 (Curiosity)
   ↓
探索 (Exploration)
   ↓
发现 (Discovery)
   ↓
满足 (Satisfaction)
```

**感受演变**:
1. **好奇**: "那是什么？" - 微弱吸引，可选择忽略
2. **探索**: "去看看吧" - 主动靠近，期待增强
3. **发现**: "原来在这！" - 确认时刻，惊喜感
4. **满足**: "我找到了" - 成就感，奖励感

---

## 核心设计原则

### 原则 1: 渐进式信息披露

**不要**: 一次性展示所有信息（如直接显示"钓点"标记）

**要**: 分阶段渐进披露
- **远距离**: 微弱暗示（环境异常）
- **中距离**: 信息增强（细节显现）
- **近距离**: 完全确认（交互提示）

**案例**: 钓鱼游戏钓点发现
```
30米: 看不见
20米: 水面反光、鸟类盘旋（暗示）
10米: 钓点光圈显现（确认）
5米: UI提示"按E钓鱼"（交互）
```

### 原则 2: 环境融入优先于UI标记

**不要**: 使用突兀的游戏UI（如悬浮图标、箭头指引）

**要**: 利用环境中的自然现象
- 光影变化（阳光、反光）
- 生物行为（鸟类、昆虫）
- 声音线索（水流、风声）
- 物理现象（烟雾、粒子）

**好处**:
- 增强沉浸感
- 减少"游戏感"
- 玩家感觉"自己发现的"而非"游戏告诉的"

### 原则 3: 奖励首次发现

**首次发现 ≠ 重复访问**

**首次发现应该**:
- 更强的视觉/听觉反馈（+10-20%）
- 特殊仪式或动画
- 记录到图鉴/日志
- 给予成就/奖励

**重复访问应该**:
- 简化流程，快速进入
- 保留基础识别标记
- 无特殊效果

**原因**: 强化"探索"的价值，鼓励玩家主动发现新内容

---

## 三阶段实现框架

### 阶段 1: 暗示阶段 (Hint Stage)

**目标**: 吸引玩家注意，但不强制

**感受强度**: 低 (15-30%)  
**持续时间**: 2-5 秒  
**触发距离**: 视野范围的 60-80%

**设计要素**:
- **视觉**: 微弱光效、颜色差异、移动元素
- **听觉**: 方向性环境音、微弱提示音
- **交互**: 无强制，玩家可选择忽略

**关键**: 暗示要"可被注意到"但"不明显"

### 阶段 2: 探索阶段 (Exploration Stage)

**目标**: 确认玩家的选择，增强期待

**感受强度**: 渐强 (30% → 70%)  
**持续时间**: 3-8 秒  
**触发距离**: 视野范围的 30-60%

**设计要素**:
- **视觉**: 清晰度提升、颜色饱和度增强、动画显现
- **听觉**: 音量增强、音效分层
- **交互**: 轻微引导（相机、移动速度）

**关键**: 渐变要平滑，避免跳变

### 阶段 3: 确认阶段 (Confirmation Stage)

**目标**: 清晰传达"可以交互"

**感受强度**: 最高 (100%)  
**持续时间**: <1 秒  
**触发距离**: 交互范围内 (通常 <10%)

**设计要素**:
- **视觉**: UI提示、高亮、特效
- **听觉**: 确认音效
- **交互**: 交互键激活、明确提示

**关键**: 瞬间清晰，无歧义

## 参数设计指南

### 距离梯度设置

**公式**:
```
暗示距离 = 视野距离 × 0.6-0.8
探索距离 = 视野距离 × 0.3-0.6
确认距离 = 交互距离 (固定，如 5-10 米)
```

**示例**: 视野 50 米的开放世界
- 暗示: 30-40 米
- 探索: 15-30 米
- 确认: 5 米

### 感受强度曲线

**推荐曲线**: 对数渐进 (平缓开始 → 快速增强 → 趋于稳定)

```
强度 %
100 │                       ╱───
    │                    ╱
 70 │                 ╱
    │              ╱
 30 │       ───╱
    │   ╱──
 15 └───────────────────────────
    暗示    探索初    探索末   确认
```

**避免**: 线性渐变（缺少节奏感）或阶跃（跳变感强）

### 首次强化系数

**视觉**: +10-20% 强度
**听觉**: +15-25% 音量或特殊音效
**交互**: +特殊仪式（2-10 秒）

**调整建议**: 根据发现频率调整
- 高频发现（每分钟多次）: +10%
- 低频发现（每 5-10 分钟）: +20%
- 稀有发现（每小时）: +30-50%

## 边界情况处理

### 情况 1: 玩家快速经过

**现象**: 玩家移动速度快，跳过暗示和探索阶段

**处理**:
- ✅ 允许错过（发现点不稀缺）
- ⚠️ 确认音效可作为"错过提示"
- ❌ 不要强制停止玩家或明显阻挡

### 情况 2: 玩家在附近徘徊

**现象**: 玩家在探索距离内来回移动

**处理**:
- 感受强度基于**当前距离**，而非移动方向
- 音效有冷却时间（避免重复触发）
- 视觉效果持续显示（不闪烁）

### 情况 3: 多个发现点同时可见

**现象**: 玩家视野内有 2+ 个发现点

**处理**:
- 使用**距离优先**原则（近的更明显）
- 音效使用**空间化**（方向性）
- 视觉使用**透明度衰减**（远的更透明）

### 情况 4: 重复访问同一点

**现象**: 玩家离开后再次靠近

**处理**:
- 设置**时间阈值**（如 30 秒）
- 阈值内: 跳过暗示阶段，直接探索/确认
- 阈值外: 重新触发完整流程

---

## 注意事项

### ⚠️ 常见错误

**错误 1: 暗示过弱**
- **症状**: 玩家完全错过发现点
- **原因**: 暗示强度 <10%，距离过远
- **解决**: 提升至 15-20%，缩短暗示距离

**错误 2: 强度跳变**
- **症状**: 玩家感觉"突然出现"
- **原因**: 阶段间没有渐变，或渐变时间 <0.5 秒
- **解决**: 使用 Lerp 插值，渐变时间 1-2 秒

**错误 3: UI 过早出现**
- **症状**: 破坏发现的惊喜感
- **原因**: 确认阶段距离过远（如 20 米）
- **解决**: UI 仅在交互距离内出现（5-10 米）

**错误 4: 首次重复无差异**
- **症状**: 玩家不重视探索
- **原因**: 首次和重复体验一样
- **解决**: 增强首次发现（奖励、仪式、记录）

### ✅ 最佳实践

**实践 1: 使用环境叙事**
- 不要只依赖光效和音效
- 加入生态元素（动物、植物、地形）
- 让发现点"属于世界"而非"游戏标记"

**实践 2: A/B 测试强度**
- 不同玩家对"微弱暗示"的感知不同
- 提供可调节的"发现辅助"选项
- 收集玩家反馈，迭代参数

**实践 3: 设计发现教学**
- 首个发现点应该更明显（强度 +50%）
- 提供教学文本"观察环境寻找线索"
- 逐步降低后续发现点的明显程度

## 案例: Fishing 项目钓点发现

### 设计目标

让玩家通过观察环境（水面、鸟类、声音）自然发现钓点，而非通过 UI 标记。

### 实现方案

**阶段 1: 好奇 (15-30 米)**
- 视觉: 水面微弱反光（5-8 个粒子），鱼影跃出，鸟类盘旋
- 听觉: 方向性水流声 (+10%)，鸟鸣，微弱叮声
- 交互: 无

**阶段 2: 靠近 (5-15 米)**
- 视觉: 钓点光圈显现（透明度 20% → 60%），水纹动画，颜色增强
- 听觉: 水流声增强 (+20%)，探索音乐淡入
- 交互: 相机轻微引导 (3-5°)，移动速度 -10%

**阶段 3: 确认 (<5 米)**
- 视觉: UI 提示"按 E 钓鱼"，光圈 100% 透明度
- 听觉: 清脆确认音 (2000-3000 Hz)
- 交互: E 键激活，准星锁定

**首次发现强化**:
- 光柱特效（向天空）
- 特殊和弦音效
- "NEW!" 标签
- 10 秒仪式流程
- 图鉴记录

### 效果评估

**成功指标**:
- 90%+ 玩家能发现第一个钓点
- 玩家报告"自己发现的"而非"游戏告诉的"
- 首次发现有成就感

## 扩展应用

### 应用 1: 宝箱发现

**暗示**: 微弱金光、宝石反光、神秘音乐
**探索**: 光芒增强、音乐清晰、环境粒子
**确认**: 宝箱完全显现、"打开"提示

### 应用 2: NPC 发现

**暗示**: 远处人影、对话声、篝火烟雾
**探索**: NPC 细节显现、对话内容预览
**确认**: NPC 名称显示、"交谈"提示

### 应用 3: 解谜线索

**暗示**: 墙上微弱划痕、特殊纹理
**探索**: 划痕清晰化、发光高亮
**确认**: 线索文本提示、"调查"选项

## 实现检查清单

### Phase 1: 暗示阶段设计
- [ ] 确定暗示距离（视野的 60-80%）
- [ ] 设计环境暗示元素（非 UI）
- [ ] 设置强度范围（15-30%）
- [ ] 确保可被注意但不明显

### Phase 2: 探索阶段设计
- [ ] 设计渐变曲线（平滑，无跳变）
- [ ] 视觉/听觉协同增强
- [ ] 轻微交互引导（可取消）
- [ ] 测试不同移动速度下的体验

### Phase 3: 确认阶段设计
- [ ] UI 提示清晰明确
- [ ] 音效瞬间可辨
- [ ] 交互范围合理（5-10 米）
- [ ] 无歧义

### Phase 4: 首次强化设计
- [ ] 视觉特效（+10-20%）
- [ ] 特殊音效或仪式
- [ ] 记录系统（图鉴/成就）
- [ ] 奖励机制

### Phase 5: 边界测试
- [ ] 快速经过
- [ ] 徘徊移动
- [ ] 多点同时可见
- [ ] 重复访问
