# 知识资产 (Reports)

这里存放从项目中沉淀的知识资产，包括复盘、洞察和设计经验。

---

## 目录结构

```
reports/
├── retrospectives/   # 项目复盘
└── insights/         # 行业洞察
```

---

## 项目复盘 (Retrospectives)

项目完成后的经验总结：

```markdown
# [项目名称] 复盘

## 项目概述
[项目基本信息]

## 成功经验
[哪些设计方法有效]

## 失败教训
[哪些地方踩坑了]

## 提炼的模式
[可复用的设计模式]

## 改进建议
[下次如何做得更好]
```

---

## 行业洞察 (Insights)

对游戏设计趋势和方法的研究：

```markdown
# [主题] 洞察

## 问题陈述
[要研究的问题]

## 调研发现
[收集的信息和案例]

## 分析结论
[得出的结论]

## 应用建议
[如何应用到设计中]
```

---

## 知识流向

```
项目实践 → 复盘总结 → 提炼模式 → 沉淀到 skills/
         ↓
行业研究 → 洞察报告 → 验证方法 → 应用到项目
```
