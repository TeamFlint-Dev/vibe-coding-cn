# Fishing 项目感受定义标准

> **用途**: 为 Fishing 项目的设计提供感受定义的标准和模板  
> **维护者**: Game Design Studio  
> **适用范围**: Fishing 及类似的放置钓鱼游戏

---

## 核心感受目标

Fishing 项目的设计围绕以下核心感受：

### 1. 期待感 (Anticipation)

**定义**: 玩家对即将发生的事情充满期待和好奇

**适用场景**:
- 钓鱼等待咬钩时
- 升级即将完成时
- 接近发现新鱼种时

**传递方式**:
- 视觉: 进度条、闪烁提示、渐变光效
- 听觉: 音乐渐强、环境音变化
- 交互: 倒计时、浮漂摇晃

**强度等级**:
| 等级 | 持续时间 | 场景 | 实现方式 |
|------|----------|------|----------|
| 低 | 1-3s | 投竿动画 | 简单视觉反馈 |
| 中 | 3-10s | 钓鱼等待 | 浮漂摇晃 + 环境音 |
| 高 | 10s+ | 接近稀有鱼 | 水面异常 + 音乐变化 |

---

### 2. 惊喜感 (Surprise)

**定义**: 超出预期的积极体验

**适用场景**:
- 咬钩触发时
- 钓到稀有鱼时
- 首次发现新鱼种时

**传递方式**:
- 视觉: 突然的光效、水花、粒子爆发
- 听觉: 清脆的提示音、特殊音效
- 交互: 屏幕振动、时间缓慢

**差异化设计**:
| 稀有度 | 光效颜色 | 音效 | 特殊反馈 |
|--------|----------|------|----------|
| Common | 白色 | 标准提示音 | 无 |
| Uncommon | 绿色 | +铃声 | 小光晕 |
| Rare | 蓝色 | +和弦 | 水圈扩散 |
| Epic | 紫色 | +号角 | 屏幕闪光 |

---

### 3. 满足感 (Satisfaction)

**定义**: 完成行为后的成就感和愉悦感

**适用场景**:
- 成功收杆时
- 放置鱼到展示位时
- 完成升级时

**传递方式**:
- 视觉: 完成动画、勾选标记、数字跳动
- 听觉: 悦耳的完成音效
- 交互: 奖励展示、成就解锁

**层次设计**:
| 层次 | 场景 | 反馈强度 | 持续时间 |
|------|------|----------|----------|
| 微满足 | 成功收杆 | 低 | 1s |
| 中满足 | 放置展示 | 中 | 2s |
| 大满足 | 完成升级 | 高 | 3-5s |

---

### 4. 进步感 (Progress)

**定义**: 持续成长和接近目标的感觉

**适用场景**:
- 金币累积时
- 鱼竿升级时
- 图鉴完成度提升时

**传递方式**:
- 视觉: 进度条、数字增长、等级标识
- 听觉: 阶段性音效
- 交互: 里程碑提示

**节奏设计**:
```
早期（0-15min）: 快速进步（每 5 分钟一个里程碑）
中期（15-60min）: 稳定进步（每 15 分钟一个里程碑）
后期（60min+）: 缓慢进步（每 30 分钟一个里程碑）
```

---

### 5. 自豪感 (Pride)

**定义**: 对自己成就的骄傲和展示欲

**适用场景**:
- 展示稀有鱼时
- 图鉴完成时
- 满级达成时

**传递方式**:
- 视觉: 华丽的展示效果、稀有度光效
- 听觉: 庄重的音乐
- 交互: 分享功能（预留）

**展示设计**:
- 展示位视觉华丽（3D 模型 + 光效）
- 稀有鱼有独特展示台
- 可访问他人基地（社交）

## 感受组合模式

### 钓鱼循环感受链

```
平静 → 期待 → 紧张期待 → 惊喜 → 满足 → 平静
(IDLE)  (CASTING) (WAITING)  (HOOKED) (SUCCESS) (IDLE)
```

**设计检查**:
- [ ] 每个状态都有明确的感受目标
- [ ] 感受转换有清晰的视觉/听觉标识
- [ ] 高潮时刻（HOOKED）反馈最强烈
- [ ] 结束后有"呼吸感"（返回平静）

### 展示收益感受链

```
选择 → 期待 → 放置 → 满足 → 持续满足
(选鱼) (确认) (动画) (完成) (收益累积)
```

**设计检查**:
- [ ] 放置有仪式感（不是瞬间完成）
- [ ] 展示位视觉变化明显
- [ ] 收益累积有持续反馈（数字跳动）

### 升级感受链

```
积累 → 期待 → 决策 → 满足 → 进步 → 验证
(攒钱) (够钱) (确认) (升级) (能力提升) (下次钓鱼)
```

**设计检查**:
- [ ] 升级前有明确的"差多少"提示
- [ ] 升级确认有仪式感
- [ ] 升级后立即能感受到效果
- [ ] 下一个目标清晰可见

## 感受验证清单

在设计或评审时，必须检查：

### 钓鱼系统
- [ ] 等待阶段是否传递"期待感"？（浮漂摇晃、音效）
- [ ] 咬钩时刻是否有"惊喜感"？（视觉/听觉爆发）
- [ ] 成功时是否有"满足感"？（完成动画、奖励展示）
- [ ] 稀有鱼的惊喜感是否显著强于普通鱼？

### 展示系统
- [ ] 放置时是否有"自豪感"？（华丽展示）
- [ ] 基地视觉是否随展示变化？（丰富感）
- [ ] 是否有持续的"满足感"？（收益跳动）

### 升级系统
- [ ] 升级前是否有"期待感"？（接近目标提示）
- [ ] 升级时是否有"进步感"？（能力提升明显）
- [ ] 升级后是否立即可验证？（下次钓鱼感受到）

### 收集系统
- [ ] 首次发现是否有"惊喜感"？（NEW 标识、奖励）
- [ ] 图鉴进度是否传递"进步感"？（百分比、里程碑）
- [ ] 完成收集是否有"成就感"？（全发现奖励）

## 反面案例警示

### ❌ 缺乏期待感

**场景**: 投竿后立即咬钩

**问题**: 无等待，无悬念，无期待

**修正**: 加入 3-15s 随机等待，创造"即将到来"的悬念

### ❌ 惊喜感不足

**场景**: 稀有鱼和普通鱼的反馈一样

**问题**: 稀有失去价值，无惊喜

**修正**: 稀有鱼有独特光效、音效、动画

### ❌ 满足感缺失

**场景**: 成功收杆后直接进入背包，无动画

**问题**: 无仪式感，无满足感

**修正**: 加入鱼跃出水面动画、奖励悬浮展示

### ❌ 进步感不明显

**场景**: 升级后看不出能力提升

**问题**: 无成长感，升级无意义

**修正**: 升级后下一次钓鱼立即钓到稀有鱼（正反馈）

## 使用指南

### 设计新功能时

1. **识别核心感受**: 这个功能要传递什么感受？
2. **选择传递方式**: 视觉/听觉/交互如何协同？
3. **设计感受流程**: 感受如何随时间演变？
4. **差异化设计**: 不同情况下感受如何变化？
5. **验证检查**: 使用感受验证清单检查

### 评审现有设计时

1. **感受识别**: 玩家实际感受到了什么？
2. **对比目标**: 是否符合目标感受？
3. **找出断点**: 哪里破坏了感受？
4. **改进方案**: 如何修复？
5. **重新验证**: 修复后是否达成目标？
