## Phase 0: 设计模式审查与策略制定

### 📚 相关设计模式审查

#### 1. 钓鱼循环模式 (Fishing Loop Pattern)

**来源**: `skills/designPatterns/fishing-loop-pattern.md`

**关键洞察**:
- 钓鱼循环的核心感受链: **期待 → 惊喜 → 满足**
- IDLE 状态是整个循环的"探索阶段"
- 探索阶段的目标: 让玩家产生"想要钓鱼"的欲望

**可复用的设计要素**:
- 三阶段感受流程的结构（准备 → 触发 → 结果）
- 渐进式反馈的原则
- 期待感的时间节奏控制（1-3秒的微期待）

**本任务的应用**:
- 将"发现钓点"设计为三阶段: **好奇 → 靠近 → 确认**
- 每个阶段对应不同的感受强度和传递方式
- 整体时间控制在 5-10 秒（从远处看到到靠近确认）

#### 2. 环境联动设计模式 (Environmental Integration Pattern)

**来源**: `skills/designPatterns/environmental-integration-pattern.md`

**关键洞察**:
- 环境不仅是背景，而是玩法的一部分
- 钓点应该是环境中的"有机元素"，而非突兀的UI标记
- 环境暗示可以增强发现的真实感和满足感

**可复用的设计要素**:
- 视觉暗示的层次设计（远 → 中 → 近）
- 环境音效的空间化设计
- 自然元素作为交互提示

**本任务的应用**:
- 远处: 水面反光、鸟类聚集等环境暗示
- 中距离: 水纹、声音等细节强化
- 近距离: 清晰的钓点标识

#### 3. 感受定义标准 (Fishing Feeling Standards)

**来源**: `products/fishing-feeling-standards.md`

**关键洞察**:
- 期待感的强度等级: 低(1-3s) / 中(3-10s) / 高(10s+)
- 惊喜感的差异化设计（稀有度）
- 探索感受的传递方式

**本任务的应用**:
- 发现钓点属于"低期待感"（1-3秒快速识别）
- 但首次发现可以提升到"中期待感"（3-5秒特殊仪式）
- 使用视觉+听觉+交互三位一体设计

### 🔍 相关决策记录审查

#### FD-001: 采用状态机设计钓鱼流程

**决策内容**: IDLE → CASTING → WAITING → HOOKED → SUCCESS/FAILED

**对本任务的影响**:
- IDLE 状态是状态机的起点
- "发现钓点"是从 IDLE 到 CASTING 的触发器
- 必须清晰传达"可以开始钓鱼"的信号

#### FD-005: 环境联动采用响应式设计

**决策内容**: 环境元素动态响应玩家行为和游戏状态

**对本任务的影响**:
- 钓点不是静态的UI元素，而是会"响应"玩家靠近的环境元素
- 设计应该包含距离梯度的响应变化
- 视觉/听觉/交互应该协同变化

### 📊 现有案例分析

#### 塞尔达系列的"发现设计"

**参考价值**:
- "啊哈时刻"的营造: 从远处的好奇到靠近的确认
- 环境暗示的层次: 光柱、粒子、音乐变化
- 首次发现的特殊仪式

**可借鉴的要素**:
- 远处的视觉引导（但不要太明显）
- 靠近时的渐强反馈
- 确认时的满足感

**需要调整的部分**:
- 塞尔达的发现更偏向"探索奖励"，我们的钓点是"功能入口"
- 钓点的发现应该更频繁、更快速
- 不需要过于复杂的仪式（避免打断钓鱼节奏）

## 🎯 设计策略

### 核心设计目标

**目标感受**: 好奇 → 靠近 → 确认

**设计原则**:
1. **快速识别**: 5-10秒内从看到到确认
2. **自然融入**: 钓点是环境的一部分，而非UI标记
3. **渐进反馈**: 距离越近，提示越清晰
4. **差异化**: 首次发现 vs 重复发现的感受差异

### 三阶段感受设计框架

#### 阶段 1: 好奇 (Curiosity)

**触发距离**: 15-30 米  
**持续时间**: 2-3 秒  
**感受强度**: 微弱  
**目标**: 吸引玩家注意，产生"那是什么？"的好奇

**传递方式（待Phase 1详细设计）**:
- 视觉: 远处的暗示（水面反光/鸟类/光点）
- 听觉: 微弱的环境音（水流声/鸟鸣）
- 交互: 无强制交互，玩家可选择忽略或靠近

#### 阶段 2: 靠近 (Approach)

**触发距离**: 5-15 米  
**持续时间**: 3-5 秒  
**感受强度**: 渐强  
**目标**: 确认玩家的选择，增强期待

**传递方式（待Phase 1详细设计）**:
- 视觉: 细节渐清（钓点标识、水纹）
- 听觉: 音效增强（水声清晰化）
- 交互: 视角轻微引导（相机倾向钓点）

#### 阶段 3: 确认 (Confirmation)

**触发距离**: 0-5 米  
**持续时间**: 瞬间（< 1秒）  
**感受强度**: 明确  
**目标**: 清晰传达"可以交互"，提供行动指引

**传递方式（待Phase 1详细设计）**:
- 视觉: UI提示出现（"按E钓鱼"）
- 听觉: 确认音效（清脆提示音）
- 交互: 交互键可用，准星/视角锁定钓点

### 差异化设计策略

#### 首次发现 vs 重复发现

**首次发现**:
- 更强的视觉反馈（特殊光效/粒子）
- 更明显的音效（发现音效）
- 可能包含教学提示
- 图鉴记录提示

**重复发现**:
- 简化的视觉反馈（标准标识）
- 标准音效
- 无教学提示
- 快速进入交互
