## 1. 感受定义 (3 任务)

### IDLE-F-01: 定义"发现钓点"的感受流程
- **优先级**: P0
- **设计产出**: 好奇→靠近→确认 的感受描述
- **参考资料**: 
  - [感受定义标准](../../../fishing-feeling-standards.md)
  - [钓鱼循环模式](../../../../skills/designPatterns/fishing-loop-pattern.md)
- **验收标准**:
  - [ ] 感受流程清晰分为 3 个阶段
  - [ ] 每个阶段有明确的感受强度和传递方式
  - [ ] 考虑了首次/重复发现的差异
  - [ ] Guardian 评审通过

### IDLE-F-02: 定义钓点"吸引力"的感受属性
- **优先级**: P0
- **设计产出**: 视觉/空间/暗示的吸引力强度
- **参考资料**: 
  - [感受定义标准](../../../fishing-feeling-standards.md)
- **验收标准**:
  - [ ] 定义了视觉吸引力的具体设计
  - [ ] 定义了空间吸引力的距离范围
  - [ ] 定义了暗示系统的强度层级
  - [ ] Guardian 评审通过

### IDLE-F-03: 定义多钓点场景下的选择感受
- **优先级**: P1
- **设计产出**: 比较、权衡、决策的感受设计
- **参考资料**: 
  - [感受定义标准](../../../fishing-feeling-standards.md)
- **验收标准**:
  - [ ] 定义了选择时的感受演变
  - [ ] 考虑了不同钓点的差异化
  - [ ] 设计了选择的反馈机制
  - [ ] Guardian 评审通过

## 2. 氛围营造 (4 任务)

### IDLE-A-01: 设计钓点的视觉识别系统
- **优先级**: P0
- **设计产出**: 颜色、形状、光效的设计方案
- **参考资料**: 
  - [设计检查清单](../../../fishing-design-checklist.md)
- **验收标准**:
  - [ ] 定义了钓点的核心视觉元素
  - [ ] 设计了不同状态下的视觉变化
  - [ ] 考虑了可见性和辨识度
  - [ ] Guardian 评审通过

### IDLE-A-02: 设计环境音效的探索氛围
- **优先级**: P1
- **设计产出**: 水流声、鸟鸣、风声的配置
- **参考资料**: 
  - [环境联动设计模式](../../../../skills/designPatterns/environmental-integration-pattern.md)
- **验收标准**:
  - [ ] 设计了完整的环境音效列表
  - [ ] 定义了音效的空间化和动态变化
  - [ ] 考虑了不同天气/时间的音效差异
  - [ ] Guardian 评审通过

### IDLE-A-03: 设计钓点的空间布局原则
- **优先级**: P1
- **设计产出**: 距离、密度、视野的布局规则
- **参考资料**: 
  - [钓鱼循环模式](../../../../skills/designPatterns/fishing-loop-pattern.md)
- **验收标准**:
  - [ ] 定义了钓点的最小/最大距离
  - [ ] 设计了钓点密度的分布规则
  - [ ] 考虑了视野范围和探索节奏
  - [ ] Guardian 评审通过

### IDLE-A-04: 设计时间/天气对氛围的影响
- **优先级**: P2
- **设计产出**: 昼夜/天气变化的氛围差异
- **参考资料**: 
  - [环境联动设计模式](../../../../skills/designPatterns/environmental-integration-pattern.md)
  - [决策记录 FD-006](../../../../DECISION-LOG.md#fd-006)
- **验收标准**:
  - [ ] 设计了3种天气下的氛围差异
  - [ ] 定义了昼夜变化的视觉/听觉效果
  - [ ] 确保环境变化不影响玩法平衡
  - [ ] Guardian 评审通过
