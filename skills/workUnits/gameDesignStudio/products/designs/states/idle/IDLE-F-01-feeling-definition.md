# IDLE-F-01: "发现钓点"感受定义

> **任务ID**: IDLE-F-01  
> **任务名称**: 定义"发现钓点"的感受流程  
> **优先级**: P0  
> **当前阶段**: Phase 1 - 感受定义  
> **创建时间**: 2026-01-13

---

## 核心感受目标

### 目标感受链

```
好奇 (Curiosity) → 靠近 (Approach) → 确认 (Confirmation)
```

**设计理念**:  
"发现钓点"不是一个瞬间的事件，而是一个渐进的体验过程。玩家从远处捕捉到模糊的暗示，逐渐靠近，直到清晰确认——这个过程本身就是探索的乐趣。

---

## 阶段 1: 好奇 (Curiosity)

### 感受描述

**玩家此刻的心理状态**:
- "咦，那边有什么？"
- "那个光点/水纹看起来不太一样"
- "或许值得过去看看"

**感受属性**:
| 属性 | 值 | 说明 |
|------|-----|------|
| **感受强度** | 微弱 (20%) | 不强制吸引注意力，而是"勾起好奇" |
| **持续时间** | 2-3 秒 | 从看到到决定靠近 |
| **情绪色彩** | 轻松、好奇 | 没有压力，纯粹的探索欲 |
| **注意力需求** | 低 | 玩家可以选择忽略 |

### 感受传递方式

**视觉暗示**（权重 60%）:
- 远处水面的微弱反光（像钻石的闪光）
- 偶尔跃出水面的鱼影（快速、不清晰）
- 聚集的鸟类（在钓点上空盘旋）
- 水面的细微涟漪（与普通水面略有不同）

**听觉暗示**（权重 30%）:
- 方向性的水流声（略微突出于环境音）
- 鸟鸣声（从钓点方向传来）
- 微弱的"叮"声（像风铃，极其轻微）

**交互暗示**（权重 10%）:
- 无强制交互
- 玩家可以选择忽略或靠近
- 不出现任何UI提示

### Socratic Questioning

**Q1: 为什么不直接显示UI提示？**  
A: 因为我们要营造"真实发现"的感受，而非"游戏任务标记"。UI提示会破坏探索的真实感和满足感。玩家应该感觉是"自己发现了钓点"，而非"游戏告诉我这里有钓点"。

**Q2: 20%的强度会不会太弱，导致玩家错过？**  
A: 这是有意为之。钓点不应该是稀缺资源，错过一个并不重要。我们要的是"发现的惊喜"，而非"寻找的焦虑"。如果玩家真的在寻找钓点，他们会主动留意这些暗示。

**Q3: 2-3秒的持续时间如何保证？**  
A: 这取决于玩家的移动速度和钓点的分布密度。我们需要确保钓点的间距让玩家在这个时间窗口内可以"看到→思考→决定"。

**Q4: 首次发现和重复发现有什么区别？**  
A: 首次发现应该更明显一些（强度30%），因为玩家还在学习"什么是钓点的标志"。重复发现时，玩家已经知道要找什么，所以可以更微妙（强度15-20%）。

## 阶段 2: 靠近 (Approach)

### 感受描述

**玩家此刻的心理状态**:
- "没错，这确实是个钓点！"
- "越来越清楚了，期待感上升"
- "我做了正确的选择"

**感受属性**:
| 属性 | 值 | 说明 |
|------|-----|------|
| **感受强度** | 渐强 (20% → 60%) | 随距离递减而递增 |
| **持续时间** | 3-5 秒 | 从15米靠近到5米 |
| **情绪色彩** | 期待、确信 | 从"可能是"到"肯定是" |
| **注意力需求** | 中 | 玩家主动关注钓点 |

### 感受强度曲线

```
强度 %
60  │                    ╱───
    │                 ╱
40  │              ╱
    │           ╱
20  │     ╱────
    │  ╱
0   └─────────────────────
    15m  10m  8m   5m  3m (距离)
```

**曲线设计理由**:
- **15-10米**: 缓慢上升，确认玩家的选择
- **10-5米**: 加速上升，期待感增强
- **5米内**: 趋于稳定，准备进入确认阶段

### 感受传递方式

**视觉强化**（权重 60%）:
- 钓点标识逐渐清晰（从模糊到清晰）
- 水纹变得更明显（涟漪扩大）
- 颜色饱和度提升（从灰蓝到亮蓝）
- 细节逐渐显现（鱼影、水草）

**听觉强化**（权重 30%）:
- 水流声音量增加
- 音效的频率变化（从低沉到清脆）
- 环境音乐淡入（如果有探索主题音乐）

**交互反馈**（权重 10%）:
- 相机轻微倾向钓点（不强制，只是暗示）
- 移动速度可能略微放慢（沉浸感）
- 准星/瞄准点会"吸附"钓点（微弱）

### Socratic Questioning

**Q1: 为什么需要渐进式而非一步到位？**  
A: 因为渐进式能创造"期待的积累"。瞬间的清晰会失去这个积累过程，让体验变得扁平。我们要的是"越来越期待"，而非"突然知道"。

**Q2: 3-5秒会不会太长？**  
A: 不会。玩家在移动中，这3-5秒是自然的"走过去"的时间。如果缩短，会让玩家感觉"跳跃"，破坏真实感。如果延长，会让玩家不耐烦。

**Q3: 相机倾向会不会破坏玩家的控制感？**  
A: 只要是"轻微"的倾向（比如5-10度的偏移），并且不锁定相机，就不会破坏控制感。这更像是"暗示"而非"强制"。玩家依然可以完全控制视角。

**Q4: 首次和重复有何区别？**  
A: 首次发现时，可以有更明显的视觉效果（比如粒子特效、光柱）。重复发现时，这些效果应该减弱或去除，只保留基础的标识，避免视觉疲劳。

## 阶段 3: 确认 (Confirmation)

### 感受描述

**玩家此刻的心理状态**:
- "就是这里了！"
- "我可以开始钓鱼了"
- "满足感和行动冲动"

**感受属性**:
| 属性 | 值 | 说明 |
|------|-----|------|
| **感受强度** | 明确 (100%) | 清晰、无歧义 |
| **持续时间** | 瞬间 (< 1秒) | 进入交互范围即触发 |
| **情绪色彩** | 满足、准备就绪 | "我找到了"+"我准备好了" |
| **注意力需求** | 高 | 玩家专注于交互提示 |

### 感受传递方式

**视觉确认**（权重 50%）:
- UI提示出现："按 E 开始钓鱼"
- 钓点标识完全清晰（图标/光圈）
- 颜色达到最高饱和度（鲜明的蓝色或其他主题色）
- 可能有微弱的脉冲动画（呼吸感）

**听觉确认**（权重 30%）:
- 确认音效（清脆的"叮"声）
- 音效应该是悦耳的，传达"正确"的感觉
- 音量适中，不突兀

**交互确认**（权重 20%）:
- 交互键变为可用状态
- 准星/准心锁定钓点（如果有）
- 角色可能有微小的动作（比如停下脚步）

### Socratic Questioning

**Q1: 为什么要设计成"瞬间"？**  
A: 因为玩家已经走过了"好奇→靠近"的过程，期待已经积累完成。这时候需要的是"立即满足"，而非继续等待。延迟会让玩家感觉"为什么还不能交互？"

**Q2: UI提示是否违背了"真实感"？**  
A: 不违背。前两个阶段营造的是"发现的真实感"，第三阶段是"交互的清晰度"。这时候玩家已经沉浸在发现中，UI提示是必要的"功能说明"，不会破坏体验。

**Q3: 音效会不会太频繁？（如果钓点很多）**  
A: 这是个好问题。如果玩家短时间内经过多个钓点，确认音效可能会变得烦人。解决方案：
- 音效有CD时间（5秒内同一钓点只响一次）
- 音效音量随频率递减
- 玩家可以在设置中关闭

**Q4: 首次和重复的区别？**  
A: 首次发现时，可以有额外的视觉效果（比如"NEW!"标签、图鉴提示）。这强化了"发现新事物"的满足感。重复发现时，只保留基础的交互提示。

## 差异化设计

### 首次发现 vs 重复发现

#### 首次发现（强化版）

**阶段 1 - 好奇**:
- 强度: 30%（比重复版高10%）
- 额外视觉: 微弱的光柱（向天空延伸）
- 额外音效: 神秘的音乐和弦

**阶段 2 - 靠近**:
- 强度曲线: 30% → 80%（比重复版整体高20%）
- 额外视觉: 粒子特效（环绕钓点）
- 额外音效: "发现"主题音乐淡入

**阶段 3 - 确认**:
- UI提示: 增加"首次发现！"文字
- 额外视觉: 光芒闪烁效果
- 额外音效: 特殊的发现音效（更悦耳）
- 额外功能: 图鉴记录提示

**设计理由**: 首次发现是"学习时刻"，需要更强的反馈帮助玩家建立"这就是钓点"的认知。同时，首次发现本身就是一种成就，值得庆祝。

#### 重复发现（标准版）

**阶段 1 - 好奇**:
- 强度: 20%
- 视觉: 水面反光、鱼影、鸟类
- 音效: 环境音

**阶段 2 - 靠近**:
- 强度曲线: 20% → 60%
- 视觉: 标识渐清、颜色渐强
- 音效: 水流声增强

**阶段 3 - 确认**:
- UI提示: "按 E 开始钓鱼"
- 视觉: 标准图标
- 音效: 标准确认音

**设计理由**: 玩家已经熟悉钓点的样子，不需要过多的强化。简洁、快速，让玩家可以专注于钓鱼本身。

## 边界与极端情况

### 边界情况 1: 玩家快速跑过钓点

**场景**: 玩家没有注意到暗示，快速移动经过钓点

**感受影响**:
- 阶段 1 可能被跳过（没有看到暗示）
- 阶段 2 时间压缩（3-5秒 → 1-2秒）
- 阶段 3 可能出现"错过"的遗憾感

**设计应对**:
- 不阻止玩家错过（错过也是正常的探索体验）
- 确认音效可以作为"提醒"（玩家经过时听到）
- 钓点不稀缺，错过一个不影响游戏

### 边界情况 2: 玩家在钓点附近徘徊

**场景**: 玩家在5-15米范围内来回移动

**感受影响**:
- 阶段 2 的渐强效果可能反复触发
- 可能产生"不确定"的感觉

**设计应对**:
- 感受强度基于当前距离，而非移动方向
- 靠近时增强，远离时减弱，形成自然的反馈
- 确认音效只在首次进入范围时触发（避免重复）

### 边界情况 3: 多个钓点同时在视野内

**场景**: 玩家能同时看到2个或更多钓点

**感受影响**:
- 好奇感可能分散（不知道看哪个）
- 靠近感受可能混淆（哪个在变强？）

**设计应对**:
- 视觉暗示使用距离衰减（近的更明显）
- 音效使用方向性和音量差异
- 玩家可以自由选择目标（选择本身是乐趣）

### 边界情况 4: 玩家远离后再靠近同一钓点

**场景**: 玩家走开后又回来

**感受影响**:
- 是否应该重新触发完整流程？
- 还是直接进入确认阶段？

**设计应对**:
- 如果间隔 < 30秒: 跳过阶段1，直接从阶段2开始
- 如果间隔 > 30秒: 完整流程（玩家可能"忘记"了）
- 这个时间阈值需要在测试中调整

## 验收标准

### 感受层面

- [ ] 玩家能够清晰感受到三个阶段的区别
- [ ] 每个阶段的感受强度符合设计预期
- [ ] 首次和重复发现的感受差异明显
- [ ] 整体体验流畅，没有断裂感

### 传递方式

- [ ] 视觉/听觉/交互三位一体协同工作
- [ ] 每个传递方式都清晰传达了目标感受
- [ ] 没有冗余或相互干扰的反馈

### 边界情况

- [ ] 快速经过、徘徊、多钓点等极端情况都有合理处理
- [ ] 玩家在任何情况下都不会感到困惑或挫败

- [ ] 通过 Guardian 的感受达成度评审
- [ ] 通过 Guardian 的逻辑自洽性评审
