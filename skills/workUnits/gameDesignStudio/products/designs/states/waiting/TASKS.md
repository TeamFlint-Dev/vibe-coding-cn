## 1. 感受定义 (4 任务)

### WAIT-F-01: 定义等待时间的"悬念曲线"
- **优先级**: P0
- **设计产出**: 期待→紧张→焦虑的感受演变

### WAIT-F-02: 定义"假咬钩"的惊吓感
- **优先级**: P1
- **设计产出**: 误导性视觉/听觉的感受设计

### WAIT-F-03: 定义等待超时的"失望感"
- **优先级**: P1
- **设计产出**: 从期待到放弃的感受过渡

### WAIT-F-04: 定义不同等待时长的感受差异
- **优先级**: P1
- **设计产出**: 短(3s)/中(8s)/长(15s)的感受对比

## 2. 氛围营造 (5 任务)

### WAIT-A-01: 设计鱼漂的微动画系统
- **优先级**: P0
- **设计产出**: 浮动、摇晃、下沉的动画变体

### WAIT-A-02: 设计等待时的环境音效变化
- **优先级**: P1
- **设计产出**: 音乐淡入、水声、呼吸感

### WAIT-A-03: 设计水面的涟漪和反光
- **优先级**: P2
- **设计产出**: 实时水面效果的细节设计

### WAIT-A-04: 设计等待时的UI最小化
- **优先级**: P1
- **设计产出**: 清理屏幕、聚焦钓点的设计

### WAIT-A-05: 设计环境中的"假信号"
- **优先级**: P2
- **设计产出**: 鱼影、水花、鸟鸣的干扰元素

## 3. 仪式设计 (4 任务)

### WAIT-R-01: 设计取消钓鱼的仪式
- **优先级**: P1
- **设计产出**: 按键提示、确认、收杆的流程

### WAIT-R-02: 设计等待时的"小动作"系统
- **优先级**: P2
- **设计产出**: 角色的无聊动作、环顾等

### WAIT-R-03: 设计等待进度的暗示系统
- **优先级**: P1
- **设计产出**: 不明确的时间提示（避免焦虑）

### WAIT-R-04: 设计连续钓鱼时的等待优化
- **优先级**: P2
- **设计产出**: 第N次钓鱼的等待感受调整

## 4. 故事包裹 (3 任务)

### WAIT-S-01: 设计等待时的角色对话/独白
- **优先级**: P2
- **设计产出**: 随机对话、思考的叙事内容

### WAIT-S-02: 设计环境事件的随机触发
- **优先级**: P2
- **设计产出**: 鸟落水、鱼跃出、NPC经过

### WAIT-S-03: 设计天气对等待体验的影响
- **优先级**: P2
- **设计产出**: 雨天、起风对等待的叙事强化

## 5. 无瑕打磨 (2 任务)

### WAIT-P-01: 检查等待超时的边界处理
- **优先级**: P0
- **设计产出**: 15秒+边界的强制触发逻辑

### WAIT-P-02: 检查等待时的物理/碰撞
- **优先级**: P1
- **设计产出**: 鱼漂位置、鱼线拉伸的合理性
