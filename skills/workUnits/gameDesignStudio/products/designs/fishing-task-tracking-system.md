## 📊 任务状态定义

### 任务生命周期

```
⬜ 未开始 → 🔵 规划中 → 🟡 进行中 → 🟢 已完成 → ✅ 已验收
              ↓            ↓
           ⏸️ 暂停      ⚠️ 有问题
                          ↓
                       🔄 返工中
```

### 状态说明

| 状态图标 | 状态名称 | 说明 | 下一步 |
|---------|---------|------|--------|
| ⬜ | 未开始 | 任务尚未开始 | 检查前置依赖，开始规划 |
| 🔵 | 规划中 | 正在理解需求、收集资料 | 完成规划后进入执行 |
| 🟡 | 进行中 | 正在设计或创作 | 完成设计产出 |
| 🟢 | 已完成 | 设计产出已提交 | 等待验收评审 |
| ⏸️ | 暂停 | 因依赖或资源暂停 | 解决阻塞后继续 |
| ⚠️ | 有问题 | 评审发现问题 | 根据反馈返工 |
| 🔄 | 返工中 | 正在修复问题 | 修复后重新提交 |
| ✅ | 已验收 | 通过质量验收 | 任务关闭 |

## 🗂️ 任务组织方式

### 方式 1: 按状态组织 (推荐用于顺序执行)

**适用场景**: 集中完成某个状态的所有设计

```
本周目标: 完成 IDLE 状态的所有设计 (15 任务)

第 1 天: IDLE 感受定义 (3 任务)
├─ IDLE-F-01 🟡 进行中 - 定义"发现钓点"的感受流程
├─ IDLE-F-02 ⬜ 未开始
└─ IDLE-F-03 ⬜ 未开始

第 2 天: IDLE 氛围营造 (4 任务)
├─ IDLE-A-01 ⬜ 未开始
├─ IDLE-A-02 ⬜ 未开始
├─ IDLE-A-03 ⬜ 未开始
└─ IDLE-A-04 ⬜ 未开始

第 3 天: IDLE 仪式+故事 (6 任务)
...
```

### 方式 2: 按维度组织 (推荐用于并行协作)

**适用场景**: 多人协作，每人负责一个维度

```
Visionary: 负责所有 F (感受定义) 任务
├─ IDLE-F-01/02/03
├─ CAST-F-01/02
├─ WAIT-F-01/02/03/04
└─ ...

美术设计师: 负责所有 A (氛围营造) 任务
├─ IDLE-A-01/02/03/04
├─ CAST-A-01/02/03
└─ ...

交互设计师: 负责所有 R (仪式设计) 任务
...
```

### 方式 3: 按优先级组织 (推荐用于MVP)

**适用场景**: 快速验证核心体验

```
Sprint 1: 所有 P0 任务 (42 个) - 核心循环可玩
Sprint 2: 所有 P1 任务 (58 个) - 完整体验
Sprint 3: 所有 P2 任务 (22 个) - 细节打磨
```

````markdown
## IDLE-F-01: 定义"发现钓点"的感受流程

**状态**: ✅ 已验收  
**优先级**: P0  
**预估工时**: 4 小时  
**实际工时**: 5 小时  
**负责人**: Visionary  
**开始日期**: 2026-01-13  
**完成日期**: 2026-01-13

### 任务描述
设计玩家从看到钓点到决定钓鱼的完整感受流程，确保这个过程有探索的乐趣和发现的惊喜。

### 设计产出 (Deliverables)
- [x] 感受描述文档
- [x] 感受强度曲线图
- [x] 差异化设计方案（首次发现 vs 重复发现）

### 参考资料
- [钓鱼循环模式](../../skills/designPatterns/fishing-loop-pattern.md) - 探索阶段设计
- [塞尔达的发现设计](案例分析) - "啊哈时刻"的营造

### 设计方案摘要
**三阶段感受流程**:
1. **好奇 (Curiosity)** - 远处看到钓点的暗示
   - 强度: 微弱
   - 持续: 1-3秒
   - 传递: 视觉暗示（反光、水纹）
   
2. **靠近 (Approach)** - 走近钓点的期待
   - 强度: 渐强
   - 持续: 2-5秒
   - 传递: 视觉强化（清晰度提升）+ 音效（水声）
   
3. **确认 (Confirmation)** - 确认可以钓鱼的满足
   - 强度: 明确
   - 持续: 瞬间
   - 传递: UI提示出现 + 交互反馈

### 评审反馈
**Guardian 评审 (2026-01-13)**:
- ✅ P0 问题: 无
- ✅ P1 问题: 无
- 💡 建议: 考虑添加首次发现的特殊仪式 (已采纳，见 IDLE-R-03)

### 验收标准
- [x] 三阶段流程清晰
- [x] 每阶段有明确的感受强度和传递方式
- [x] 考虑了首次/重复的差异
- [x] Guardian 评审通过

### 知识沉淀
- 记录到设计手记: `journals/visionary/20260113-idle-discovery-design.md`
- 提炼设计模式: "发现类设计的三阶段流程"
````

## ✅ 产出验收标准

### 验收流程

```
设计完成 → 自查 → 提交评审 → Guardian评审 → 修复问题 → 再次评审 → 验收通过
    │                                    │
    └────── 如果自查不通过 ──────────────┘
```

### 自查清单

```markdown
## 任务自查清单 - [任务ID]

- [ ] 所有设计产出已完成
- [ ] 产出文档格式规范
- [ ] 文档已保存到正确路径
- [ ] 引用的资源链接有效

### 质量检查
- [ ] 符合感受驱动设计哲学
- [ ] 与现有设计一致（无矛盾）
- [ ] 引用了相关设计模式
- [ ] 考虑了边界情况

### 维度特定检查
**如果是 F (感受定义) 任务**:
- [ ] 明确定义了目标感受
- [ ] 描述了感受的强度和持续时间
- [ ] 说明了感受的传递方式
- [ ] 考虑了差异化设计

- [ ] 视觉/听觉/交互三位一体

- [ ] 增强了沉浸感

**如果是 P (无瑕打磨) 任务**:
- [ ] 识别了所有出戏点

- [ ] 记录了设计笔记和决策
```

### Guardian 评审标准

**Guardian 会从以下维度评审**:

1. **感受达成度** - 设计是否真的能传递目标感受？
2. **逻辑自洽性** - 设计在世界观中是否合理？
3. **流畅度** - 是否有出戏的断裂点？
4. **完整性** - 是否遗漏了重要细节？
5. **一致性** - 与其他设计是否一致？

**评审结果分级**:
- ✅ **通过** - 无问题或仅有 P2 建议
- ⚠️ **有条件通过** - 有 P1 问题需要修复
- ❌ **不通过** - 有 P0 问题必须返工

## 🤝 协作工具

### 任务分配建议

**单人设计场景** (推荐流程):

```
Week 1-2: IDLE + CASTING 状态
├─ Day 1-2: IDLE 感受定义 (F)
├─ Day 3-4: IDLE 氛围营造 (A)
├─ Day 5-6: IDLE 仪式+故事 (R+S)
├─ Day 7: IDLE 无瑕打磨 (P)
└─ Day 8-10: CASTING 状态 (F+A+R+S+P)

Week 3-4: WAITING + HOOKED 状态
Week 5-6: SUCCESS + FAILED + MISSED 状态
Week 7: 转换设计 + 全局系统
Week 8: 细节打磨 + 知识沉淀
```

**多人协作场景** (角色分工):

```
Visionary (感受+叙事专家)
├─ 所有 F 维度任务 (23个)
├─ 所有 S 维度任务 (18个)
└─ 评审其他维度的感受达成度

美术设计师 (视觉专家)
├─ A 维度的视觉设计任务 (~15个)
├─ UI 设计任务
└─ 特效和动画设计

音效设计师 (听觉专家)
├─ A 维度的音效设计任务 (~10个)
└─ 音乐和环境音设计

交互设计师 (体验专家)
├─ 所有 R 维度任务 (25个)
├─ A 维度的交互设计任务 (~7个)
└─ 教学和引导设计

系统设计师 (逻辑专家)
├─ 所有 P 维度任务 (13个)
├─ 全局系统设计 (11个)
└─ 数值平衡设计

Guardian (质量顾问)
├─ 所有任务的评审
└─ 瑕疵识别和修复建议
```

## 📋 快速参考

### 任务优先级判断

```
P0 (必须做): 影响核心体验的设计
├─ 核心循环的感受定义
├─ 关键时刻的氛围营造
├─ 基础交互仪式
└─ 必要的瑕疵修复

P1 (应该做): 完善体验的设计
├─ 深度氛围营造
├─ 完整仪式设计
├─ 叙事整合
└─ 重要的瑕疵修复

P2 (可选做): 锦上添花的设计
├─ 高级特性
├─ 细节打磨
├─ 扩展设计
└─ 可选的优化
```

### 常见问题速查

**Q: 任务太多不知道从哪开始？**
A: 按状态顺序 (IDLE→CASTING→...) 按维度顺序 (F→A→R→S→P) 执行

**Q: 一个任务应该花多长时间？**
A: F维度 2-4h，A维度 3-6h，R维度 2-4h，S维度 1-3h，P维度 1-2h

**Q: 如何知道任务是否完成？**
A: 检查验收标准清单，通过 Guardian 评审

**Q: 遇到阻塞怎么办？**
A: 标记为 ⏸️ 暂停，记录阻塞原因，开始其他任务

**Q: 评审不通过怎么办？**
A: 标记为 ⚠️ 有问题，根据反馈修复后重新提交
