# Fishing 项目设计检查清单

> **用途**: 设计或评审 Fishing 项目功能时的完整检查清单  
> **使用时机**: 设计新功能、评审现有设计、验收实现  
> **维护者**: Game Design Studio

---

## 🎯 核心循环检查

### 钓鱼系统

#### 状态机完整性
- [ ] IDLE 状态：玩家可自由移动
- [ ] IDLE 状态：钓点有清晰的交互提示
- [ ] CASTING 状态：投竿动画流畅（0.5s）
- [ ] CASTING 状态：有投竿音效和视觉反馈
- [ ] WAITING 状态：浮漂在水面摇晃
- [ ] WAITING 状态：等待时间在 3-15s 范围
- [ ] WAITING 状态：允许玩家取消
- [ ] HOOKED 状态：咬钩有强烈视觉提示（浮漂下沉）
- [ ] HOOKED 状态：咬钩有清晰听觉提示（咔嗒声）
- [ ] HOOKED 状态：屏幕提示"收杆！"
- [ ] HOOKED 状态：窗口时间为 2s
- [ ] SUCCESS 状态：鱼跃出水面动画
- [ ] SUCCESS 状态：稀有度差异化光效
- [ ] SUCCESS 状态：展示鱼名和稀有度
- [ ] FAILED 状态：有失败提示（轻松语气）
- [ ] 状态转换流畅无卡顿

#### 感受传递
- [ ] 期待感：等待阶段有悬念感
- [ ] 惊喜感：咬钩时刻令人兴奋
- [ ] 满足感：成功时有成就感
- [ ] 节奏感：紧张-放松交替明显
- [ ] 呼吸感：循环结束后有放松时刻

#### 稀有度差异
- [ ] Common：基础反馈（白色光效）
- [ ] Uncommon：增强反馈（绿色 + 铃声）
- [ ] Rare：明显反馈（蓝色 + 和弦 + 光晕）
- [ ] Epic：极强反馈（紫色 + 号角 + 粒子）
- [ ] 首次发现有特殊标识（"NEW!"）

---

### 展示系统

#### 展示位管理
- [ ] 展示位数量为 3 个（MVP）
- [ ] 展示位有固定位置和朝向
- [ ] 展示位可放置/替换鱼
- [ ] 放置时有确认流程（避免误操作）
- [ ] 替换时有提示（当前鱼会移除）
- [ ] 展示位视觉清晰可辨认

#### 展示效果
- [ ] 鱼有 3D 模型展示
- [ ] 鱼有稀有度光效
- [ ] 鱼有铭牌显示名称
- [ ] 展示位满时基地视觉丰富
- [ ] 不同稀有度的展示效果有差异

#### 感受传递
- [ ] 自豪感：展示华丽有仪式感
- [ ] 满足感：放置完成有成就感
- [ ] 炫耀感：展示可被他人看到（预留）

---

### 收益系统

#### 收益计算
- [ ] 收益公式正确：Σ(baseIncome × rarityMultiplier)
- [ ] 稀有度倍率：1x, 2x, 5x, 15x
- [ ] 收益每秒累积
- [ ] 累积有上限（避免溢出）
- [ ] 离线收益有上限（8-12 小时）

#### 收益反馈
- [ ] 金币数字每秒跳动
- [ ] 累积动画流畅
- [ ] 展示位悬停显示收益速率
- [ ] 领取离线收益有仪式感

#### 感受传递
- [ ] 持续满足感：金币持续增长
- [ ] 被欢迎感：离线收益欢迎回归
- [ ] 富足感：金币充裕无紧缺

---

### 升级系统

#### 升级逻辑
- [ ] 鱼竿升级共 10 级
- [ ] 升级费用按 1.5 倍递增
- [ ] 升级费用公式：100 × 1.5^(n-1)
- [ ] 每级 +5% 稀有掉落率
- [ ] 满级总费用为 7,486 金币

#### 升级流程
- [ ] 升级界面清晰显示当前等级
- [ ] 显示下一级费用和效果
- [ ] 金币不足时置灰升级按钮
- [ ] 升级确认有提示（避免误操作）
- [ ] 升级完成有动画和音效

#### 升级反馈
- [ ] 升级时有视觉变化（鱼竿外观/特效）
- [ ] 升级后属性对比清晰
- [ ] 下一次钓鱼能立即感受到效果

#### 感受传递
- [ ] 期待感：接近升级时有提示
- [ ] 进步感：升级后能力明显提升
- [ ] 验证感：下次钓鱼验证效果

---

### 收集系统

#### 图鉴管理
- [ ] 图鉴显示所有 5 种鱼
- [ ] 未发现的鱼显示为剪影
- [ ] 已发现的鱼显示完整信息
- [ ] 图鉴进度百分比清晰
- [ ] 首次发现有奖励提示

#### 图鉴信息
- [ ] 鱼名称
- [ ] 稀有度
- [ ] 基础收益
- [ ] 发现时间
- [ ] 捕获次数
- [ ] 简短描述（可选）

#### 感受传递
- [ ] 惊喜感：首次发现有特殊反馈
- [ ] 进步感：图鉴完成度提升
- [ ] 成就感：完成收集有大奖励

---

## 📊 数值平衡检查

### 产消比验证
- [ ] 30 分钟产消比 > 3:1
- [ ] 2 小时产消比 > 3:1
- [ ] 玩家感到富足而非紧缺

### 升级节奏
- [ ] 升级间隔在 30-120 秒范围
- [ ] 早期升级快速（5 分钟内首次）
- [ ] 后期升级放缓（延长内容）
- [ ] 满级时间在 1-3 小时

### 稀有度概率
- [ ] Common 约 70%
- [ ] Uncommon 约 19%
- [ ] Rare 约 8%
- [ ] Epic 约 4%
- [ ] 升级提升稀有概率明显

### 收益成长曲线
- [ ] 早期收益快速增长（鼓励继续）
- [ ] 中期收益稳定增长（稳定体验）
- [ ] 后期收益缓慢增长（延长内容）

---

## 🎨 视觉设计检查

### 氛围营造
- [ ] 钓点环境宁静放松
- [ ] 水面反射和波纹自然
- [ ] 光影效果柔和
- [ ] 色调统一和谐

### UI/UX
- [ ] UI 风格与游戏主题一致
- [ ] 重要信息易于发现
- [ ] 交互按钮大小合适
- [ ] 字体清晰可读
- [ ] 色彩对比度足够

### 视觉反馈
- [ ] 所有操作有即时反馈
- [ ] 状态转换有视觉标识
- [ ] 稀有度有差异化视觉
- [ ] 进度有可视化展示

---

## 🔊 音效设计检查

### 音效完整性
- [ ] 投竿音效
- [ ] 落水音效
- [ ] 咬钩提示音
- [ ] 成功音效
- [ ] 失败音效
- [ ] 放置展示音效
- [ ] 升级完成音效
- [ ] 金币累积音效

### 音效差异化
- [ ] 稀有度音效有变体
- [ ] 首次发现音效特殊
- [ ] 里程碑音效庄重

### 音乐氛围
- [ ] 背景音乐宁静舒缓
- [ ] 重要时刻音乐变化
- [ ] 音乐循环自然

---

## 🎮 交互设计检查

### 操作流畅性
- [ ] 操作响应时间 < 100ms
- [ ] 无卡顿或延迟
- [ ] 多次点击不会bug
- [ ] 边界情况处理合理

### 反馈及时性
- [ ] 点击有即时反馈
- [ ] 加载有进度提示
- [ ] 等待有占位内容
- [ ] 错误有明确提示

### 可用性
- [ ] 教程简洁清晰
- [ ] 首次体验流畅
- [ ] 无需频繁查看帮助
- [ ] 核心操作易于发现

---

## ⏱️ 节奏设计检查

### 单次循环节奏
- [ ] 钓鱼循环 10-30 秒
- [ ] 投竿 0.5s
- [ ] 等待 3-15s
- [ ] 咬钩窗口 2s
- [ ] 结算 2s

### 游玩节奏
- [ ] 单次游玩 15-30 分钟
- [ ] 每分钟约 5 次钓鱼
- [ ] 每 30-120 秒完成升级
- [ ] 每 5-15 分钟达成里程碑

### 长期节奏
- [ ] 1-3 小时完成 MVP 内容
- [ ] 离线收益促进回归
- [ ] 内容消耗不过快

---

## 🚫 出戏检查（避免破坏沉浸感）

### 逻辑自洽
- [ ] 钓鱼机制在世界观中合理
- [ ] 规则易于理解
- [ ] 无明显逻辑矛盾

### 反馈完整
- [ ] 所有操作都有反馈
- [ ] 无"没反应"的情况
- [ ] 状态转换清晰

### 氛围连贯
- [ ] 无破坏氛围的元素
- [ ] 视觉/听觉/交互协同
- [ ] 风格统一

### 边界处理
- [ ] 异常情况有合理提示
- [ ] 网络断开有友好提示
- [ ] 数据加载有占位

## 📱 技术实现检查

### 性能优化
- [ ] 帧率稳定 ≥ 60 FPS
- [ ] 内存占用合理
- [ ] 加载时间 < 5 秒

### 兼容性
- [ ] 不同平台测试通过
- [ ] 不同分辨率适配
- [ ] 不同输入设备支持

### 数据持久化
- [ ] 玩家数据正确保存
- [ ] 离线收益正确计算
- [ ] 进度不会丢失

### 功能测试
- [ ] 核心流程完整可用
- [ ] 边界情况正常处理
- [ ] 无阻塞性 bug

### 体验测试
- [ ] 感受符合设计目标
- [ ] 节奏流畅无卡顿
- [ ] 无明显出戏点

### 数值测试
- [ ] 产消比符合预期
- [ ] 升级节奏合理

## ✅ 发布前最终检查

### 必须项（P0）
- [ ] 核心循环完整可玩
- [ ] 无阻塞性 bug
- [ ] 感受目标基本达成
- [ ] 数值平衡合理

### 重要项（P1）
- [ ] 所有音效就位
- [ ] 视觉反馈完整
- [ ] UI/UX 友好

### 可选项（P2）
- [ ] 进阶教程
- [ ] 成就系统

## 使用指南

### 设计阶段
- 逐项检查设计方案是否覆盖所有要点
- 标记不确定的项，需要测试验证
- 记录设计决策理由

### 实现阶段
- 开发完成后逐项验证
- 发现问题立即记录
- 优先修复 P0 和 P1 问题

### 评审阶段
- 完整走一遍检查清单
- 记录所有发现的问题
- 分类问题优先级
