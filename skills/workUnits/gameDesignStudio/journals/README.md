# 手记 (Journals)

创始人的设计思考和经验记录。

---

## 目录结构

```
journals/
├── visionary/    # Visionary（愿景者）的设计手记
└── guardian/     # Guardian（守护者）的审查手记
```

---

## Visionary 手记

Visionary 在设计过程中的思考：

- 设计灵感和顿悟
- 感受设计的探索过程
- 氛围营造的实验记录
- 比喻和表达技巧

**格式**：`YYYYMMDD-{主题}.md`

---

## Guardian 手记

Guardian 在审查过程中的发现：

- 有效的审查方法
- 常见的问题模式
- 边界情况清单
- 审查技巧和经验

**格式**：`YYYYMMDD-{主题}.md`

---

## 手记 vs 报告

| 类型 | 说明 | 位置 |
|------|------|------|
| **手记** | 个人思考和即时记录 | `journals/` |
| **报告** | 正式的总结和分析 | `reports/` |

手记更随性和即时，报告更结构化和正式。
