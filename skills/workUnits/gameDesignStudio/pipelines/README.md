## 管线格式

每个管线文档应包含：

```markdown
# [管线名称]

## 目的
[这个管线解决什么问题]

## 流程图
[可视化的流程]

## 详细步骤
[每个步骤的说明]

## 输入输出
[每个步骤的输入和输出]

## 注意事项
[关键点和常见问题]
```

---

## 管线 vs 技能

| 类型 | 说明 |
|------|------|
| **管线** | 完整的工作流程 |
| **技能** | 流程中的某个环节方法 |

管线串联多个技能，形成端到端的工作流。
