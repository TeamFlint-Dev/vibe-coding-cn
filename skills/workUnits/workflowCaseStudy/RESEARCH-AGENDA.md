## 📊 主题匹配度评分规则

工作流的"主题匹配度"将影响价值评估（权重 35%）。

### 匹配度评分标准

| 匹配度 | 分数 | 判断标准 |
|--------|------|----------|
| **完全匹配** | 100 | 工作流直接解决 P0 主题的核心问题 |
| **高度相关** | 80 | 工作流涉及 P1 主题的关键方面 |
| **部分相关** | 50 | 工作流涉及 P2 主题或 P1 主题的边缘问题 |
| **间接相关** | 30 | 工作流可能提供相关启发，但非直接解决 |
| **无关** | 0 | 工作流与当前议程主题无关 |

### 评分示例

**场景 1**: 发现一个工作流使用 `lock-for-agent` 进行多 Agent 协调

- 主题: Agent 协作模式（P1）
- 匹配度: 高度相关
- 得分: 80

**场景 2**: 发现一个工作流使用 `schedule` 定时收集指标

- 主题: 工作流可观测性（P1）
- 匹配度: 部分相关（收集指标是可观测性的一部分）
- 得分: 50

## 📝 议程维护指南

### 何时更新议程

1. **发现新需求**: 项目发展产生新的知识需求
2. **完成主题**: 某个主题已充分研究，可以下架
3. **优先级变化**: 项目重点调整，主题优先级需要重新排序
4. **调研完成**: 待调研主题完成评估，决定是否进入主焦点

### 如何更新议程

1. Agent 在运行中发现议程需要调整，创建 Issue 说明理由
2. 人工审核 Issue，决定是否接受
3. 接受后，Agent 或人工更新本文档
4. 在"议程更新历史"中记录变更

### 议程更新原则

- ✅ 基于实际项目需求，不要空想
- ✅ 优先级明确，不要全部标 P0
- ✅ 可量化产出，不要模糊目标
- ✅ 设定时间约束，避免无限膨胀
- ❌ 不要频繁调整，至少运行 5 次后再评估

## 🎯 使用指南

### 对于 workflow-case-study Agent

在 Phase 1（选择研究目标）时：

1. 读取本文档，了解当前焦点主题
2. 评估候选工作流与主题的匹配度
3. 在价值评估框架中，将"主题匹配度"作为 35% 权重维度
4. 选择时优先考虑高匹配度的工作流
5. 在分析报告中说明该工作流如何支持当前议程

### 对于人类维护者

- 定期审查议程的有效性（建议每月一次）
- 根据项目进展调整优先级
- 将已完成的主题移到"议程更新历史"
- 添加新的紧急主题到 P0 或 P1
