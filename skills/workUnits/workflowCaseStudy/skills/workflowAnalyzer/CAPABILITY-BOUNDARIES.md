# Workflow Analyzer 能力边界

## ✅ 能做的事（绿灯区）

| 类别 | 具体能力 |
|------|----------|
| **静态分析** | 分析 Frontmatter 配置 |
| **静态分析** | 分析 Prompt 结构 |
| **模式识别** | 识别常见设计模式 |
| **质量评估** | 评估配置和 Prompt 质量 |
| **知识提取** | 提取可复用的片段 |

## ❌ 不能做的事（红灯区）

| 类别 | 限制说明 |
|------|----------|
| **运行时分析** | 无法分析工作流实际运行行为 |
| **性能分析** | 无法评估 Token 消耗 |
| **跨仓库** | 只能分析本仓库内的工作流 |

## ⚠️ 有条件能做（黄灯区）

| 类别 | 条件 |
|------|------|
| **MCP 分析** | 需要对应 MCP 的文档支持 |
| **复杂条件** | Handlebars 复杂嵌套可能理解不完整 |
