# Autonomy Boundaries Skill

> **用途**: 判断自主行动的边界，决定立即执行还是创建 Issue  
> **阅读时机**: Phase 5（自主行动）开始时

| 区域 | 位置 | 说明 |

| 🟢 绿灯区 | `./rules/GREEN-ZONE.md` | 可立即执行的任务类型 |

| 🟡 黄灯区 | `./rules/YELLOW-ZONE.md` | 需要判断的任务类型 |

---

## 💡 启发式提示

> 在决定是否自主行动前，问自己：

1. **这个操作可逆吗？** 如果做错了，能轻松撤回吗？
2. **影响范围有多大？** 只改一行，还是动很多文件？
3. **我有多确定这是对的？** 100% 确定，还是在赌？

## 🎯 How To Do

### 快速判断（10 秒）

| 判断 | 行动 |
|------|------|
| 修正明显错误（拼写/链接） | → 🟢 立即执行 |
| 增量添加信息 | → 🟢 立即执行 |
| 删除/重构/改核心逻辑 | → 🔴 创建 Issue |
| 不确定 | → 用三原则评估 |

### 三原则评估

| 原则 | 问题 |
|------|------|
| 可逆性 | 做错了能撤回吗？ |
| 影响范围 | 改动涉及多少文件？ |
| 置信度 | 我有多确定？ |

**决策**：
- 三项绿灯 → 执行
- 有一项黄灯 → 可执行，需详细记录
- 有一项红灯 → 创建 Issue

→ 详细规则见 `./rules/` 目录

### 时间预算

| 限制 | 值 |
|------|---|
| 自主行动总时间 | ≤ 10 分钟 |
| 单个任务 | ≤ 5 分钟 |
| 超时 | → 创建 Issue |

## 🔄 进化信号

| 观察 | 行动 |
|------|------|
| 某类自主行动经常出问题 | 移到红灯区 |
| 某类 Issue 总是直接批准 | 移到绿灯区 |
| 发现新的任务类型 | 添加到对应区域 |
