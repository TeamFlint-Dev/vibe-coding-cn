# 四维评估详细标准

## 1. 主题匹配度 (35%)

参考 `RESEARCH-AGENDA.md` 中的议程优先级：

| 匹配程度 | 分数 | 说明 |
|---------|------|------|
| 💯 完全匹配 P0 主题 | 100 | 直接推进核心需求 |
| 🎯 高度相关 P1 主题 | 80 | 重要方向，值得深入 |
| 🔍 部分相关 P2 主题 | 60 | 探索领域，适度投入 |
| 💡 间接相关 | 40 | 可能带来启发 |
| ⏭️ 无关 | 0 | 暂时跳过 |

---

## 2. Skill 空白度 (30%)

评估这个工作流能填补多少现有知识的空白：

| 空白程度 | 分数 | 说明 |
|---------|------|------|
| 🆕 全新领域 | 100 | 我们完全没有这类知识 |
| 📈 显著扩展 | 80 | 能大幅丰富现有分类 |
| ➕ 有所补充 | 60 | 能添加一些新内容 |
| 🔄 边际增量 | 40 | 只是现有知识的变体 |
| ♻️ 重复已知 | 0 | 已有充分的类似案例 |

**如何判断空白度**：
1. 阅读 `workflowAnalyzer/patterns/INDEX.md` 的设计模式列表
2. 快速扫描目标工作流的特征
3. 判断是否有未见过的模式或组合

---

## 3. 模式新颖度 (20%)

评估工作流的设计独特性：

| 新颖程度 | 分数 | 说明 |
|---------|------|------|
| 🌟 范式创新 | 100 | 全新的设计范式 |
| ✨ 独特组合 | 80 | 已知模式的创新组合 |
| 🎨 有趣变体 | 60 | 对常见模式有改良 |
| 📋 标准实现 | 40 | 常规的模式应用 |
| 📎 基础示例 | 0 | 入门级的基础案例 |

**新颖性线索**：
- 文件长度异常长/短？
- 使用了罕见的配置？
- 工具组合不常见？
- 有大量自定义逻辑？

---

## 4. 实用价值 (15%)

评估知识的可迁移性：

| 实用程度 | 分数 | 说明 |
|---------|------|------|
| 🎯 直接复用 | 100 | 可直接用于我们的项目 |
| 🔧 小改可用 | 80 | 稍作调整即可复用 |
| 📚 参考价值 | 60 | 提供思路，但需重新设计 |
| 🎓 学习价值 | 40 | 只有教育意义 |
| 🚫 不可迁移 | 0 | 场景差异太大 |

---

## 综合评分公式

```
总分 = 主题匹配度 × 0.35 + Skill空白度 × 0.30 + 模式新颖度 × 0.20 + 实用价值 × 0.15
```

## 决策阈值

| 总分范围 | 决策 | 行动 |
|---------|------|------|
| **≥ 70** | 高价值 | 立即深入分析 |
| **50-69** | 中等价值 | 快速分析，提取关键点 |
| **30-49** | 低价值 | 简单记录，后续再说 |
| **< 30** | 暂时跳过 | 换一个目标 |
