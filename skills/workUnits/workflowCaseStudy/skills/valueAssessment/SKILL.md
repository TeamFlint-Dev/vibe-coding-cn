## 💡 启发式提示

> 在评估之前，先问自己：

1. **这个工作流在解决什么问题？** 是我们关心的问题吗？
2. **我在 workflowAnalyzer 的模式库里见过类似的吗？** 如果没有，可能是新发现。
3. **如果我学会了它的设计，能用在哪里？** 没有应用场景 = 低价值。

---

## 🎯 How To Do

### Step 1: 快速筛选（30秒）

扫描候选工作流的基本信息，用启发式规则过滤。

→ 详细规则见 `./scoring/HEURISTICS.md`

**快速判断**：
- 🟢 最近更新 / 多MCP / 复杂触发 → 值得看
- 🔴 太短 / 已有类似 / demo → 跳过

### Step 2: 四维打分

对通过筛选的候选，进行四维评分：

| 维度 | 权重 | 核心问题 |
|------|------|---------|
| 主题匹配度 | 35% | 与 RESEARCH-AGENDA 的共鸣？ |
| Skill 空白度 | 30% | 能填补知识空白吗？ |
| 模式新颖度 | 20% | 设计有独特之处吗？ |
| 实用价值 | 15% | 能迁移到我们的场景吗？ |

→ 详细评分标准见 `./scoring/DIMENSIONS.md`

### Step 3: 决策

| 总分 | 行动 |
|------|------|
| ≥ 70 | 深入分析 |
| 50-69 | 快速分析 |
| < 50 | 换一个 |

## 🔄 进化信号

| 观察 | 行动 |
|------|------|
| 高分但实际价值低 | 更新 `./scoring/DIMENSIONS.md` 调整标准 |
| 发现新的价值指标 | 添加到评估维度 |
| 某个维度经常不适用 | 考虑调整权重 |
