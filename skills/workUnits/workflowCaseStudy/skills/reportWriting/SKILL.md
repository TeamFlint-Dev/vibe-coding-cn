# Report Writing Skill

> **用途**: 撰写高质量的分析报告、PR 描述、Issue 和工作日志  
> **阅读时机**: Phase 3（知识沉淀）和 Phase 4（创建 PR）开始时

---

## 📖 索引

| 模板 | 位置 | 用途 |
|------|------|------|
| 分析报告 | `./templates/ANALYSIS-REPORT.md` | 工作流分析报告结构 |
| PR 描述 | `./templates/PR-DESCRIPTION.md` | 调研 PR 和重构 PR |
| Issue | `./templates/ISSUE.md` | Agent 建议的任务 |
| 工作日志 | `./templates/JOURNAL.md` | 每日研究日志 |

---

## 💡 启发式提示

> 在写作之前，先问自己：

1. **读者是谁？** 未来的自己 / 团队成员 / 自动化 Agent？
2. **读者需要什么信息？** 不需要的信息就是噪音。
3. **30 秒能看懂核心吗？** 先结论，后论证。

---

## 🎯 How To Do

### 写分析报告

1. 先写「核心发现」（3-5 句话）
2. 再填充其他章节
3. 检查：新人读了能学到东西吗？

→ 模板见 `./templates/ANALYSIS-REPORT.md`

### 写 PR 描述

1. 标题遵循格式：`[workflow-study]` 或 `[skills-refactor]`
2. 描述控制在 500 字以内
3. 必须包含 checklist

→ 模板见 `./templates/PR-DESCRIPTION.md`

### 写 Issue

1. 标题格式：`[类型] 简洁描述`
2. 说清楚：问题是什么 / 建议方案 / 预期收益
3. 标注决策点

→ 模板见 `./templates/ISSUE.md`

### 写工作日志

1. 一句话摘要
2. 关键发现（3-5 个）
3. Skill 更新记录

→ 模板见 `./templates/JOURNAL.md`

---

## ✅ 质量检查

| 检查项 | 问题 |
|--------|------|
| 结论先行 | 核心发现在最前面吗？ |
| 来源可追溯 | 每个结论都标注了来源吗？ |
| 具体胜于抽象 | 是说「设计得好」还是说「用了 X 模式」？ |
| 可行动 | 读完能直接采取行动吗？ |

---
