# 猜想模式 Skill

> **类型**: 通用模式 - 知识演进  
> **职责**: 提供科学方法论驱动的探索框架  
> **核心理念**: 提出可验证的猜想 → 收集证据 → 证实/证伪/修正 → 衍生新猜想

## 💡 启发式提示

### 何时提出猜想？
- 观察到模式但不确定是否普遍适用？→ **提出猜想**
- 发现反直觉的现象？→ **提出猜想**
- 想验证某个最佳实践？→ **提出猜想**

### 好猜想的特征
- ✅ **可验证**: 有明确的证实/证伪标准
- ✅ **具体**: 不是"X 很好"，而是"X 在 Y 条件下能达到 Z 效果"
- ✅ **有依据**: 基于观察，不是凭空想象

### 坏猜想的特征
- ❌ 无法验证："代码质量很重要"
- ❌ 过于宽泛："所有工作流都应该用 X"
- ❌ 无观察依据：没有看到任何现象就提出

## 🎯 How To Do

### 提出新猜想

1. **观察到现象** → 记录你看到了什么
2. **形成假设** → "我认为...是因为..."
3. **定义验证标准** → 什么能证实？什么能证伪？
4. **使用模板** → `templates/HYPOTHESIS.md`

### 为猜想贡献证据

1. **阅读猜想库** → 了解待验证的猜想
2. **工作中留意** → 遇到相关发现时记录
3. **判断证据类型** → 支持/反例/边界条件
4. **更新证据链** → 按 `templates/EVIDENCE.md` 格式

### 更新猜想状态

1. **检查证据量** → 参考 `rules/STATUS-CRITERIA.md`
2. **判断结论** → 证实/证伪/修正
3. **处理衍生** → 创建子猜想或关联猜想

## 🔄 猜想生命周期

```
提出(proposed)
    │
    ▼
待验证(investigating) ←─────┐
    │                       │
    ├── 收集到支持证据      │
    ├── 收集到反例          │
    └── 发现边界条件 ───────┘
    │
    ▼
┌───────────────────────────────┐
│  证据足够，进入判定           │
├───────────────────────────────┤
│  已证实(confirmed)            │ → 可提炼为最佳实践
│  已证伪(refuted)              │ → 记录为反模式
│  已修正(revised)              │ → 衍生新猜想
│  已合并(merged)               │ → 并入其他猜想
│  已废弃(abandoned)            │ → 不再有验证价值
└───────────────────────────────┘
```

## 🌳 猜想关系图谱

```
[H001] ─────┬──── refines ────→ [H002] (修正版本)
            │
            ├──── derives ────→ [H003] (衍生猜想)
            │
            ├──── contradicts → [H004] (矛盾猜想)
            │
            └──── supports ───→ [H005] (互相支持)

关系类型:
- refines:     H002 是 H001 的精确化版本
- derives:     H003 基于 H001 的结论衍生
- contradicts: H004 与 H001 矛盾，最多一个为真
- supports:    H005 如果成立，增强 H001 可信度
- subsumes:    H001 包含 H006 作为特例
```
