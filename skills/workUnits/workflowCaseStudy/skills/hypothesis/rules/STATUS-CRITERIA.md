# 状态判定标准

> **用途**: 定义何时可以判定猜想为已证实/已证伪/需修正

---

## 状态转换规则

```
proposed → investigating    : 开始收集第一条证据
investigating → confirmed   : 满足证实条件
investigating → refuted     : 满足证伪条件
investigating → revised     : 需要修正猜想表述
investigating → abandoned   : 不再有验证价值
```

---

## 证实条件 (confirmed)

**基本标准**: 满足以下全部条件

| 条件 | 说明 |
|------|------|
| 支持证据 >= 3 | 至少 3 条独立的支持证据 |
| 反例 = 0 | 没有未解释的反例 |
| 证据来源多样 | 来自 2+ 个不同的工作流/场景 |
| 边界已探索 | 至少识别了 1 个边界条件 |

**快速证实**: 如果有 1 条"决定性证据"，可以提前证实

> **决定性证据**: 官方文档明确说明、作者权威解释等

---

## 证伪条件 (refuted)

**基本标准**: 满足以下任一条件

| 条件 | 说明 |
|------|------|
| 强反例 | 1 条无法解释的核心反例 |
| 累积反例 | 反例数 >= 支持证据数 |
| 逻辑矛盾 | 猜想本身存在逻辑问题 |

**证伪后处理**:
1. 如果反例揭示了边界 → 考虑 `revised`
2. 如果猜想完全错误 → `refuted` + 记录为反模式

---

## 修正条件 (revised)

**触发条件**:

| 情况 | 处理 |
|------|------|
| 发现适用边界 | 创建 H{编号}': 在 {边界} 条件下，原猜想成立 |
| 需要精确化 | 创建 H{编号}': 更精确的表述 |
| 部分成立 | 拆分为多个子猜想 |

**修正后**:
- 原猜想标记为 `revised`
- 关联到新猜想 (refines 关系)

---

## 废弃条件 (abandoned)

**触发条件**:

| 情况 | 说明 |
|------|------|
| 无法验证 | 现有手段无法收集证据 |
| 失去相关性 | 底层 API 变化导致猜想不再相关 |
| 被合并 | 已被其他猜想包含 |
| 长期无进展 | 3 个月无新证据 |

---

## 判定流程

```
检查证据链
    │
    ├── 支持 >= 3 且 反例 = 0 且 来源多样？
    │       └── 是 → confirmed
    │
    ├── 有强反例 或 反例 >= 支持？
    │       └── 是 → 反例可解释？
    │                   ├── 是 → revised (创建新版本)
    │                   └── 否 → refuted
    │
    ├── 3 个月无新证据？
    │       └── 是 → abandoned
    │
    └── 以上都不满足？
            └── 保持 investigating
```

