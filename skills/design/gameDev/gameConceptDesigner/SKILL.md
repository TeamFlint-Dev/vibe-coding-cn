# gameConceptDesigner Skill

通过结构化对话，将模糊的游戏创意转化为清晰的概念文档。聚焦核心玩法、目标玩家和独特卖点。

## When to Use This Skill

触发此 Skill 当：

- 用户有游戏创意但不知道如何系统化
- 需要定义游戏的核心玩法
- 需要明确目标玩家画像
- 需要提炼游戏的独特卖点（USP）
- orchestrator 调度到概念设计阶段

## Not For / Boundaries

此 Skill 不负责：

- 详细的系统设计（交给 system-designer）
- 具体的数值平衡（交给 economy-designer）
- 技术实现方案（交给 tech-stack-planner）
- 代码架构设计（交给 architecture-planner）

必需输入：

- 用户的游戏创意（至少一句话描述）

如果用户输入不足，询问以下问题（每次 1-3 个）：

1. 你想做什么类型的游戏？
2. 玩家在游戏中主要做什么？
3. 有没有参考的游戏？

## Quick Reference

### 对话阶段

| 阶段 | 目标 | 核心问题 |
|------|------|----------|
| 1. 类型定位 | 确定游戏类型 | 什么类型？参考哪些游戏？ |
| 2. 核心玩法 | 定义核心循环 | 玩家做什么？为什么有趣？ |
| 3. 目标玩家 | 明确受众 | 谁会玩？他们想要什么？ |
| 4. 独特卖点 | 提炼差异化 | 和其他游戏有什么不同？ |
| 5. 确认总结 | 输出文档 | 生成 @concept.md |

### 核心循环模板

```
[动作] → [反馈] → [奖励] → [成长] → [动作]

例：
钓鱼 → 获得物品 → 展示赚钱 → 解锁槽位 → 钓更多鱼
```

### 常见游戏类型

| 类型 | 核心特征 | 典型循环 |
|------|----------|----------|
| 放置/挂机 | 自动产出，离线收益 | 收集 → 放置 → 产出 → 升级 |
| 养成 | 角色/物品成长 | 获取 → 培养 → 强化 → 展示 |
| 模拟经营 | 资源管理，扩张 | 生产 → 销售 → 扩张 → 优化 |
| Roguelike | 随机性，永久死亡 | 探索 → 战斗 → 死亡 → 继承 |
| 社交 | 玩家互动 | 创建 → 分享 → 互动 → 获得 |

```markdown
# 游戏概念文档

> **项目名称**：[名称]
> **游戏类型**：[类型]
> **目标平台**：[平台]
> **创建日期**：[日期]

## 一句话描述

[用一句话概括游戏的核心体验]

## 核心玩法

### 核心循环

```

[动作1] → [动作2] → [动作3] → [动作4] → (循环)

```

### 玩家做什么

- [主要行为1]
- [主要行为2]
- [主要行为3]

### 为什么有趣

- [乐趣来源1]
- [乐趣来源2]

## 目标玩家

### 玩家画像

- **年龄段**：[范围]
- **游戏偏好**：[类型]
- **游玩时长**：[频率]
- **付费意愿**：[等级]

### 玩家需求

- [需求1]
- [需求2]

## 独特卖点 (USP)

1. [卖点1]
2. [卖点2]
3. [卖点3]

## 参考游戏

| 游戏 | 借鉴点 | 差异点 |
|------|--------|--------|
| [游戏1] | [借鉴] | [差异] |
| [游戏2] | [借鉴] | [差异] |

## MVP 范围

### 包含

- [功能1]
- [功能2]
- [功能3]

- [功能A]
- [功能B]
```

### 对话引导模式

**引导式询问**：每次聚焦 1-3 个问题

```
基于你描述的[XX]，我想确认几点：
1. [问题1]？
2. [问题2]？
```

**决策确认**：总结后再继续

```
让我确认一下理解：
- 核心玩法是[XX]
- 目标玩家是[YY]
这样理解对吗？
```

**可视化辅助**：用图表展示

```
核心循环：
  收集 ──→ 展示 ──→ 收益
    ↑                 │
    └──── 升级 ←──────┘
```

## Examples

### Example 1: 放置养成游戏

**输入**: "我想做一个放置养成游戏，玩家可以收集物品然后展示赚钱"

**对话过程**:

1. 确认类型：放置 + 养成
2. 挖掘核心：收集什么？如何展示？收益机制？
3. 明确玩家：休闲玩家，碎片时间
4. 提炼卖点：独特的收藏主题？特殊的展示机制？

**输出**: 完整的 @concept.md

### Example 2: 信息不足

**输入**: "我想做个游戏"

**对话过程**:

1. 询问类型："你想做什么类型的游戏？动作、策略、休闲还是其他？"
2. 询问核心："玩家在游戏中主要做什么？"
3. 询问参考："有没有你喜欢的、想借鉴的游戏？"

**输出**: 收集足够信息后再生成 @concept.md

### Example 3: 已有详细想法

**输入**: "我要做一个UEFN岛屿养成游戏，核心循环是钓鱼获取物品，放到展示槽赚钱，用钱解锁更多槽位"

**对话过程**:

1. 确认理解正确
2. 补充细节：物品稀有度？收益计算？
3. 明确边界：MVP 包含什么？

**输出**: 直接生成 @concept.md，少量确认问题
