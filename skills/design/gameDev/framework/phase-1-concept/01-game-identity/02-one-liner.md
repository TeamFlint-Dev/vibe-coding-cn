# 02 一句话描述 (One-Liner)

> 用一句话概括游戏的核心体验

## 🎯 核心问题

**如果只能用一句话描述这个游戏，你会怎么说？**

## 📋 任务说明

一句话描述是游戏身份的最精简版本，它应该：

1. **简短**：15个字以内（中文）或 10个词以内（英文）
2. **清晰**：任何人都能理解
3. **吸引**：让人想了解更多
4. **独特**：体现游戏的差异化

用途：
- 社交媒体标签
- 朋友推荐游戏时的快速介绍
- 营销物料的主标题
- 团队对齐的口号

## 🔍 引导问题

1. 玩家在游戏中**最主要**做什么？
2. 游戏给玩家的**核心感受**是什么？
3. 如果用【动词+名词】的结构，是什么？
4. 游戏最独特的一个词是什么？
5. 什么词能让目标玩家产生共鸣？

## 📝 输出模板

### 结构模板

选择一个适合你游戏的结构：

**模板 1：动作 + 目标**
```
[动词] + [对象] + [在哪里/如何]
```

**模板 2：身份 + 行为**
```
成为 [角色]，[做什么]
```

**模板 3：体验 + 特色**
```
[核心体验] with [独特机制]
```

**模板 4：类比**
```
[熟悉元素] meets [熟悉元素]
```

## ✅ 验收标准

- [ ] 15个字以内（中文）或 10个词以内（英文）
- [ ] 包含动词（描述玩家行为）
- [ ] 任何人都能理解（无行业黑话）
- [ ] 体现游戏独特性
- [ ] 能脱口而出，容易记忆
- [ ] 团队成员对此描述达成一致

## 💡 示例

### 案例：钓鱼放置养成游戏

**一句话描述**：
```
建造海底钓鱼帝国，收集奇幻海洋生物
```

**分析**：
- ✅ 动词：建造、收集
- ✅ 核心元素：钓鱼、海洋生物
- ✅ 独特性：海底、帝国（规模感）
- ✅ 字数：14字

**其他可选版本**：
```
放置钓鱼，离线也在成长          （10字，强调放置特性）
收集数百种海洋生物的钓鱼养成    （13字，强调收集）
轻松钓鱼，打造水下乐园          （10字，强调休闲）
```

### 案例：知名游戏参考

| 游戏 | 一句话描述 | 分析 |
|------|-----------|------|
| **《吸血鬼幸存者》** | "对抗成千上万敌人的弹幕射击" | 核心冲突 + 规模感 |
| **《星露谷物语》** | "在乡村开始新生活" | 转变 + 场景 |
| **《戴森球计划》** | "建造跨星系的工业帝国" | 行为 + 规模 |
| **《动物森友会》** | "在无人岛打造理想生活" | 场景 + 目标 |
| **《哈迪斯》** | "从地狱逃脱的Roguelike" | 目标 + 类型 |
| **《宝可梦》** | "收集训练对战神奇宝贝" | 核心循环三动词 |
| **《我的世界》** | "建造你想象中的一切" | 自由度 + 创造 |
| **《黑暗之魂》** | "在绝望中挑战极限" | 情感 + 挑战 |

### 不同侧重点的描述

同一个钓鱼游戏，根据侧重点不同：

**侧重收集**：
```
收集数百种奇幻海洋生物
```

**侧重放置**：
```
离线也能钓鱼的放置游戏
```

**侧重建造**：
```
打造你的海底钓鱼王国
```

**侧重治愈**：
```
禅意钓鱼，随时放松
```

选择哪个取决于：
- 游戏最核心的卖点是什么
- 目标玩家最看重什么
- 市场上缺少什么定位

## 💭 常见错误

### ❌ 错误1：过于宽泛

```
一个有趣的游戏
充满挑战的冒险
创新的游戏体验
```

**问题**：可以套用到任何游戏，毫无识别度。

### ❌ 错误2：堆砌特性

```
建造、收集、战斗、探索、社交、养成的RPG
```

**问题**：什么都想说，反而什么都没说清楚。

### ❌ 错误3：使用行业术语

```
Roguelike肉鸽DBG卡牌构筑游戏
```

**问题**：只有核心玩家能懂，排除了潜在受众。

### ❌ 错误4：没有动词

```
海底世界的故事
神秘的钓鱼传说
```

**问题**：不知道玩家要"做"什么。

### ✅ 改进版本

```
在海底建造自动钓鱼帝国
```

- ✅ 有动词（建造）
- ✅ 有场景（海底）
- ✅ 有特色（自动、帝国）
- ✅ 易懂易记

## 📚 提示

### 如何测试你的一句话描述

1. **5秒测试**：让陌生人看5秒，问他们记住了什么
2. **转述测试**：让别人用自己的话复述，看是否理解正确
3. **竞品对比**：放在同类游戏旁边，是否有识别度
4. **社交测试**：发到社交媒体，看点击率和反馈

### 迭代建议

1. 第一版：先写10个不同的版本
2. 筛选：选出3个最喜欢的
3. 测试：让目标玩家投票
4. 优化：根据反馈调整用词
5. 定稿：全团队确认
