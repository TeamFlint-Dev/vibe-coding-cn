# 76 进度终点

> 定义游戏的进度终点维度

## 🎯 核心问题

**游戏有终点吗？终点后是什么？**

## 📋 任务说明

本任务帮助你思考游戏设计中的"进度终点"，确保这个维度的设计完整性。

### 为什么重要

进度终点直接影响玩家的体验和游戏的整体质量。清晰定义这个维度能够：

- 帮助团队对齐理解
- 为后续设计提供指导
- 确保设计的一致性

## 🔍 引导问题

1. 从玩家角度：玩家会如何感受这个维度？
2. 从设计角度：这个维度如何影响其他设计？
3. 从竞品角度：其他游戏如何处理？
4. 从创新角度：我们能做什么不同的？

## 📝 输出模板

```markdown
### 进度终点

#### 核心定义
[用1-2句话定义]

#### 详细说明
[具体阐述]

#### 设计影响
[对其他系统的影响]

#### 验证方式
[如何验证这个设计]
```

## ✅ 验收标准

- [ ] 完成核心定义
- [ ] 回答了所有引导问题
- [ ] 与整体设计保持一致
- [ ] 团队达成共识

## 💡 示例：钓鱼放置养成游戏

```markdown
### 进度终点

（示例：钓鱼游戏在此维度的设计）

这是一个放置钓鱼游戏的进度终点设计。
玩家在海底建造自动化钓鱼设施，收集奇幻海洋生物。

具体设计要点：
- [要点1]
- [要点2]
- [要点3]
```
