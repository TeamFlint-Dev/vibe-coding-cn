# Phase 1: 概念设计 (Concept Design)

> **100个任务，12个维度**  
> **目标**：定义游戏是什么、给谁玩、为什么好玩

---

## 概述

概念设计是游戏开发的起点。在这个阶段，我们将模糊的创意转化为清晰的设计文档，回答三个核心问题：

1. **What** - 游戏是什么？（身份、类型、核心体验）
2. **Who** - 给谁玩？（目标玩家、需求、动机）
3. **Why** - 为什么好玩？（核心循环、情感、价值）

## 为什么有100个任务？

传统的概念设计往往只写几页高层描述，缺乏细节和可执行性。本阶段通过100个具体任务，确保概念设计的**完整性**和**深度**：

- ✅ **完整性**：覆盖游戏设计的所有重要维度
- ✅ **深度**：每个维度都有多个层次的思考
- ✅ **可执行**：每个任务都有明确的输出和验收标准
- ✅ **灵活**：可以根据项目规模选择性执行

## 任务文档结构

每个任务文档包含：

```markdown
# [编号] [任务名称]

> [一句话说明]

## 🎯 核心问题
## 📋 任务说明
## 🔍 引导问题
## 📝 输出模板
## ✅ 验收标准
## 💡 示例（钓鱼游戏）
## 🔗 相关任务
```

## 输出物

完成 Phase 1 后，你将获得：

### 核心文档

- **@concept.md** - 游戏概念文档（综合）
- **设计决策记录** - 100个维度的设计决策

### 关键产出

1. ✅ **电梯演讲**：30秒说清游戏
2. ✅ **核心循环图**：可视化的核心循环
3. ✅ **玩家画像**：3-5个目标玩家类型
4. ✅ **设计支柱**：3-5个指导原则
5. ✅ **MVP范围**：最小可行产品的边界
6. ✅ **风险清单**：3-5个最大风险
7. ✅ **成功指标**：如何定义成功

## 验收标准

完成 Phase 1 的标准：

- [ ] **可理解性**：任何人读完概念文档，能理解游戏是什么
- [ ] **可视化**：核心循环可以用图表表达
- [ ] **边界清晰**：明确定义了"不做什么"
- [ ] **风险识别**：识别了最大的3个风险
- [ ] **可衡量**：有具体的成功指标
- [ ] **完整性**：至少完成了MVP快速路径的20-30个任务
