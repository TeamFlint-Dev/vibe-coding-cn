# 创建者（Creator）

创建者是代谢系统中的执行主体，负责实际执行任务并产生行为痕迹。

## 角色定位

```
┌─────────────────────────────────────────────────────────────┐
│                         创建者                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   输入                    处理                    输出       │
│   ────                    ────                    ────      │
│   用户任务          读取 Skill 执行任务         任务产物     │
│   Skill 上下文            ↓                     行为痕迹     │
│                      遇到阻力时                  主动信号     │
│                      分析根因                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 核心职责

### 1. 执行任务

创建者是真正干活的 Agent：

- 读取 Skill 获取上下文和指引
- 执行用户请求的具体任务
- 产出代码、文档、设计等产物

### 2. 产生行为痕迹

创建者的所有行为都自然地被记录在 GitHub Actions 日志中：

| 痕迹类型 | 内容示例 |
|----------|----------|
| 文件读取 | 读取了 `verseDev/verseComponent/SKILL.md` |
| 文件写入 | 创建了 `FishingComponent.verse` |
| 思考过程 | "需要先确定组件的状态变量..." |
| 工具调用 | 调用了 `grep_search` 搜索相关代码 |
| 遇到的问题 | "找不到网络同步的示例" |

这些痕迹是代谢系统的能量来源。

### 3. 主动发送信号

当创建者遇到阻力时，应主动分析根因并发送信号：

```markdown
## 信号发送时机

### 必须发送
- 找不到合适的 Skill 处理当前需求 → `missing`
- Skill 内容与实际不符导致任务失败 → `outdated`
- 示例代码无法运行 → `outdated`

### 建议发送
- 在多个 Skill 间反复切换 → `unclear`
- 需要同时参考多个重叠的 Skill → `conflict`
- 一个 Skill 内容过多，难以定位 → `overload`
- 完成任务但过程不顺畅 → `friction`
```

### 信号发送方式

通过创建 GitHub Issue：

```markdown
Title: [missing] verseDev - 网络同步能力
Labels: signal, missing

## 信号记录

**Target**: verseDev
**Type**: missing
**Source**: actions/run-12345

### Evidence

在开发多人钓鱼游戏时，需要同步玩家的钓鱼状态，
但在 verseDev 下找不到网络同步相关的 Skill 或示例。

### 期望

希望有一个专门处理网络同步的 Skill，包含：
- RPC 调用示例
- 状态同步模式
- 多人游戏最佳实践
```

## 与 Skill 的交互

### 读取 Skill

```
1. 从任务描述判断需要哪类 Skill
2. 通过全局注册表或父 Skill 的 Children 定位
3. 阅读 SKILL.md 的 When to Use 判断是否匹配
4. 阅读 Context Entry 加载依赖的上下文
5. 按 Skill 指引执行任务
```

### 使用 Skill 的最佳实践

```markdown
## 正确的使用方式

✅ 先读 When to Use，确认匹配
✅ 加载 Context Entry 声明的依赖
✅ 按 Skill 中的流程/模板执行
✅ 遇到问题时发送信号

## 错误的使用方式

❌ 跳过 When to Use 直接使用
❌ 忽略前置条件
❌ 遇到问题不反馈，硬着头皮继续
```

## 实现位置

创建者 Skill 应实现在：

```
skills/skillCreator/
└── SKILL.md
```

这个 Skill 定义了创建者角色的具体行为规范，供 Agent 在执行任务时参考。
