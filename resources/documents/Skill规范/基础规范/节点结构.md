# 节点结构

## 基本结构

每个 Skill 是一个文件夹，必须包含 `SKILL.md` 作为入口。

### 最小结构（叶子节点）

```
skillName/
└── SKILL.md           # 必须：入口文件
```

### 完整结构（分支节点）

```
skillName/
├── SKILL.md           # 必须：入口文件
├── childSkillA/       # 可选：子 Skill
│   └── SKILL.md
├── childSkillB/       # 可选：子 Skill
│   └── SKILL.md
├── checklists/        # 可选：检查清单
├── templates/         # 可选：模板文件
├── references/        # 可选：参考资料
└── @changelog.md      # 可选：变更日志
```

## 入口文件规则

### 统一使用 SKILL.md

所有 Skill 统一使用 `SKILL.md` 作为入口，不使用：

- ❌ `Index.md`
- ❌ `README.md`
- ❌ 其他变体

### 入口文件职责

SKILL.md 负责：

1. 声明 Skill 的元数据（Frontmatter）
2. 说明何时使用此 Skill
3. 提供上下文入口
4. 列出子 Skill（如有）

## 子目录规范

### 语义化命名

子目录应使用有意义的名称，反映其内容类型：

| 目录名 | 用途 |
|--------|------|
| `checklists/` | 检查清单、审计依据 |
| `templates/` | 模板文件、请求格式 |
| `references/` | 参考资料、外部文档 |
| `examples/` | 使用示例 |
| `scripts/` | 辅助脚本 |

### 子 Skill 目录

子 Skill 是独立的 Skill 节点，必须：

- 使用驼峰命名（如 `childSkillA/`）
- 包含自己的 `SKILL.md`
- 符合完整的 Skill 规范

## 元数据文件

以 `@` 开头的文件是元数据文件，用于存储动态信息：

| 文件名 | 用途 |
|--------|------|
| `@changelog.md` | 变更日志 |
| `@feedback.md` | 使用反馈 |
| `@audit-report.md` | 审计报告 |

元数据文件不是必须的，根据 Skill 的进化需求添加。

## 目录层级

### 无深度限制

Skill 树没有强制的层级深度限制：

```
rootSkill/
└── level1/
    └── level2/
        └── level3/
            └── ...
```

### 深层访问

深层节点可以通过以下方式高效访问：

1. 父节点的 Children 章节提供直接入口
2. 全局注册表提供扁平化索引
3. 语义搜索直接定位

## 与外部资源的关系

### 资源存放位置

资源可以存放在 Skill 外部：

- 公共 API 文档
- 跨 Skill 共享的模板
- 大型参考资料

### 入口提供方式

Skill 必须在 SKILL.md 中提供外部资源的入口：

```markdown
## Context Entry

### 外部资源

| 资源 | 路径 | 用途 |
|------|------|------|
| API Digest | `../shared/api-digests/` | 查询 API |
| 架构模式库 | `../shared/architecture-library.md` | 参考架构 |
```

## 结构示例

### 叶子节点示例

```
verseHelpers/
├── SKILL.md
└── templates/
    └── helper-request.md
```

### 分支节点示例

```
verseDev/
├── SKILL.md                    # 生态入口
├── verseOrchestrator/          # 子 Skill
│   └── SKILL.md
├── verseComponent/             # 子 Skill
│   ├── SKILL.md
│   └── checklists/
│       └── component-checklist.md
├── verseCodeLibrary/           # 资源型子 Skill
│   ├── SKILL.md
│   └── Components/
│       └── ...
└── @changelog.md               # 元数据
```
