# 章节模板

## 标准章节结构

SKILL.md 应按以下顺序组织章节：

```markdown
---
[Frontmatter]
---

# [Skill 标题]

## When to Use

## Context Entry

## Children（仅分支节点）

## Self-Reflection

## [其他可选章节]
```

## 必须章节

### When to Use

**目的**：让 Agent 快速判断是否需要使用此 Skill。

**内容要求**：

1. 明确的触发条件
2. 适用场景列表
3. 不适用场景（Boundaries）

**模板**：

```markdown
## When to Use

### 适用场景

当满足以下条件时，使用此 Skill：

- 条件 1：具体描述
- 条件 2：具体描述
- 条件 3：具体描述

### 不适用场景

以下情况不应使用此 Skill：

- 场景 1：应使用 [替代 Skill]
- 场景 2：应使用 [替代 Skill]
```

### Context Entry

**目的**：声明此 Skill 的上下文能力，包括提供和依赖两个维度。

**内容要求**：

1. 作为提供者：别的 Skill 何时需要读我
2. 作为消费者：我执行时需要读谁
3. 外部资源入口：如何访问外部资源

**模板**：

```markdown
## Context Entry

### 本 Skill 提供的上下文

当其他 Skill 需要以下能力时，应读取本 Skill：

| 能力 | 入口文件 | 说明 |
|------|----------|------|
| 能力 A | `./file-a.md` | 简要说明 |
| 能力 B | `./file-b.md` | 简要说明 |

### 本 Skill 依赖的上下文

执行本 Skill 前，需确保已加载：

| 依赖 | 来源 | 必需/可选 |
|------|------|-----------|
| 依赖 A | `path/to/skill` | 必需 |
| 依赖 B | `path/to/skill` | 可选 |

### 外部资源入口

| 资源 | 路径 | 用途 |
|------|------|------|
| 资源 A | `path/to/resource` | 说明 |
```

## 分支节点必须章节

### Children

**目的**：列出所有子 Skill，并提供选择指引。

**内容要求**：

1. 子 Skill 列表及简介
2. 选择流程或决策树
3. 常见组合模式

**模板**：

```markdown
## Children

### 子 Skill 列表

| 子 Skill | 说明 | 适用场景 |
|----------|------|----------|
| [childA](childA/SKILL.md) | 简介 | 场景 |
| [childB](childB/SKILL.md) | 简介 | 场景 |

### 选择指引

```

需要什么？
├─ 需求 A → childA
├─ 需求 B → childB
└─ 需求 C → childA + childB

```

### 常见组合

- **场景 1**：childA → childB
- **场景 2**：childB（独立使用）
```

## 可选章节

### Self-Reflection

**目的**：声明 Skill 的局限性和改进方向，支持自进化。

**模板**：

```markdown
## Self-Reflection

### 已知局限

- 局限 1：描述
- 局限 2：描述

### 待改进项

- [ ] 改进项 1
- [ ] 改进项 2

### 依赖假设

- 假设 1：如果不成立，影响是...
- 假设 2：如果不成立，影响是...
```

### Quick Reference

**目的**：提供快速参考信息，如命令、API、配置等。

```markdown
## Quick Reference

### 常用命令

| 命令 | 说明 |
|------|------|
| `command1` | 说明 |

### 关键配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `config1` | `value` | 说明 |
```

### Examples

**目的**：提供使用示例。

```markdown
## Examples

### 示例 1：标题

场景描述...

```code
示例代码
```

### 示例 2：标题

...

```

### Related Skills

**目的**：提供相关 Skill 的链接。

```markdown
## Related Skills

- [相关 Skill 1](path/to/skill) - 关系说明
- [相关 Skill 2](path/to/skill) - 关系说明
```

## 章节顺序

推荐的章节顺序（从上到下）：

1. Frontmatter
2. 标题
3. **When to Use**（必须）
4. **Context Entry**（必须）
5. **Children**（分支节点必须）
6. Quick Reference
7. Examples
8. Self-Reflection
9. Related Skills

## 语言规范

- 使用中文编写
- 技术术语可保留英文（如 Skill、Agent、Frontmatter）
- 标题保持英文（如 When to Use、Context Entry），正文用中文
- 代码示例使用英文标识符
