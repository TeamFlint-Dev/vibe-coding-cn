# 元数据规范

## Frontmatter 格式

SKILL.md 必须以 YAML Frontmatter 开头：

```yaml
---
name: skillName
description: "触发条件 + 能力描述"
status: stable
version: 1.0.0
created: 2026-01-02
lastAudit: 2026-01-02
---
```

## 必须字段

### name

**类型**: string  
**格式**: 中文

Skill 的唯一标识符，必须与目录名一致。

```yaml
name: verseComponent
```

### description

**类型**: string  
**要求**: 必须包含可判定的触发条件

description 是 Agent 判断是否需要此 Skill 的关键依据，应包含：
1. **触发条件**：何时需要使用
2. **能力描述**：能做什么

**好的示例**：
```yaml
description: "当需要开发 Verse 组件时，提供组件设计、编码和审计的完整流程"
```

**不好的示例**：
```yaml
description: "组件开发工具"  # 过于简短，无法判定
```

## 可选字段

### status

**类型**: enum  
**可选值**: `draft` | `stable` | `deprecated`  
**默认值**: `stable`

Skill 的生命周期状态：

| 状态 | 含义 | Agent 行为 |
|------|------|-----------|
| `draft` | 草稿，正在开发中 | 谨慎使用，可能不完整 |
| `stable` | 稳定，可正常使用 | 正常使用 |
| `deprecated` | 废弃，不建议使用 | 寻找替代方案 |

```yaml
status: stable
```

### version

**类型**: string  
**格式**: 语义化版本 `major.minor.patch`

```yaml
version: 1.2.0
```

版本号变更规则：
- `major`：不兼容的重大变更
- `minor`：向后兼容的功能新增
- `patch`：向后兼容的问题修复

### created

**类型**: date  
**格式**: `YYYY-MM-DD`

Skill 创建日期。

```yaml
created: 2026-01-02
```

### lastAudit

**类型**: date  
**格式**: `YYYY-MM-DD`

上次审计日期，用于触发定期审计。

```yaml
lastAudit: 2026-01-02
```

### tags

**类型**: string[]

用于全局注册表的语义标签。

```yaml
tags:
  - verse
  - uefn
  - component
  - scenegraph
```

### complexity

**类型**: enum  
**可选值**: `low` | `medium` | `high`

Skill 的复杂度评估，帮助 Agent 规划执行时间。

```yaml
complexity: medium
```

### parent

**类型**: string

父 Skill 的路径，用于逆向导航。

```yaml
parent: programming/verseDev
```

## 完整示例

### 叶子节点

```yaml
---
name: verseHelpers
description: "当需要开发 Verse Helper 函数时，提供模块化、可复用的工具函数开发规范"
status: stable
version: 1.0.0
created: 2026-01-02
tags:
  - verse
  - helpers
  - utility
complexity: low
parent: programming/verseDev
---
```

### 分支节点

```yaml
---
name: verseDev
description: "当进行 UEFN/Verse 游戏开发时，提供完整的开发技能生态，包括架构设计、组件开发、事件流设计等"
status: stable
version: 2.0.0
created: 2025-06-15
lastAudit: 2026-01-02
tags:
  - verse
  - uefn
  - game-dev
  - scenegraph
complexity: high
---
```

## 验证规则

| 字段 | 验证规则 |
|------|----------|
| name | 必须与目录名一致 |
| description | 不能少于 20 个字符 |
| description | 应包含"当...时"的触发条件 |
| status | 必须是有效的枚举值 |
| version | 必须符合语义化版本格式 |
| created | 必须是有效日期 |
