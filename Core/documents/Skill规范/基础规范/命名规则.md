# 命名规则

## 目录命名

### Skill 目录

使用**小驼峰命名法**（lowerCamelCase）：

```
✅ verseDev
✅ verseComponent
✅ gameConceptDesigner
✅ githubActionsWorkflows

❌ verse-dev          # 禁止使用连字符
❌ verse_dev          # 禁止使用下划线
❌ VerseDev           # 禁止大写开头
❌ VERSE_DEV          # 禁止全大写
```

**原因**：
- UEFN 编译器对 `-` 等特殊字符敏感
- 保持与代码标识符风格一致
- 便于在各种系统间传递

### 语义化子目录

功能性子目录使用**全小写**，可用连字符分隔：

```
✅ checklists/
✅ templates/
✅ references/
✅ code-library/       # 多词时可用连字符

❌ Checklists/         # 禁止首字母大写
❌ TEMPLATES/          # 禁止全大写
```

### 子 Skill 目录

子 Skill 目录遵循与顶层 Skill 相同的命名规则：

```
verseDev/
├── verseOrchestrator/    # 子 Skill，驼峰命名
├── verseComponent/       # 子 Skill，驼峰命名
└── templates/            # 功能目录，小写命名
```

## 文件命名

### SKILL.md

入口文件必须命名为 `SKILL.md`（全大写）：

```
✅ SKILL.md

❌ skill.md
❌ Skill.md
❌ index.md
❌ README.md
```

### 元数据文件

以 `@` 开头，使用连字符分隔：

```
✅ @changelog.md
✅ @audit-report.md
✅ @feedback.md

❌ changelog.md         # 缺少 @ 前缀
❌ @changeLog.md        # 不要用驼峰
```

### 普通 Markdown 文件

使用连字符分隔的小写命名：

```
✅ component-checklist.md
✅ architecture-guide.md
✅ quick-start.md

❌ componentChecklist.md  # 不要用驼峰
❌ Component_Checklist.md # 不要用下划线
```

### 代码和模板文件

根据文件类型使用相应的命名约定：

```
# Verse 代码
✅ FishingComponent.verse
✅ StateManager.verse

# JSON/YAML 配置
✅ tactical-schema.json
✅ registry.yaml

# 脚本文件
✅ build.ps1
✅ deploy.sh
```

## Frontmatter 中的 name 字段

必须与目录名完全一致：

```yaml
# 目录：verseComponent/SKILL.md
---
name: verseComponent    # ✅ 与目录名一致
---

# 错误示例
---
name: verse-component   # ❌ 与目录名不一致
name: VerseComponent    # ❌ 大小写不一致
---
```

## 路径引用

### 相对路径

在 SKILL.md 中引用资源时使用相对路径：

```markdown
✅ [子 Skill](childSkill/SKILL.md)
✅ [检查清单](checklists/code-quality.md)
✅ [外部资源](../shared/api-digests/)

❌ [绝对路径](/Core/skills/...)  # 避免绝对路径
```

### 链接文本

链接文本应该有意义：

```markdown
✅ 参考 [组件开发指南](component-guide.md)
✅ 使用 [verseComponent](verseComponent/SKILL.md) 进行开发

❌ 点击 [这里](component-guide.md)
❌ [链接](verseComponent/SKILL.md)
```

## 命名一致性检查

创建或修改 Skill 时，确保：

| 检查项 | 说明 |
|--------|------|
| 目录名 = frontmatter.name | 必须完全一致 |
| 目录名符合驼峰规范 | 无特殊字符 |
| SKILL.md 全大写 | 不是 skill.md |
| 子目录命名正确 | Skill 用驼峰，功能目录用小写 |
| 链接路径有效 | 所有链接可访问 |

## 特殊情况

### 缩写词

常见缩写词全大写或全小写，不要混合：

```
✅ apiDigest
✅ uiComponent
✅ httpClient

❌ ApiDigest      # API 应该小写开头或全大写
❌ UIComponent    # 首字母应小写
```

### 数字

避免以数字开头：

```
✅ version2Handler
✅ layer5Skill

❌ 2dRenderer
❌ 3dComponent
```
