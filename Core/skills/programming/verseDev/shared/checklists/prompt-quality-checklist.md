# Prompt 质量检查清单

> 用于 versePromptAuditor 的 Skill Prompt 质量检查依据

---

## 职责清晰度

### PRM-001: 核心职责定义

**检查**: SKILL.md 必须有明确的职责声明

**必需元素**:
```markdown
> **类型**: [层级/角色]  
> **职责**: [一句话核心职责描述]
```

**评分标准**:
- ✅ 完整：有类型和职责声明
- ⚠️ 部分：只有其中之一
- ❌ 缺失：没有职责声明

---

### PRM-002: 职责边界

**检查**: 明确"做什么"和"不做什么"

**必需元素**:
```markdown
**职责范围**:
- ✅ [应该做的事项]
- ❌ [不应该做的事项]
```

**评分标准**:
- ✅ 完整：有正面和负面边界
- ⚠️ 部分：只有正面描述
- ❌ 缺失：没有边界定义

---

## 输入输出

### PRM-003: 期望输入

**检查**: 明确说明 Skill 需要什么信息

**必需元素**:
```markdown
**期望输入**:
- [输入项 1]
- [输入项 2]
- ...
```

**评分标准**:
- ✅ 完整：列出所有必要输入
- ⚠️ 部分：输入描述不完整
- ❌ 缺失：没有输入说明

---

### PRM-004: 输出格式

**检查**: 明确说明输出的格式和结构

**必需元素**:
```markdown
**输出格式**:
[代码模板或结构说明]
```

**评分标准**:
- ✅ 完整：有代码模板
- ⚠️ 部分：只有文字描述
- ❌ 缺失：没有输出说明

---

## 示例完整性

### PRM-005: 示例数量

**检查**: 至少 2 个不同场景的示例

**要求**:
- 最少 2 个示例
- 覆盖不同使用场景
- 从简单到复杂递进

**评分标准**:
- ✅ 充足：≥2 个不同场景示例
- ⚠️ 不足：只有 1 个示例
- ❌ 缺失：没有示例

---

### PRM-006: 示例质量

**检查**: 示例应该是可运行的完整代码

**要求**:
- 代码完整可运行
- 有适当的注释
- 展示最佳实践

**评分标准**:
- ✅ 高质量：完整、有注释、展示最佳实践
- ⚠️ 中等：完整但缺少注释
- ❌ 低质量：代码片段不完整

---

## 协作接口

### PRM-007: 上下游关系

**检查**: 明确与其他 Skill 的协作关系

**必需元素**:
```markdown
**上游依赖**:
- `[Skill名]`: [提供什么]

**下游服务**:
- `[Skill名]`: [提供什么]
```

**评分标准**:
- ✅ 完整：上下游都有说明
- ⚠️ 部分：只说明了一方
- ❌ 缺失：没有协作说明

---

### PRM-008: 接口格式

**检查**: 与其他 Skill 的数据交换格式一致

**要求**:
- 请求格式与上游输出匹配
- 输出格式与下游期望匹配

**评分标准**:
- ✅ 一致：格式匹配
- ⚠️ 模糊：格式描述不精确
- ❌ 不一致：格式明显不匹配

---

## 安全边界

### PRM-009: 可变/不可变区域

**检查**: 明确改进模式的安全边界

**必需元素**（仅适用于协调器和执行层 Skill）:
```markdown
**安全边界**:
- 不可变区: [列表]
- 可变区: [列表]
```

**评分标准**:
- ✅ 完整：两个区域都有定义
- ⚠️ 部分：只定义了一个区域
- ❌ 缺失：没有安全边界定义

---

## 文档质量

### PRM-010: 章节结构

**检查**: SKILL.md 必须包含标准章节

**必需章节**:
1. 元数据（name, description, version）
2. 标题和职责声明
3. When to Use This Skill
4. 核心内容（流程/模板/检查项）
5. Reference Files

**评分标准**:
- ✅ 完整：包含所有必需章节
- ⚠️ 部分：缺少 1-2 个章节
- ❌ 不完整：缺少多个关键章节

---

### PRM-011: 术语一致性

**检查**: 使用的术语与其他 Skill 一致

**术语表**:
| 术语 | 标准写法 |
|------|----------|
| 组件 | Component |
| 实体 | Entity |
| 事件 | Event |
| 助手 | Helper |
| 层级 | Layer X (L1-L5) |
| 协调器 | Orchestrator |
| 调度 | Dispatch |

**评分标准**:
- ✅ 一致：使用标准术语
- ⚠️ 部分一致：偶尔使用非标准术语
- ❌ 不一致：大量非标准术语

---

## 检查流程

### 逐文件检查

对每个 `SKILL.md` 文件执行：

1. **PRM-001/002**: 检查职责声明
2. **PRM-003/004**: 检查输入输出定义
3. **PRM-005/006**: 检查示例数量和质量
4. **PRM-007/008**: 检查协作接口
5. **PRM-009**: 检查安全边界（如适用）
6. **PRM-010/011**: 检查文档结构和术语

### 跨 Skill 检查

完成单文件检查后：

1. 验证术语一致性
2. 验证协作接口匹配
3. 验证格式规范统一

### 问题记录格式

```markdown
### [PRM-XXX] Skill名称
**问题**: [问题描述]
**当前**: [当前状态]
**建议**: [改进建议]
```
