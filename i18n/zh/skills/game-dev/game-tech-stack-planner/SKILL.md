# game-tech-stack-planner Skill

根据游戏类型、平台需求和团队能力，选择最合适的游戏引擎、框架、工具链和技术方案。

## When to Use This Skill

触发此 Skill 当：
- 系统设计完成，需要选择技术方案
- 需要评估不同引擎/框架
- 需要确定开发工具链
- orchestrator 调度到技术规划阶段

## Not For / Boundaries

此 Skill 不负责：
- 游戏设计内容（需要设计文档作为输入）
- 具体代码架构（交给 implementation-planner）
- 项目进度规划（交给 implementation-planner）
- 实际代码编写（交给开发阶段）

必需输入：
- `@concept.md` - 游戏概念（了解类型和规模）
- `@systems-breakdown.md` - 系统设计（了解技术需求）

## Quick Reference

### 技术选型流程

```
1. 分析游戏需求（类型、平台、规模）
2. 识别技术约束（团队、时间、预算）
3. 评估候选技术栈
4. 选择并论证技术方案
5. 输出 @tech-stack.md
```

### 需求分析矩阵

| 需求维度 | 问题 |
|----------|------|
| **平台** | 目标平台？跨平台需求？ |
| **图形** | 2D/3D？画面质量要求？ |
| **联网** | 单机/联机？实时性要求？ |
| **规模** | 内容量？同时在线？ |
| **周期** | 开发时间？迭代计划？ |
| **团队** | 技术栈经验？人员规模？ |

### 主流引擎对比

| 引擎 | 适用场景 | 优势 | 劣势 |
|------|----------|------|------|
| **Unity** | 2D/3D 全平台 | 生态丰富、学习资源多 | 运行时费用、包体大 |
| **Unreal** | AAA 3D 游戏 | 画面强、蓝图可视化 | 学习曲线陡、硬件要求高 |
| **Godot** | 独立游戏、2D | 开源免费、轻量 | 3D 较弱、生态小 |
| **UEFN** | Fortnite 岛屿 | 即开即用、平台流量 | 仅限 Fortnite 平台 |
| **Cocos** | 2D/轻度 3D | 国内生态好、Web 友好 | 3D 能力有限 |
| **Phaser** | 2D Web 游戏 | 纯 JS、快速原型 | 仅 Web、性能受限 |

### 平台技术约束

| 平台 | 约束 | 推荐技术 |
|------|------|----------|
| **Web** | 包体、内存、无插件 | Phaser、Cocos、PlayCanvas |
| **Mobile** | 电量、发热、碎片化 | Unity、Cocos、原生 |
| **PC** | 较少约束 | Unity、Unreal、Godot |
| **Console** | 认证、SDK | Unity、Unreal |
| **VR/AR** | 性能、交互 | Unity、Unreal |
| **Fortnite** | 仅 UEFN | Verse 语言 |

### 技术选型决策树

```
游戏类型?
├─ 2D 休闲
│   └─ 平台?
│       ├─ Web → Phaser / Cocos Creator
│       └─ Mobile → Unity 2D / Cocos
├─ 3D 游戏
│   └─ 画面要求?
│       ├─ AAA 级 → Unreal Engine
│       └─ 普通 → Unity
├─ Fortnite 岛屿
│   └─ UEFN + Verse
└─ 快速原型
    └─ 经验?
        ├─ 有 Unity 经验 → Unity
        └─ 无经验 → Godot
```

## Examples

### Example 1: UEFN 放置养成游戏

**输入**: 
- @concept.md: Fortnite 岛屿放置养成游戏
- @systems-breakdown.md: 收集/展示/经济/进度系统

**分析**:
- 平台限定：Fortnite
- 类型：放置养成
- 联网：Fortnite 内置多人

**选型结果**:
```
引擎：UEFN
语言：Verse
存储：Fortnite 持久化 API
UI：UEFN 原生 UI 系统
```

**输出**: @tech-stack.md

### Example 2: 独立 2D 游戏

**输入**: 
- @concept.md: 2D 像素风 Roguelike
- @systems-breakdown.md: 战斗/物品/地图生成系统

**分析**:
- 平台：PC + Steam
- 类型：2D 动作
- 团队：独立开发者

**选型结果**:
```
引擎：Godot 4.x
语言：GDScript
存储：本地 JSON
音效：FMOD（可选）
```

**输出**: @tech-stack.md

### Example 3: 多人 Web 游戏

**输入**: 
- @concept.md: 实时多人 Web 游戏
- @systems-breakdown.md: 匹配/战斗/社交系统

**分析**:
- 平台：Web 浏览器
- 联网：实时多人
- 规模：中等

**选型结果**:
```
引擎：Phaser 3
语言：TypeScript
服务端：Node.js + Socket.io
数据库：Redis + PostgreSQL
部署：Docker + AWS
```

**输出**: @tech-stack.md
