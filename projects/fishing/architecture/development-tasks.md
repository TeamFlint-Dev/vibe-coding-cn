# 🎮 Fishing Game - 开发任务详细拆解

> **版本**：v1.0  
> **创建时间**：2026-01-13  
> **任务总数**：120+ 设计与开发任务  
> **预计周期**：3-4 周

---

## 📋 任务拆解总览

本文档将 Fishing 游戏的开发工作拆解为 **120+ 个具体任务**，涵盖：
- **设计任务** (Phase D)：感受设计、氛围营造、叙事包裹
- **开发任务** (Phase M)：技术实现、系统集成、数据持久化
- **体验任务** (Phase X)：UX 流程、反馈设计、打磨优化
- **测试任务** (Phase T)：功能验证、平衡调优、性能测试

---

## 🎯 任务分类框架

```
┌─────────────────────────────────────────────────────────────┐
│  D - Design (设计)      │  M - Mechanics (机制)            │
│  X - Experience (体验)   │  T - Testing (测试)              │
│  U - UI/UX (界面体验)    │  P - Polish (打磨)               │
│  A - Audio (音效)        │  V - Visual (视觉)               │
└─────────────────────────────────────────────────────────────┘
```

## 🌊 Phase 1: 核心感受设计 (20 任务)

### D1: 钓鱼感受设计 (5 任务)

**D1.1 - 定义钓鱼核心感受**
- 输出：钓鱼时刻的目标感受定义文档
- 感受维度：期待感、惊喜、节奏控制
- 持续时间分析：投竿 (0.5s) → 等待 (3-15s) → 咬钩 (2s) → 收杆 (2s)

**D1.2 - 等待阶段体验设计**
- 设计目标：将等待变为期待而非无聊
- 氛围元素：水面涟漪、环境音、浮标轻微晃动
- 互动元素：玩家可观察周围、自由移动视角

**D1.3 - 咬钩瞬间设计**
- 目标感受：强烈的提示 + 紧迫感
- 多通道反馈：视觉（屏幕闪光）、听觉（咬钩音）、触觉（震动）
- 时间窗口设计：2 秒内收杆，容错设计

**D1.4 - 收杆成功仪式**
- 目标感受：成就感、满足感
- 仪式阶段：收杆动画 (1s) → 鱼跃出水面 (1s) → 获得弹窗 (2s)
- 稀有度差异：普通鱼简洁，稀有鱼华丽

**D1.5 - 钓鱼失败处理**
- 目标感受：轻微失望但不挫败，鼓励再试
- 反馈设计：简短提示 "鱼跑了..."，无惩罚
- 立即引导：自动返回 IDLE，可立即重试

### D2: 展示感受设计 (5 任务)

**D2.1 - 定义展示核心感受**
- 目标感受：自豪、炫耀、收藏成就感
- 场景设计：玩家基地中的战利品展示馆
- 社交价值：为未来社交访问预留设计空间

**D2.2 - 放置仪式设计**
- 目标感受：珍贵物品的郑重感
- 仪式流程：选择鱼 → 预览位置 → 确认放置 → 光芒特效
- 视觉反馈：鱼模型淡入 + 铭牌显示 + 展示台光效

**D2.3 - 展示位美学设计**
- 视觉风格：展示台设计（玻璃罩 / 底座 / 灯光）
- 空间布局：3 个展示位的位置、朝向、间距
- 稀有度视觉差异：Common 简洁，Epic 华丽

**D2.4 - 收益可视化设计**
- 目标感受：持续的满足感，能看到"钱在涨"
- 视觉设计：展示位上方浮现金币图标
- 动画节奏：每秒一次，柔和不打扰

**D2.5 - 展示优化决策设计**
- 目标感受：策略思考的乐趣
- UI 支持：显示替换前后收益对比
- 引导提示：建议玩家替换低价值鱼

### D3: 升级感受设计 (4 任务)

**D3.1 - 定义升级核心感受**
- 目标感受：能力提升、进步感、期待下一级
- 关键时刻：升级按钮可用 → 确认升级 → 能力提升
- 频率设计：早期快（正反馈），后期慢（延长内容）

**D3.2 - 升级前的期待感**
- 设计目标：让玩家渴望升级
- 提示设计：金币达标时，升级按钮发光脉动
- 预览设计：显示下一级效果，激发期待

**D3.3 - 升级瞬间仪式**
- 目标感受：强烈的成就感
- 特效设计：金币飞向鱼竿图标 → 等级数字变化 → 光芒闪烁
- 音效设计：激昂的升级音效

**D3.4 - 升级后的能力感知**
- 设计目标：让玩家感受到变强了
- 直接反馈：稀有鱼开始频繁出现
- 数据对比：显示升级前后稀有掉落率

### D4: 收集感受设计 (3 任务)

**D4.1 - 定义收集核心感受**
- 目标感受：完整性追求、发现的喜悦
- 图鉴设计：展示已发现/未发现鱼种
- 进度可视化：X / 5 种鱼已发现

**D4.2 - 首次发现仪式**
- 目标感受：发现新物种的惊喜
- 特殊标识：获得弹窗显示 "NEW!" 
- 奖励设计：首次发现奖励金币

**D4.3 - 图鉴完成仪式**
- 目标感受：收集完成的巨大成就感
- 仪式设计：图鉴 100% 弹窗 + 额外奖励
- 后续引导：暗示未来更多内容

### D5: 离线收益感受设计 (3 任务)

**D5.1 - 回归欢迎设计**
- 目标感受：被欢迎、进度未停止
- 弹窗设计："欢迎回来！" + 离线时长 + 收益
- 收益展示：详细列出每条鱼的贡献

**D5.2 - 离线收益的满足感**
- 设计目标：即使不在线也有收获
- 视觉设计：金币增长动画
- 上限说明：透明展示上限机制

**D5.3 - 优化引导设计**
- 设计目标：激励玩家优化展示配置
- 提示内容："放置更高稀有度的鱼可提升离线收益！"
- 快捷入口：提供"查看展示位"按钮

## 🏗️ Phase 2: 系统机制实现 (40 任务)

### M1: 钓鱼系统 (8 任务)

**M1.1 - 钓鱼状态机实现**
- 技术：Verse 状态机
- 状态：IDLE → CASTING → WAITING → HOOKED → SUCCESS / FAIL
- 输出：fishing_system.verse

**M1.2 - 钓点交互检测**
- 技术：Trigger Device + Verse
- 功能：玩家靠近显示提示，按 E 触发钓鱼
- 输出：fishing_spot_device.verse

**M1.3 - 投竿动画与反馈**
- 技术：Character Animation + Audio
- 动画：玩家投竿动作 (0.5s)
- 音效：甩竿声

**M1.4 - 等待阶段计时器**
- 技术：Verse Timer / Sleep
- 逻辑：3-15s 随机等待时间
- 可中断：Esc 取消钓鱼

**M1.5 - 咬钩事件触发**
- 技术：Event System
- 触发：随机时间后触发咬钩
- 多通道反馈：音效 + 视觉 + 震动

**M1.6 - 收杆判定逻辑**
- 技术：Input Detection + Timer
- 成功：2s 内按 E → SUCCESS
- 失败：超时 → FAIL → 返回 IDLE

**M1.7 - 掉落表系统**
- 技术：加权随机算法
- 数据：fish_database.verse
- 公式：基础权重 × (1 + RodLevel × 0.05)

**M1.8 - 鱼种数据配置**
- 技术：DataTable / Struct
- 数据：5 种鱼（ID, Name, Rarity, BaseIncome, Weight）
- 输出：fish_data.verse

### M2: 战利品系统 (7 任务)

**M2.1 - 背包数据结构**
- 技术：persistable struct
- 字段：[FishId]Quantity
- 容量：无限（或 99 / 种）

**M2.2 - 背包 CRUD 逻辑**
- 功能：Add, Remove, Get, Clear
- 持久化：每次变更自动保存
- 输出：trophy_system.verse

**M2.3 - 图鉴数据结构**
- 技术：persistable struct
- 字段：[FishId]bool (已发现)
- 功能：记录首次发现

**M2.4 - 首次发现检测**
- 逻辑：检查 Collection[FishId]
- 奖励：触发首发奖励事件
- 输出：OnFirstDiscovery Event

**M2.5 - 稀有度判定逻辑**
- 输入：FishId
- 输出：Rarity, Color, Multiplier
- 配置：constants.verse

**M2.6 - 图鉴 UI 数据提供**
- 功能：返回已发现鱼列表
- 格式：[{FishId, Name, Rarity, Count, IsNew}]
- 输出：GetCollectionData()

**M2.7 - 背包 UI 数据提供**
- 功能：返回背包鱼列表（可排序）
- 排序：按稀有度 / 按收益 / 按数量
- 输出：GetInventoryData(SortBy)

### M3: 基地系统 (7 任务)

**M3.1 - 展示位数据结构**
- 技术：persistable struct
- 字段：[SlotId]?FishId (可空)
- 容量：3 个展示位

**M3.2 - 展示位放置逻辑**
- 功能：PlaceFish(SlotId, FishId)
- 验证：背包有鱼、展示位未占用
- 效果：鱼从背包移除，显示在展示位

**M3.3 - 展示位移除逻辑**
- 功能：RemoveFish(SlotId)
- 效果：鱼返回背包，展示位清空

**M3.4 - 展示位替换逻辑**
- 功能：ReplaceFish(SlotId, NewFishId)
- 效果：旧鱼回背包，新鱼放置

**M3.5 - 展示位 Prop 管理**
- 技术：Creative Prop + Prop Manipulator
- 功能：显示/隐藏鱼 3D 模型
- 输出：display_slot_device.verse

**M3.6 - 展示位视觉渲染**
- 功能：根据 FishId 加载对应模型
- 位置：固定坐标 + 旋转
- 铭牌：显示鱼名 + 稀有度

**M3.7 - 展示位交互入口**
- 技术：Button Device
- 提示："[E] 放置战利品"
- 输出：打开展示管理 UI

### M4: 收益系统 (6 任务)

**M4.1 - 收益计算逻辑**
- 公式：Σ(Fish.BaseIncome × RarityMultiplier)
- 输入：展示位当前鱼列表
- 输出：每秒收益值

**M4.2 - 累积定时器**
- 技术：Verse 主循环 (每秒触发)
- 功能：每秒累加 Gold
- 输出：income_system.verse

**M4.3 - 金币数据持久化**
- 技术：persistable int
- 时机：每次金币变化
- 防止：数据丢失机制

**M4.4 - 收益 UI 实时更新**
- 功能：顶栏金币数字实时刷新
- 动画：数字跳动效果
- 频率：每秒更新

**M4.5 - 离线收益计算**
- 逻辑：(CurrentTime - LastExitTime) × IncomeRate
- 上限：4 小时封顶
- 输出：CalculateOfflineIncome()

**M4.6 - 离线收益领取**
- 功能：弹窗展示 → 确认领取 → 金币到账
- 动画：金币增长动画
- 输出：ClaimOfflineIncome()

### M5: 升级系统 (6 任务)

**M5.1 - 升级数据结构**
- 技术：persistable struct
- 字段：RodLevel, RarityBonus
- 费用表：[Level]Cost

**M5.2 - 升级条件验证**
- 逻辑：金币 >= UpgradeCost[NextLevel]
- 输出：CanUpgrade() : bool

**M5.3 - 升级执行逻辑**
- 功能：扣除金币，等级 +1
- 效果应用：RarityBonus += 5%
- 输出：UpgradeRod()

**M5.4 - 稀有度加成应用**
- 逻辑：在掉落表计算时应用加成
- 公式：Weight × (1 + RarityBonus)
- 位置：fishing_system.verse

**M5.5 - 升级 UI 数据提供**
- 功能：返回当前等级、下一级效果、费用
- 格式：{CurrentLevel, NextEffect, Cost, CanAfford}
- 输出：GetUpgradeInfo()

**M5.6 - 升级反馈触发**
- 事件：OnLevelUp Event
- 效果：特效 + 音效 + 弹窗
- 输出：TriggerUpgradeEffect()

### M6: 数据持久化 (6 任务)

**M6.1 - 玩家存档数据结构**
- 技术：persistable class
- 字段：Gold, RodLevel, Inventory, Collection, DisplaySlots, LastPlayTime
- 输出：player_save_data.verse

**M6.2 - 存档保存逻辑**
- 功能：SavePlayerData(Player)
- 时机：钓鱼、放置、升级、退出
- 输出：player_data_manager.verse

**M6.3 - 存档加载逻辑**
- 功能：LoadPlayerData(Player)
- 时机：玩家进入游戏
- 默认值：新玩家初始数据

**M6.4 - 数据迁移机制**
- 功能：处理版本更新时的数据兼容
- 逻辑：检测数据版本，执行迁移
- 备份：保留旧数据副本

**M6.5 - 数据校验机制**
- 功能：检测数据完整性
- 异常处理：数据损坏时重置为默认
- 日志：记录异常事件

**M6.6 - 时间戳管理**
- 功能：记录 LastPlayTime
- 用途：计算离线收益
- 精度：秒级时间戳

## 🎨 Phase 3: UX 与 UI 实现 (25 任务)

### U1: 新手引导 (6 任务)

**U1.1 - 首次进入引导设计**
- 视觉：钓点发光高亮
- 文字：最小化，仅在必要时显示
- 流程：自然引导，无强制弹窗

**U1.2 - 首次钓鱼提示**
- 触发：靠近钓点
- 提示："[E] 钓鱼" (屏幕中央，3s 淡出)
- 位置：钓点上方浮动

**U1.3 - 首次获得鱼弹窗**
- 内容：鱼 3D 模型 + 名称 + 稀有度 + "NEW!"
- 按钮："好的，继续" (可快速关闭)
- 自动关闭：3 秒后自动关闭

**U1.4 - 首次展示引导**
- 触发：获得首条鱼后
- 视觉：展示位区域发光脉动
- 提示："试试放置鱼到展示位？" (箭头指向)

**U1.5 - 首次升级提示**
- 触发：金币 >= 100
- 提示：屏幕右上角通知 "可以升级鱼竿了！"
- 按钮：升级按钮发光脉动

**U1.6 - 引导完成成就**
- 触发：完成首次升级
- 弹窗："🎉 钓手入门！" + 已掌握功能列表
- 按钮："开始冒险！"

## 🧪 Phase 5: 测试与平衡 (20 任务)

### T1: 功能测试 (8 任务)

**T1.1 - 钓鱼流程测试**
- 验证：完成 10 次钓鱼无报错
- 覆盖：正常流程、失败流程、中断流程
- 通过标准：100% 成功率

**T1.2 - 战利品系统测试**
- 验证：背包正确记录所有鱼
- 测试：添加、移除、查询
- 通过标准：数据准确无误

**T1.3 - 展示系统测试**
- 验证：3 个展示位正常放置/移除/替换
- 测试：边界情况（已满、空位）
- 通过标准：操作无报错

**T1.4 - 收益系统测试**
- 验证：金币累积计算正确
- 测试：单鱼、多鱼、不同稀有度
- 通过标准：收益公式准确

**T1.5 - 升级系统测试**
- 验证：能从 Lv.1 升到 Lv.10
- 测试：费用扣除、效果应用
- 通过标准：每级升级正常

**T1.6 - 存档系统测试**
- 验证：退出重进数据完整
- 测试：金币、等级、背包、展示位
- 通过标准：无数据丢失

**T1.7 - 离线收益测试**
- 验证：离线收益计算正确
- 测试：不同离线时长（1h, 4h, 24h）
- 通过标准：公式准确 + 上限生效

**T1.8 - 稀有度掉落测试**
- 验证：掉落概率符合设计
- 测试：钓鱼 1000 次，统计分布
- 通过标准：误差 < 5%

### T2: 平衡调优 (6 任务)

**T2.1 - 新手体验测试**
- 指标：首次钓鱼 < 30s
- 指标：首次放置 < 2min
- 指标：首次升级 < 5min

**T2.2 - 升级节奏测试**
- 指标：Lv.2 约 5min
- 指标：Lv.5 约 30min
- 指标：Lv.10 约 2h

**T2.3 - 经济平衡测试**
- 验证：产出 > 消耗
- 测试：30min、1h、2h 金币余额
- 调整：费用倍率、收益基数

**T2.4 - 稀有度体验测试**
- 指标：Epic 首获时间 20-60min
- 调整：掉落权重、升级加成
- 通过标准：玩家满意度 > 70%

**T2.5 - 内容消耗速度测试**
- 指标：图鉴完成时间 1-2h
- 指标：满级时间 1.5-2.5h
- 调整：数值参数

**T2.6 - 离线收益平衡测试**
- 验证：离线收益不过高/过低
- 上限：4h 收益 = 1h 在线收益
- 调整：上限时间、收益倍率

### T3: 性能测试 (3 任务)

**T3.1 - 帧率测试**
- 标准：≥ 30 FPS
- 场景：钓鱼、展示位、UI 打开
- 优化：减少粒子数、模型 LOD

**T3.2 - 加载时间测试**
- 标准：< 5 秒
- 测试：首次进入、重新进入
- 优化：资源预加载、延迟加载

**T3.3 - 响应延迟测试**
- 标准：按键响应 < 200ms
- 测试：钓鱼交互、UI 操作
- 优化：减少计算复杂度

### T4: 压力测试 (3 任务)

**T4.1 - 长时间运行测试**
- 时长：连续运行 4 小时
- 监控：内存占用、帧率稳定性
- 通过标准：无崩溃、无明显卡顿

**T4.2 - 频繁操作测试**
- 测试：快速连续钓鱼 100 次
- 监控：系统响应、数据正确性
- 通过标准：无报错、数据准确

**T4.3 - 边界情况测试**
- 测试：金币溢出、背包满、展示位满
- 验证：错误处理、用户提示
- 通过标准：无崩溃、提示清晰
