## 🎯 里程碑详细规划

### M0: 原型验证 (Day 1-5)

**目标**：验证核心技术可行性，确认钓鱼循环体验

```
Day 1-2: 感受设计 + 环境搭建
├─ D1.1-D1.5: 钓鱼感受设计文档
├─ 创建 UEFN 项目
└─ 搭建基础场景（钓点 + 基地）

Day 3-4: 核心系统实现
├─ M1.1: 钓鱼状态机
├─ M1.2: 钓点交互检测
├─ M1.4: 等待计时器
├─ M1.5: 咬钩事件
├─ M1.6: 收杆判定
└─ M1.7: 掉落表系统

Day 5: 集成测试
├─ 完整钓鱼流程验证
├─ 技术风险评估
└─ M0 里程碑评审
```

**交付物**：
- ✅ 钓鱼感受设计文档
- ✅ 可运行的原型场景
- ✅ 技术风险评估报告
- ✅ M1 详细计划

**验收标准**：
- 能完成一次完整钓鱼流程（投竿 → 等待 → 咬钩 → 收杆 → 获得鱼）
- 掉落系统能产出不同稀有度的鱼
- 无阻塞性技术问题

```
Day 16-17: 音效与视觉

├─ 数值调整（掉落率、费用、收益）
├─ T1.8: 稀有度掉落统计测试

├─ T3.1-T3.3: 性能测试（帧率、加载、响应）
├─ T4.1-T4.3: 压力测试
├─ 性能优化（粒子、模型 LOD）
└─ P2.1-P2.4: 音效打磨

Day 21: 流程优化
├─ P4.1-P4.4: 流程打磨
├─ 外部测试（邀请 5-10 人）

├─ 根据测试反馈修复问题
├─ 最终数值调整
└─ M2 里程碑评审
```

**交付物**：
- ✅ 完整音效与视觉特效
- ✅ 平衡测试报告
- ✅ 性能测试报告
- ✅ 外部测试反馈
- ✅ 体验完善版本

**验收标准**：
- 所有功能测试通过
- 帧率 ≥ 30 FPS，加载 < 5s
- 新手 < 30s 完成首次钓鱼

- 外部测试满意度 > 70%

## ⚠️ 风险预警

### 高风险点

| 风险 | 时间点 | 影响 | 应对策略 |
|------|--------|------|----------|
| **Verse 技术限制** | Day 3-4 | M0 延期 | 提前验证关键功能 |
| **数据持久化问题** | Day 10-11 | M1 阻塞 | 多重测试，备份方案 |
| **平衡性问题** | Day 18-19 | 体验不佳 | 提前设计数值表，快速调整 |
| **性能不达标** | Day 20 | 发布延期 | 预留优化时间，简化视觉 |

### 缓冲时间

每个里程碑预留 1-2 天缓冲：
- M0: Day 5 → Day 6-7 缓冲
- M1: Day 15 → Day 16 缓冲
- M2: Day 22 → Day 23 缓冲

## 🎮 玩家体验里程碑

### 新手 5 分钟体验

```
0:00 - 进入游戏
  │
0:30 - 看到钓点发光，走过去
  │
1:00 - 完成首次钓鱼，获得首条鱼
  │
2:00 - 引导放置鱼到展示位
  │
3:00 - 观察到金币开始增长
  │
4:00 - 金币达到 100，引导升级
  │
5:00 - 完成首次升级，理解核心循环 ✅
```

### 30 分钟体验目标

```
玩家在 30 分钟内应该：
✅ 完成 20+ 次钓鱼
✅ 发现 3-4 种鱼（至少 1 种 Uncommon）
✅ 展示位放满（3/3）
✅ 升级到 Lv.4-5
✅ 累积金币 1,000+
✅ 理解优化展示配置的策略
```

### 2 小时体验目标

```
玩家在 2 小时内应该：
✅ 图鉴完成 100%（全 5 种鱼）
✅ 鱼竿升级到 Lv.10（满级）
✅ 展示位全为 Rare+ 鱼
✅ 体验到稀有鱼的震撼感
✅ 完成所有 MVP 内容
```
