## 📝 任务执行规范

### 任务完成标准
每个任务完成后，必须：
1. ✅ 标记任务为已完成
2. 📄 输出对应的文档/代码
3. 🧪 进行基本验证测试
4. 💾 提交到版本控制

### 任务依赖关系
- **串行任务**：必须按顺序完成（如 M1.1 → M1.2）
- **并行任务**：可以同时进行（如 M1 和 U1 可并行）
- **阻塞任务**：标记为 ⭐ 的任务是关键路径

### 问题反馈机制
遇到问题时：
1. 在任务旁边添加 🚫 标记
2. 记录问题描述和尝试过的解决方案
3. 在每日站会中提出讨论
4. 问题解决后移除标记，继续任务
