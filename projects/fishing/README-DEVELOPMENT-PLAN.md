## 🌟 设计方法论

本次任务拆解遵循 **Game Design Studio** 的核心理念：

### 感受驱动设计

所有设计任务都从**目标感受**出发：
- **钓鱼**：期待感、惊喜、节奏控制
- **展示**：自豪、炫耀、收藏成就感
- **升级**：能力提升、进步感、期待下一级
- **收集**：完整性追求、发现的喜悦
- **离线收益**：被欢迎、进度未停止

### 仪式设计

为重要时刻设计了完整的仪式结构：
- **首次钓鱼**：投竿 → 等待 → 咬钩 → 收杆 → 获得
- **首次展示**：选择 → 预览 → 确认 → 光芒特效
- **首次升级**：期待 → 确认 → 能力提升 → 感知变化
- **首次发现**："NEW!" 标识 + 奖励

### 细节偏执

每个交互都设计了多通道反馈：
- **视觉**：动画、特效、光芒
- **听觉**：音效、环境音
- **触觉**：震动（手柄）
- **数据**：即时更新

## 💡 关键原则

### 1. 宁缺毋滥

- 一个完美的机制胜过十个凑合的功能
- 做减法保持纯粹
- 删除不服务核心感受的元素

### 2. 感受优先

- 如果技术限制破坏感受，改技术而非妥协
- 感受是唯一的评判标准

### 3. 细节打磨

- 重要细节必须完美
- 宁可延期也不妥协品质

### 4. 自验证

- 写完代码，自己先跑一遍
- Verse 代码：提交前运行远程编译验证
