## 一、UX 设计原则

### 1.1 核心原则

| 原则 | 说明 | 应用 |
|------|------|------|
| **简洁明了** | 每个界面只传达一个核心信息 | 避免信息过载 |
| **即时反馈** | 每个操作立即给予视觉/音效反馈 | 强化行为关联 |
| **流畅自然** | 减少不必要的确认和等待 | 优化操作路径 |
| **可发现性** | 新功能主动提示，不需要摸索 | 引导式展示 |
| **可预测性** | 系统行为符合玩家预期 | 一致性设计 |

### 1.2 目标玩家 UX 需求

基于 `@concept.md` 中的目标玩家画像：

| 玩家特征 | UX 需求 | 设计应对 |
|----------|---------|----------|
| 12-25 岁 | 熟悉移动/游戏操作 | 简洁的现代 UI |
| 每次 15-30 分钟 | 快速上手，立即游玩 | 无强制教程 |
| 每日 2-3 次 | 离线收益提示 | 回归奖励展示 |
| 休闲玩家 | 低压力，随时可暂停 | 无失败惩罚 |

## 二、新手引导流程

### 2.1 引导设计哲学

**无强制教程,自然引导**：

- 不使用弹窗教程，通过视觉引导 + 自然操作完成学习
- 每个核心功能在首次遇到时自动触发引导
- 玩家可随时跳过/忽略引导

### 2.3 引导设计要点

| 阶段 | 设计原则 | 避免 |
|------|----------|------|
| 首次钓鱼 | 视觉引导为主，文字极少 | 长篇文字说明 |
| 首次展示 | 自然提示，不强制 | 阻断玩家操作 |
| 首次升级 | 功能可用时提示 | 过早告知未解锁功能 |
| 引导结束 | 成就感 + 鼓励探索 | 突然停止反馈 |

## 三、核心循环 UX 设计

### 3.1 钓鱼流程 UX

```text
┌──────────────────────────────────────────────────────────┐
│ 靠近钓点                                                 │
│   ↓                                                      │
│ 显示交互提示 "[E] 钓鱼"（钓点高亮）                      │
│   ↓                                                      │
│ 按 E → 投竿动画（0.5s，不可打断）                        │
│   ↓                                                      │
│ 等待阶段（3-15s）                                        │
│   - 视觉：水面涟漪、浮标晃动                             │
│   - 音效：环境音（水声、鸟鸣）                           │
│   - UI：屏幕中央淡提示"等待鱼咬钩..."                    │
│   - 可打断：玩家可按 Esc 取消（返回 IDLE）               │
│   ↓                                                      │
│ 咬钩事件                                                 │
│   - 音效：咬钩声（明显）+ 震动                           │
│   - 视觉：浮标剧烈晃动 + 屏幕边缘闪光                    │
│   - UI：大号提示 "[E] 收杆！"（脉动，持续 2s）           │
│   ↓                                                      │
│ 收杆判定                                                 │
│   - 成功：2s 内按 E → 收杆动画 → 获得鱼弹窗              │
│   - 失败：超时 → 鱼逃脱提示 "鱼跑了..." → 返回 IDLE      │
│   ↓                                                      │
│ 获得鱼弹窗（如上"首次获得鱼弹窗"）                        │
│   - 快速确认：按 Enter / Space / E 立即关闭              │
│   - 自动关闭：3 秒后自动关闭（避免卡住）                 │
│   ↓                                                      │
│ 返回 IDLE，可继续钓鱼                                    │
└──────────────────────────────────────────────────────────┘
```

**UX 优化点**：

- **响应式提示**：咬钩提示必须明显（视觉 + 音效 + 震动）
- **容错设计**：超时自动关闭弹窗，避免卡死
- **快速循环**：按 E 可快速确认，支持连续钓鱼
- **可中断性**：等待阶段可取消，尊重玩家时间

- **收益预览**：实时显示放置/替换后的总收益变化

### 4.2 信息层级

| 层级 | 信息 | 显示方式 | 优先级 |
|------|------|----------|--------|
| **L1 - 核心数据** | 金币、鱼竿等级 | 顶栏常驻 | 最高 |
| **L2 - 当前状态** | 可交互对象提示 | 屏幕中央/底栏 | 高 |
| **L3 - 反馈信息** | 获得鱼、升级成功 | 弹窗（可快速关闭） | 中 |
| **L4 - 详细数据** | 图鉴、收益详情 | 独立界面（按键打开） | 低 |

```text
主界面
  │
  ├─ [E] 钓点 → 钓鱼流程
  ├─ [E] 展示位 → 展示管理界面
  ├─ [顶栏] 升级按钮 → 升级商店
  ├─ [Tab] 图鉴 → 图鉴界面
  └─ [I] 背包 → 背包界面（与展示管理合并）
```

**导航原则**：

- 所有功能≤2 次点击可达
- 常用功能（钓鱼、展示）直接交互，无需菜单
- 辅助功能（图鉴）快捷键调出

**离线收益规则**（参考 `@economy.md`）：

- MVP 可能不实现复杂离线计算，可简化为固定上限
- 如实现：离线时间 × 展示收益速率，设置合理上限（如 4 小时封顶）
- 引导优化：提示玩家优化展示配置

### 6.2 回归激励

| 离线时长 | 奖励 | 目的 |
|----------|------|------|
| < 1 小时 | 无特殊奖励 | 不打断流程 |
| 1-4 小时 | 离线收益弹窗 | 展示进展 |
| > 4 小时 | 离线收益 + 小奖励（如 +5% 收益加成 1 小时） | 鼓励回归 |
| > 24 小时 | 每日登录奖励（如金币或特殊鱼饵） | 保持活跃 |

## 七、错误处理与边界情况

### 7.1 常见错误场景

| 场景 | 错误处理 | UX 反馈 |
|------|----------|---------|
| 钓鱼时断线 | 保存钓鱼状态，重连后恢复 | "连接中断，正在重连..." |
| 背包已满 | 自动移除最低价值鱼 | 提示："背包已满，低价值鱼已自动回收" |
| 展示位已满且背包已满 | 钓鱼后自动回收 | 提示："展示位和背包已满，鱼已自动回收为金币" |
| 金币不足升级 | 按钮禁用 + 提示 | "金币不足，还需 XXX 金币" |
| 服务器错误 | 重试机制 | "操作失败，正在重试..." |

### 7.2 极端情况处理

| 情况 | 处理方式 | UX 设计 |
|------|----------|---------|
| 玩家 AFK 钓鱼中 | 超过 30s 未收杆自动取消 | 返回 IDLE，无惩罚 |
| 玩家快速连续钓鱼 | 防刷机制（如最小间隔 1s） | 自然延迟，无明显提示 |
| 玩家离线超长（7 天+） | 离线收益上限 | 提示上限并鼓励常回来 |
| 首次进入网络不佳 | 缓存本地数据优先 | "离线模式，部分功能受限" |
