## 一、进度系统概述

### 1.1 设计目标

- **清晰的成长感**：玩家始终知道下一步目标是什么
- **合理的节奏**：既不过快失去挑战，也不过慢失去动力
- **多维度进度**：收集、升级、收益三线并行，互相促进

### 1.2 进度维度

| 维度 | 指标 | 目标 |
|------|------|------|
| **收集进度** | 图鉴完成度 | 发现全部 5 种鱼 |
| **能力进度** | 鱼竿等级 | 升级到 Lv.10 |
| **经济进度** | 金币累积 | 满足升级和扩展需求 |
| **展示进度** | 展示位利用率 | 3 个展示位全部使用 |

## 二、玩家成长路径

### 2.1 成长阶段划分

```text
新手期 (0-15min)
    │
    ├─ 完成首次钓鱼
    ├─ 发现 Common 鱼
    ├─ 放置首条鱼到展示位
    ├─ 理解收益机制
    └─ 完成首次升级 (Lv.2)
    │
    ▼
成长期 (15-45min)
    │
    ├─ 展示位放满 (3/3)
    ├─ 发现 Uncommon 和 Rare 鱼
    ├─ 升级到 Lv.5-6
    ├─ 优化展示配置（替换低价值鱼）
    └─ 累积金币 1,000+
    │
    ▼
中期 (45-90min)
    │
    ├─ 发现 Epic 鱼
    ├─ 升级到 Lv.8-9
    ├─ 图鉴完成度 80%+
    └─ 累积金币 5,000+
    │
    ▼
后期 (90-120min)
    │
    ├─ 图鉴 100% 完成
    ├─ 鱼竿满级 (Lv.10)
    ├─ 展示位全为 Rare+ 鱼
    └─ MVP 内容完成
```

### 2.2 关键里程碑

| 里程碑 | 预期时间 | 触发条件 | 奖励 |
|--------|----------|----------|------|
| **首次钓鱼** | 1 分钟 | 完成新手引导 | - |
| **首次展示** | 3 分钟 | 放置首条鱼 | 收益开始累积 |
| **首次升级** | 5 分钟 | 升级鱼竿到 Lv.2 | +5% 稀有掉落 |
| **展示位满** | 15 分钟 | 3 个展示位全部使用 | 解锁成就提示 |
| **发现稀有** | 30 分钟 | 首次获得 Rare 鱼 | +100 金币 |
| **中级钓手** | 45 分钟 | 鱼竿 Lv.6 | +25% 稀有掉落 |
| **发现传说** | 60 分钟 | 首次获得 Epic 鱼 | +500 金币 |
| **图鉴大师** | 90 分钟 | 图鉴 100% 完成 | +500 金币（全发现奖励） |
| **满级钓手** | 120 分钟 | 鱼竿 Lv.10 | MVP 完成标志 |

## 三、解锁节奏设计

### 3.1 能力解锁路线

```text
Lv.1 (初始)
  │
  ├─ 可钓鱼种：Common (69.6%)
  ├─ 展示位：3 个
  └─ 钓点：单一钓点
  │
  ▼
Lv.2-3 (早期, 5-10min)
  │
  ├─ 稀有提升：+5% → +10%
  ├─ Uncommon 概率提升
  └─ 收益加速
  │
  ▼
Lv.4-6 (成长, 15-30min)
  │
  ├─ 稀有提升：+15% → +25%
  ├─ Rare 鱼开始频繁出现
  └─ 收益显著增长
  │
  ▼
Lv.7-9 (中期, 45-90min)
  │
  ├─ 稀有提升：+30% → +40%
  ├─ Epic 鱼可获取
  └─ 收益进入高速期
  │
  ▼
Lv.10 (满级, 120min)
  │
  ├─ 稀有提升：+45%
  ├─ Epic 概率最大化
  └─ MVP 内容完成
```

### 3.2 内容解锁时间表

| 内容 | 解锁条件 | 预期时间 |
|------|----------|----------|
| **钓鱼功能** | 游戏开始 | 0 分钟 |
| **展示功能** | 获得首条鱼 | 1-2 分钟 |
| **升级功能** | 累积 100 金币 | 3-5 分钟 |
| **Uncommon 鱼** | Lv.1 即可（低概率） | 5-10 分钟 |
| **Rare 鱼** | Lv.1 即可（极低概率） | 10-30 分钟 |
| **Epic 鱼** | Lv.1 即可（罕见） | 30-90 分钟 |
| **图鉴完成** | 收集全 5 种鱼 | 60-120 分钟 |

**设计理念**：

- 所有内容从 Lv.1 即可获得，升级只是提升概率
- 避免硬性解锁造成的挫败感
- 早期也有概率钓到稀有鱼，保持惊喜感

## 五、进度体验设计

### 5.1 新手期体验 (0-15min)

**目标**：理解核心循环，获得初步成就感

| 时间 | 玩家行为 | 系统反馈 | 进度标志 |
|------|----------|----------|----------|
| 0-1min | 首次钓鱼 | 动画 + 获得提示 | "钓到第一条鱼！" |
| 1-3min | 首次展示 | 收益开始累积 | "展示馆已激活" |
| 3-5min | 累积金币 | 金币数字跳动 | "100 金币达成" |
| 5-10min | 首次升级 | 鱼竿视觉变化 | "Lv.2 钓手" |
| 10-15min | 展示位满 | 基地视觉丰富 | "展示馆已满" |

**关键设计**：

- 每 1-3 分钟必有新进展
- 视觉/音效反馈明确
- 引导自然，无强制教程

### 5.2 成长期体验 (15-45min)

**目标**：优化策略，追求更高价值

| 阶段 | 玩家决策 | 策略深度 |
|------|----------|----------|
| 展示优化 | 是否替换低价值鱼？ | 收益最大化 |
| 升级节奏 | 攒钱一次性升级 vs 逐级升级？ | 风险偏好 |
| 目标选择 | 追求图鉴 vs 追求收益？ | 游戏风格 |

**进度感知**：

- 稀有鱼开始频繁出现
- 金币累积速度明显加快
- 展示馆配置多次调整

### 5.3 后期体验 (45-120min)

**目标**：完成收集，达成满级

| 里程碑 | 成就感来源 | 后续动力 |
|--------|-----------|----------|
| 首次 Epic | 稀有鱼视觉震撼 | 继续钓更多 Epic |
| 图鉴完成 | 收集完整性 | 优化展示配置 |
| 鱼竿满级 | 能力封顶 | 体验最强掉落率 |

**进度终点**：

- MVP 不设无限内容
- 明确告知"当前版本完成"
- 预告后续更新内容

## 七、进度平衡验证

### 7.1 时间投入预期

| 玩家类型 | 游玩时长 | 完成进度 | 满意度预期 |
|----------|----------|----------|-----------|
| 休闲玩家 | 15-30 分钟/次 | Lv.5, 图鉴 60% | 高（有明确进展） |
| 核心玩家 | 1-2 小时/次 | Lv.10, 图鉴 100% | 中高（内容适中） |
| 完美主义者 | 2+ 小时 | 全成就达成 | 中（期待更多内容） |

### 7.2 进度卡点检查

| 可能卡点 | 风险 | 缓解措施 |
|----------|------|----------|
| 金币不足升级 | 低 | 收益远大于消耗 |
| Epic 鱼太难钓 | 中 | 升级显著提升概率 |
| 展示位不够用 | 低 | MVP 只需 3 个位置 |
| 内容消耗过快 | 中 | 预告后续更新 |

### 7.3 进度节奏调优参数

| 参数 | 当前值 | 调整范围 | 影响 |
|------|--------|----------|------|
| 升级费用乘数 | 1.5 | 1.3-2.0 | 升级速度 |
| 稀有度加成 | +5%/级 | +3%-+8%/级 | 收集速度 |
| 展示收益基数 | 1/2/5/15 | ×0.5-2.0 | 金币速度 |
| 首发奖励倍率 | 10/25/100/500 | ×0.5-2.0 | 早期资金 |

## 九、UEFN 实现要点

### 9.1 进度数据结构

```verse
player_progression := class:
    # 收集进度
    var CollectionProgress : [string]int = map{}  # FishId -> Count
    var DiscoveredFish : []string = array{}       # 已发现的鱼种
    
    # 能力进度
    var RodLevel : int = 1
    var TotalFishCaught : int = 0
    
    # 经济进度
    var TotalGoldEarned : int = 0
    var TotalGoldSpent : int = 0
    
    # 展示进度
    var DisplaySlotsFilled : int = 0
    
    # 里程碑
    var AchievedMilestones : []string = array{}
```

### 9.2 进度持久化

```verse
# 保存进度到玩家数据
SaveProgression(Player : agent) : void =
    PlayerData.Set(Player, "RodLevel", RodLevel)
    PlayerData.Set(Player, "Collection", CollectionProgress)
    PlayerData.Set(Player, "Milestones", AchievedMilestones)
    ...

# 加载进度
LoadProgression(Player : agent) : void =
    set RodLevel = PlayerData.Get(Player, "RodLevel", 1)
    set CollectionProgress = PlayerData.Get(Player, "Collection", map{})
    ...
```

### 9.3 里程碑触发器

```verse
# 检查里程碑
CheckMilestones(Player : agent) : void =
    if RodLevel >= 2 and not HasMilestone("first_upgrade"):
        TriggerMilestone(Player, "first_upgrade", "首次升级")
    
    if DiscoveredFish.Length >= 5 and not HasMilestone("collection_complete"):
        TriggerMilestone(Player, "collection_complete", "图鉴大师")
        AwardGold(Player, 500)
```
